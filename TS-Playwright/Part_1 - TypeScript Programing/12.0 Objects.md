## Что такое объект — реальный мир и программирование

В реальном мире **всё вокруг — это объект**. Стул, телефон, машина, человек, животное. Каждый объект имеет две стороны:

**Свойства (properties)** — характеристики, описывающие объект. Для человека это: имя, возраст, рост, цвет волос. Для машины: цвет, марка, год выпуска, объём двигателя.

**Поведение (behavior)** — что объект умеет делать. Человек умеет: ходить, говорить, есть, работать. Машина умеет: ехать, тормозить, сигналить.

В программировании точно такая же идея:

- Свойства = **переменные** (хранят данные)
- Поведение = **методы** (функции внутри объекта)

Пример из жизни тестировщика. Возьмём объект "Тест":

- Свойства: название теста, статус (passed/failed), время выполнения, URL страницы
- Поведение: запустить тест, проверить результат, сделать скриншот, залогировать ошибку

## Три ключевых определения объекта

Лектор даёт три определения, каждое подходит для разного контекста:

**Определение 1 (общее, для всех языков):** Объект — это набор свойств и поведения.

**Определение 2 (Java, Python, C++):** Объект — это набор переменных и методов. Обязательно создаётся через класс.

**Определение 3 (JavaScript и TypeScript — самое важное для нас):** Объект — это **коллекция пар ключ-значение** (key-value pairs). Можно создать без класса.

Третье определение — это то, что ты будешь использовать каждый день в TypeScript. Ключ — это имя свойства, значение — его содержимое:

```typescript
// Каждая строчка внутри {} — это пара ключ: значение
let employee = {
    name: "John",       // ключ: name, значение: "John"
    age: 30,            // ключ: age, значение: 30
    salary: 50000,      // ключ: salary, значение: 50000
    job: "Engineer"     // ключ: job, значение: "Engineer"
};
```

## Свойства и методы — в чём разница

Внутри объекта могут жить два вида вещей:

**Свойства** — хранят данные (строки, числа, булевы и т.д.):

```typescript
let user = {
    name: "Anna",       // свойство: строка
    age: 28,            // свойство: число
    isActive: true      // свойство: булево
};
```

**Методы** — хранят функции (поведение объекта). Метод — это просто функция, которая живёт внутри объекта:

```typescript
let user = {
    name: "Anna",
    age: 28,
    // метод: функция внутри объекта
    greet: function() {
        console.log("Hello, I'm " + this.name);
    }
};
```

Разница между функцией и методом — только место жительства. Функция живёт сама по себе. Метод живёт внутри объекта. По сути это одно и то же.

## Четыре способа создать объект — обзор

```
Способы создания объекта в TypeScript:
│
├── 1. Object Type        → прямое присваивание, JS + TS
├── 2. Inline Type Object → явные типы при объявлении, только TS
├── 3. Type Alias         → переиспользуемый тип, только TS
└── 4. Class              → полноценное ООП, JS (ES6+) + TS ★
                                                             ↑
                           В Playwright используется чаще всего
```

Практический совет от лектора:

- В реальной автоматизации используй **способ 4 (Class)** — это стандарт
- Способ 1 (прямое присваивание) — второй по популярности, удобен для простых данных
- Способы 2 и 3 — важны для **понимания TypeScript** и для **интервью**
- Знай все четыре — тебя точно спросят на собеседовании

## СПОСОБ 1 — Object Type прямое присваивание

### Синтаксис Object Type

Самый простой способ. Берёшь переменную, присваиваешь ей объект через фигурные скобки `{}`, внутри указываешь пары ключ-значение через двоеточие:

```typescript
// Синтаксис: ключ: значение, разделяются запятыми
let employee = {
    name: "John",
    age: 30,
    salary: 50000,
    job: "Engineer"
};

// Можно и в одну строку
let point = { x: 10, y: 20 };
```

Обрати внимание: внутри объекта используется `:` (двоеточие), а **не** `=`. Это принципиальная разница — ты не присваиваешь переменную, ты описываешь свойство объекта.

TypeScript сам определит типы через type inference — `name` будет `string`, `age` будет `number`. Это работает и в JavaScript, и в TypeScript.

### Методы внутри объекта и this

Кроме свойств внутри объекта можно создавать методы. Метод — это ключ, значение которого является функцией:

```typescript
let employee = {
    name: "John",
    age: 30,
    salary: 50000,
    job: "Engineer",

    // метод: ключ = имя метода, значение = функция
    getDetails: function() {
        // this — это ссылка на текущий объект (employee)
        console.log(this.name, this.job, this.salary);
    }
};
```

**Важно: ключевое слово `this`**

Внутри метода нельзя обратиться к свойствам объекта просто по имени. Это вызовет ошибку:

```typescript
getDetails: function() {
    console.log(name);    // ОШИБКА: name не определён
    console.log(age);     // ОШИБКА: age не определён
}
```

Нужно использовать `this.` — это означает "обратись к текущему объекту":

```typescript
getDetails: function() {
    // this.name — возьми свойство name у текущего объекта
    return `${this.name} is ${this.job} earning ${this.salary}`;
}
```

Аналогия: `this` — как слово "мой" в человеческой речи. "Моё имя — John, моя работа — Engineer". Без "моё" непонятно чьё имя имеется в виду.

### Ограничение ключевого слова object

Если специально указать тип `object` при объявлении — TypeScript заблокирует доступ к свойствам:

```typescript
// ПРОБЛЕМА: явное указание типа object
let employee: object = {
    name: "John",
    salary: 50000
};

// ОШИБКА при попытке доступа к свойствам:
console.log(employee.name);    // Property 'name' does not exist on type 'object'
console.log(employee.salary);  // Property 'salary' does not exist on type 'object'
```

Почему? Потому что TypeScript видит только тип `object` (который может быть чем угодно) и не знает, что именно внутри.

Решение — **не указывать тип вручную**, позволить TypeScript самому вывести тип:

```typescript
// ПРАВИЛЬНО: не указываем : object
let employee = {
    name: "John",    // TS видит: name — это string
    salary: 50000    // TS видит: salary — это number
};

console.log(employee.name);    // OK: "John"
console.log(employee.salary);  // OK: 50000
```

## Как получить доступ к данным объекта

Существует два способа обратиться к свойствам и методам объекта.

### Dot notation точечная нотация

Самый популярный способ — через точку `.`:

```typescript
let employee = {
    name: "John",
    salary: 50000,
    job: "Engineer",
    getDetails: function() {
        return `${this.name} is ${this.job} earning ${this.salary}`;
    }
};

// Доступ к свойствам
console.log(employee.name);     // John
console.log(employee.salary);   // 50000
console.log(employee.job);      // Engineer

// Вызов метода
console.log(employee.getDetails()); // John is Engineer earning 50000
```

Именно этот способ используется в Playwright постоянно:

```typescript
// Примеры dot notation в реальных тестах:
await page.goto('/login');
await page.fill('#email', 'test@test.com');
await expect(page).toHaveTitle('Dashboard');
//     ↑ page — объект, goto/fill/toHaveTitle — его методы
```

### Bracket notation скобочная нотация

Второй способ — через квадратные скобки `[]`. Внутри скобок указывается **имя ключа в кавычках**:

```typescript
// Доступ к свойствам через скобки
console.log(employee["name"]);    // John
console.log(employee["salary"]);  // 50000
console.log(employee["job"]);     // Engineer

// Вызов метода через скобки
console.log(employee["getDetails"]()); // обрати внимание: () после скобок!
```

Когда bracket notation полезен? Когда имя ключа хранится в переменной:

```typescript
const key = "name";
console.log(employee[key]); // John — имя ключа из переменной
console.log(employee.key);  // undefined — ищет свойство с именем "key"
```

На практике почти всегда используется dot notation. Bracket notation — редкий случай, когда ключ динамический.

### Изменение значений в объекте

Объекты в JavaScript/TypeScript **изменяемы по умолчанию** — можно поменять любое свойство после создания:

```typescript
let employee = {
    name: "John",
    job: "Engineer"
};

console.log(employee.job);   // Engineer

// Изменяем через dot notation
employee.job = "Manager";
console.log(employee.job);   // Manager

// Изменяем через bracket notation — тот же результат
employee["job"] = "Director";
console.log(employee.job);   // Director
```

## СПОСОБ 2 — Inline Type Object явные типы

### Синтаксис Inline Type

Отличается от первого способа тем, что сначала объявляется **структура с типами**, а потом отдельно присваиваются **данные**:

```typescript
// Шаг 1: объявление структуры (декларация типов)
let student: {
    name: string;
    age: number;
    grade: string;
    getSummary: () => string; // объявление метода как arrow function
} =
// Шаг 2: присваивание данных (= и фигурные скобки с данными)
{
    name: "Alex",
    age: 15,
    grade: "A",
    getSummary: function() {
        return `${this.name} is ${this.age} years old and scored ${this.grade}`;
    }
};

console.log(student.getSummary()); // Alex is 15 years old and scored A
```

Ключевые отличия от способа 1:

- Типы указываются **явно** (`name: string`, `age: number`)
- Структура и данные **разделены** — сначала `:{}` с типами, потом `= {}` с данными
- Внутри структуры свойства разделяются **точкой с запятой** `;`
- Работает только в TypeScript (в JavaScript типов нет)

### Декларация vs Присваивание

Аналогия с обычной переменной помогает понять разницу:

```typescript
// Обычная переменная:
let x: number;   // декларация — объявляем что x это число
x = 42;          // присваивание — даём конкретное значение

// Объект в inline стиле:
let student: { name: string; age: number } = // декларация структуры
{                                              // присваивание данных
    name: "Alex",
    age: 15
};
```

### Проблема Inline Type — повторяемость

Если нужно создать несколько объектов с одинаковой структурой — придётся копировать структуру каждый раз:

```typescript
// Объект 1 — структуру пишем полностью
let student1: {
    name: string;
    age: number;
    grade: string;
    getSummary: () => string;
} = { name: "Alex", age: 15, grade: "A", getSummary: function() {...} };

// Объект 2 — структуру снова пишем полностью, та же самая!
let student2: {
    name: string;        // дублирование!
    age: number;         // дублирование!
    grade: string;       // дублирование!
    getSummary: () => string; // дублирование!
} = { name: "Maria", age: 16, grade: "B+", getSummary: function() {...} };
```

Это раздражает, загромождает код и нарушает принцип DRY (Don't Repeat Yourself). Способ 3 решает именно эту проблему.

## СПОСОБ 3 — Type Alias переиспользуемая структура

### Синтаксис Type Alias

Идея: выносим структуру объекта в отдельный именованный тип через ключевое слово `type`. Один раз объявил — используй сколько угодно раз:

```typescript
// Шаг 1: создаём тип (blueprint структуры)
type Product = {
    name: string;
    price: number;
    getInfo: () => string; // метод объявляется как arrow function
};

// Шаг 2: используем тип при создании объектов
let book1: Product = {
    name: "Learn Java",
    price: 300,
    getInfo: function() {
        return `${this.name} costs ${this.price}`;
    }
};

// Тот же тип — другие данные
let book2: Product = {
    name: "Learn Python",
    price: 500,
    getInfo: function() {
        return `${this.name} costs ${this.price}`;
    }
};

// Ещё один объект с той же структурой
let book3: Product = {
    name: "Learn JavaScript",
    price: 600,
    getInfo: function() {
        return `${this.name} costs ${this.price}`;
    }
};
```

Теперь если захочешь добавить поле `author: string` — изменишь только в типе `Product`, и все три объекта сразу получат новое поле.

### Один тип — много объектов

Вызов методов у разных объектов с одним типом:

```typescript
console.log(book1.getInfo()); // Learn Java costs 300
console.log(book2.getInfo()); // Learn Python costs 500
console.log(book3.getInfo()); // Learn JavaScript costs 600
```

**Ключевое отличие от способа 2:** `Product` — это **не объект**, это **тип**. Тип существует только во время компиляции TypeScript. В runtime JavaScript видит только объекты `book1`, `book2`, `book3`.

### Intersection Types пересечение типов

Можно скомбинировать несколько типов в один через `&` (амперсанд — логический И):

```typescript
// Тип 1: личные данные
type PersonalInfo = {
    name: string;
    age: number;
};

// Тип 2: контактные данные
type ContactInfo = {
    email: string;
    phone: number;
};

// Тип 3 = PersonalInfo + ContactInfo + свой метод
type Candidate = PersonalInfo & ContactInfo & {
    getContactInfo: () => string;
};
```

Объект типа `Candidate` обязан содержать **все** свойства из всех трёх типов:

```typescript
let ca: Candidate = {
    name: "Scott",          // из PersonalInfo
    age: 30,                // из PersonalInfo
    email: "scott@mail.com", // из ContactInfo
    phone: 34600000001,     // из ContactInfo
    getContactInfo: function() {
        return `${this.name} can be contacted as ${this.email} and ${this.phone}`;
    }
};

console.log(ca.getContactInfo());
// Scott can be contacted as scott@mail.com and 34600000001
```

Intersection types полезны когда одна сущность имеет несколько "ролей". Например, пользователь системы — одновременно `AuthUser` (логин, пароль) и `ProfileUser` (имя, фото) и `AdminUser` (права).

## СПОСОБ 4 — Class классическое ООП

### Что такое класс — аналогии

Класс — это **чертёж** (blueprint). По одному чертежу можно построить сколько угодно домов. Дома будут разные (разный цвет, мебель, жильцы), но все построены по одному плану.

Ещё аналогия: форма для печенья. Форма (класс) одна, а печенье (объекты) можно сделать сколько угодно — каждое будет иметь одинаковую форму, но разную начинку.

Сравнение с Type Alias:

- Type Alias тоже задаёт структуру и позволяет создавать несколько объектов
- Но Class — это полноценное ООП: есть конструктор, наследование, инкапсуляция
- На практике в Playwright классы используются для Page Object Model (POM)

### Синтаксис класса

```typescript
class Person {
    // Свойства класса — только объявление, через точку с запятой ;
    ssn: string;
    firstName: string;
    lastName: string;

    // Методы класса — без ключевого слова function, без запятых
    getFullName(): string {
        return this.firstName + " " + this.lastName;
    }

    getDetails(): string {
        return `${this.ssn} — ${this.getFullName()}`;
        // метод может вызывать другой метод того же класса!
    }
}
```

Важные отличия синтаксиса класса от объекта:

- Свойства разделяются `;` (точка с запятой), а не `,` (запятой)
- Нет ключевого слова `function` для методов
- Методы можно вызывать друг из друга через `this`

### Constructor — конструктор

Конструктор — специальный метод с именем `constructor`. Он вызывается **автоматически** в момент создания объекта. Его единственная задача — **присвоить переданные данные свойствам класса**.

```typescript
class Person {
    ssn: string;
    firstName: string;
    lastName: string;

    // constructor принимает параметры и раздаёт их свойствам
    constructor(ssn: string, firstName: string, lastName: string) {
        this.ssn = ssn;             // this.ssn = параметр ssn
        this.firstName = firstName; // this.firstName = параметр firstName
        this.lastName = lastName;   // this.lastName = параметр lastName
    }

    getDetails(): string {
        return `${this.ssn} — ${this.firstName} ${this.lastName}`;
    }
}
```

Почему нужен `this.ssn = ssn` а не просто `ssn = ssn`? Потому что у нас два `ssn` — параметр конструктора и свойство класса. Без `this.` компилятор не может понять какое именно `ssn` имеется в виду. `this.ssn` — явно говорит "возьми свойство текущего объекта".

### Ключевое слово this в классе

`this` всегда указывает на **текущий объект**, с которым работаешь. Это ключевая концепция ООП:

```typescript
class Car {
    brand: string;
    speed: number;

    constructor(brand: string, speed: number) {
        this.brand = brand;   // this = создаваемый объект
        this.speed = speed;
    }

    describe(): string {
        // this внутри метода = тот объект, на котором вызван метод
        return `${this.brand} goes at ${this.speed} km/h`;
    }
}

let car1 = new Car("Toyota", 120);
let car2 = new Car("BMW", 200);

// Когда вызываем car1.describe() — this внутри = car1
car1.describe(); // Toyota goes at 120 km/h

// Когда вызываем car2.describe() — this внутри = car2
car2.describe(); // BMW goes at 200 km/h
```

### Создание объектов через new

Для создания объекта из класса используется ключевое слово `new`:

```typescript
class Person {
    ssn: string;
    firstName: string;
    lastName: string;

    constructor(ssn: string, firstName: string, lastName: string) {
        this.ssn = ssn;
        this.firstName = firstName;
        this.lastName = lastName;
    }

    getDetails(): string {
        return `${this.ssn} — ${this.firstName} ${this.lastName}`;
    }
}

// Создаём объекты через new — передаём данные в конструктор
let person1 = new Person("111-111", "John", "Kennedy");
let person2 = new Person("222-222", "David", "Brown");
let person3 = new Person("333-333", "Scott", "Miller");

// Вызываем методы у каждого объекта
console.log(person1.getDetails()); // 111-111 — John Kennedy
console.log(person2.getDetails()); // 222-222 — David Brown
console.log(person3.getDetails()); // 333-333 — Scott Miller
```

Что происходит при `new Person("111-111", "John", "Kennedy")`:

1. TypeScript создаёт новый пустой объект
2. Автоматически вызывается `constructor`
3. Конструктор присваивает переданные данные свойствам объекта
4. Готовый объект сохраняется в переменную `person1`

### Объекты независимы друг от друга

Каждый объект имеет свою **собственную копию** свойств. Изменение одного не влияет на другой:

```typescript
let person1 = new Person("111", "John", "Kennedy");
let person2 = new Person("222", "Anna", "Smith");

// Меняем данные person1
person1.firstName = "Michael";

console.log(person1.getDetails()); // 111 — Michael Kennedy (изменился)
console.log(person2.getDetails()); // 222 — Anna Smith (не тронут!)
```

Все объекты происходят из одного класса, но живут независимо. Класс — это шаблон. Объект — это конкретный экземпляр по этому шаблону.

## Сравнение всех четырёх способов

|Характеристика|Способ 1|Способ 2|Способ 3|Способ 4|
|---|---|---|---|---|
|Название|Object Type|Inline Type|Type Alias|Class|
|Работает в JS|да|нет|нет|да (ES6+)|
|Работает в TS|да|да|да|да|
|Явные типы|нет|да|да|да|
|Переиспользование|нет|нет|да|да|
|Конструктор|нет|нет|нет|да|
|ООП-концепции|нет|нет|нет|да|
|Используется в Playwright|редко|редко|часто|**основной**|
|Важность для интервью|средняя|средняя|высокая|высокая|

**Когда что использовать:**

Способ 1 — быстро создать простой одноразовый объект с данными. Тестовые данные на одно использование.

Способ 2 — когда нужна типизация, но только один объект. Редко используется.

Способ 3 (Type Alias) — когда нужно несколько объектов одной формы без методов, или для типизации входящих данных функций.

Способ 4 (Class) — Page Object Model в Playwright, любой переиспользуемый компонент с поведением.

### Объект как тестовые данные

### Класс как Page Object

Способ 4 — основа Page Object Model. Каждая страница = отдельный класс:

```typescript
import { Page, Locator, expect } from '@playwright/test';

class LoginPage {
    // Свойства — локаторы элементов страницы
    private page: Page;
    private emailInput: Locator;
    private passwordInput: Locator;
    private submitButton: Locator;
    private errorMessage: Locator;

    // Конструктор — инициализирует локаторы
    constructor(page: Page) {
        this.page = page;
        this.emailInput = page.locator('#email');
        this.passwordInput = page.locator('#password');
        this.submitButton = page.locator('[type="submit"]');
        this.errorMessage = page.locator('.error-msg');
    }

    // Методы — действия на странице
    async goto(): Promise<void> {
        await this.page.goto('/login');
    }

    async fillEmail(email: string): Promise<void> {
        await this.emailInput.fill(email);
    }

    async fillPassword(password: string): Promise<void> {
        await this.passwordInput.fill(password);
    }

    async submit(): Promise<void> {
        await this.submitButton.click();
    }

    async getErrorText(): Promise<string> {
        return await this.errorMessage.textContent() ?? '';
    }

    // Составной метод — использует другие методы класса
    async login(email: string, password: string): Promise<void> {
        await this.goto();
        await this.fillEmail(email);
        await this.fillPassword(password);
        await this.submit();
    }
}

// В тестах создаём объект через new и работаем с методами
test('successful login', async ({ page }) => {
    const loginPage = new LoginPage(page);         // создаём объект
    await loginPage.login('user@test.com', 'pass123'); // вызываем метод
    await expect(page).toHaveURL('/dashboard');
});

test('failed login shows error', async ({ page }) => {
    const loginPage = new LoginPage(page);
    await loginPage.login('wrong@test.com', 'wrong');
    const error = await loginPage.getErrorText();
    expect(error).toContain('Invalid credentials');
});
```

### Intersection Types в автоматизации

Полезны когда одна функция работает с объединёнными данными:

```typescript
type BaseTestData = {
    testId: string;
    timestamp: number;
};

type UserTestData = {
    email: string;
    password: string;
};

type LoginTestCase = BaseTestData & UserTestData & {
    expectedUrl: string;
    shouldSucceed: boolean;
};

// Функция принимает объект с данными из всех трёх типов
const runLoginTest = async (page: Page, testCase: LoginTestCase): Promise<void> => {
    console.log(`Running test: ${testCase.testId}`);
    await page.fill('#email', testCase.email);
    await page.fill('#password', testCase.password);
    await page.click('[type="submit"]');

    if (testCase.shouldSucceed) {
        await expect(page).toHaveURL(testCase.expectedUrl);
    } else {
        await expect(page.locator('.error')).toBeVisible();
    }
};
```

## Частые ошибки и как их избежать

**Ошибка 1: двоеточие вместо равно при создании объекта**

```typescript
// ОШИБКА: = внутри объекта — так не работает
let user = {
    name = "John",   // SyntaxError
    age = 30
};

// ПРАВИЛЬНО: двоеточие
let user = {
    name: "John",
    age: 30
};
```

**Ошибка 2: забыл this внутри метода**

```typescript
let user = {
    name: "Anna",
    greet: function() {
        console.log(name);       // undefined или ошибка — нет this
        console.log(this.name);  // ПРАВИЛЬНО: "Anna"
    }
};
```

**Ошибка 3: указал : object и не может достучаться до свойств**

```typescript
// ОШИБКА: явный тип object блокирует доступ к свойствам
let user: object = { name: "Anna" };
console.log(user.name); // Property 'name' does not exist on type 'object'

// ПРАВИЛЬНО: не указываем object, TypeScript сам поймёт
let user = { name: "Anna" };
console.log(user.name); // "Anna"
```

**Ошибка 4: не передал параметры в new при наличии конструктора**

```typescript
class Car {
    brand: string;
    constructor(brand: string) {
        this.brand = brand;
    }
}

let car = new Car();         // ОШИБКА: Expected 1 arguments, but got 0
let car = new Car("Toyota"); // ПРАВИЛЬНО
```

**Ошибка 5: точка с запятой вместо запятой в объекте (не классе)**

```typescript
// ОШИБКА: ; внутри объекта
let user = {
    name: "Anna";   // SyntaxError!
    age: 30;        // SyntaxError!
};

// ПРАВИЛЬНО: , (запятая) в объекте
let user = {
    name: "Anna",
    age: 30
};

// Примечание: ; (точка с запятой) — только в объявлении класса
class User {
    name: string;  // OK в классе
    age: number;   // OK в классе
}
```

**Ошибка 6: попытка вызвать метод без скобок**

```typescript
let user = {
    getName: function() { return "Anna"; }
};

console.log(user.getName);    // [Function: getName] — выведет функцию, не результат
console.log(user.getName());  // "Anna" — ПРАВИЛЬНО
```

## Шпаргалка — весь синтаксис

```typescript
// ─────────────────────────────────────────────────────────
// СПОСОБ 1: Object Type (прямое присваивание)
let obj = {
    prop1: "value",         // строка
    prop2: 42,              // число
    prop3: true,            // булево
    method: function() {    // метод через function
        return this.prop1;  // this — текущий объект
    }
};

// Доступ через dot notation
console.log(obj.prop1);   // value
console.log(obj.method()); // value

// Доступ через bracket notation
console.log(obj["prop1"]); // value

// Изменение свойства
obj.prop1 = "new value";

// ─────────────────────────────────────────────────────────
// СПОСОБ 2: Inline Type Object
let student: {
    name: string;           // объявляем тип свойства
    age: number;
    getInfo: () => string;  // объявляем тип метода
} = {
    name: "Alex",           // присваиваем данные
    age: 15,
    getInfo: function() { return this.name; }
};

// ─────────────────────────────────────────────────────────
// СПОСОБ 3: Type Alias (переиспользуемый тип)
type Product = {
    name: string;
    price: number;
    getInfo: () => string;
};

let item1: Product = { name: "Book", price: 20, getInfo: function() { return this.name; } };
let item2: Product = { name: "Pen", price: 5, getInfo: function() { return this.name; } };

// Intersection Types (объединение нескольких типов)
type A = { x: string };
type B = { y: number };
type AB = A & B & { z: boolean };  // содержит x, y, z

// ─────────────────────────────────────────────────────────
// СПОСОБ 4: Class (ООП)
class Person {
    firstName: string;     // свойства через ; (точка с запятой)
    lastName: string;

    constructor(firstName: string, lastName: string) {  // конструктор
        this.firstName = firstName; // this.свойство = параметр
        this.lastName = lastName;
    }

    getFullName(): string { // метод — без function, без запятой
        return `${this.firstName} ${this.lastName}`;
    }
}

// Создание объекта через new — данные передаём в конструктор
let p1 = new Person("John", "Kennedy");
let p2 = new Person("Anna", "Smith");

// Вызов методов
console.log(p1.getFullName()); // John Kennedy
console.log(p2.getFullName()); // Anna Smith

// ─────────────────────────────────────────────────────────
// В PLAYWRIGHT — типичные паттерны

// Объект с тестовыми данными (способ 1)
const users = {
    admin: { email: "admin@test.com", password: "Admin1!" },
    user:  { email: "user@test.com",  password: "User1!"  }
};

// Type Alias для типизации данных (способ 3)
type TestUser = { email: string; password: string; role: string };
const loginAs = async (page: Page, user: TestUser): Promise<void> => {
    await page.fill('#email', user.email);
    await page.fill('#password', user.password);
    await page.click('#submit');
};

// Page Object Model (способ 4)
class LoginPage {
    constructor(private page: Page) {}

    async login(email: string, pass: string): Promise<void> {
        await this.page.goto('/login');
        await this.page.fill('#email', email);
        await this.page.fill('#password', pass);
        await this.page.click('#submit');
    }
}
// Использование: const lp = new LoginPage(page); await lp.login(...);
```
