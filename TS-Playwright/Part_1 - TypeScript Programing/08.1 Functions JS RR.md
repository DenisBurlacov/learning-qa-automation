## Зачем нужны функции — повторение с другой стороны

Представь такую задачу: вывести результат нескольких математических операций с двумя числами.

```javascript
let a = 5;
let b = 10;

// БЕЗ функции — повторяемся бесконечно
console.log("Result of Calculation - " + (a + b));
console.log("Result of Calculation - " + (a - b));
console.log("Result of Calculation - " + (a * b));
// Фраза "Result of Calculation" написана 3 раза. Если нужно поменять — правим везде.
```

Три главные причины использовать функции:

**Первая — не повторяться (DRY: Don't Repeat Yourself).** Написал один раз, используешь сколько угодно раз.

**Вторая — организация кода по функционалу.** Вместо каши из 300 строк — набор маленьких функций, каждая делает одно дело.

**Третья — читаемость.** `login(user)` читается как предложение. Ты сразу понимаешь что делает код, не разбирая каждую строку.

## Анатомия функции — разбор по частям

```javascript
//    ↓ ключевое слово
function PrintResults(a, b) {
//       ↑ название   ↑ параметры (в круглых скобках)
    console.log("Result of Calculation - " + (a + b));
//  ↑ тело функции (между фигурными скобками)
}
//         ↑ закрываем тело

// Вызов функции — пишем имя со скобками
PrintResults(5, 10);
```

Разбор каждого элемента:

- **`function`** — зарезервированное слово, без него JavaScript не поймёт что это функция
- **Название** — придумываешь сам, должно отвечать на вопрос "что делает?". Может содержать буквы и цифры, не может начинаться с цифры. По соглашению: `function Declaration` — с большой буквы, стрелочная в переменной — с маленькой
- **`()`** — обязательные круглые скобки, означают что это функция. Здесь же принимаются аргументы
- **`{}`** — фигурные скобки обозначают тело функции. Между ними — любой код
- **Вызов** — функция не выполняется сама по себе, нужно написать `ИмяФункции()` где-то в коде

## Void функции и return — когда что

Функция может либо **возвращать значение** (`return`), либо просто **делать что-то** (void):

```javascript
// VOID функция — ничего не возвращает, просто делает дело
function PrintMessage() {
    console.log("Hello World"); // выводит и всё
}
PrintMessage(); // просто вызываем

// RETURN функция — возвращает результат
function Greeting() {
    return "Hello from function!";
}

// Просто вызвать недостаточно — возвращённое значение уйдёт в никуда
Greeting(); // ← строка существует, но мы её не используем — ничего не видно!

// Нужно захватить результат или сразу вывести
console.log(Greeting()); // ← вот теперь видно: "Hello from function!"
const message = Greeting(); // или сохраняем в переменную
console.log(message);
```

Когда что использовать:

- Функция только **печатает** что-то — используй `void` (console.log внутри, return не нужен)
- Функция **вычисляет** что-то — используй `return` (результат нужно куда-то передать)
- В ~90% случаев функции что-то возвращают через `return`

**Важно:** `return` останавливает функцию. Всё, что написано после `return` — не выполнится:

```javascript
function test() {
    console.log("До return");
    return 42;
    console.log("После return"); // эта строка НИКОГДА не выполнится (VSCode подчеркнёт серым)
}
```

## Область видимости Scope — кто что видит

Это одна из самых важных концепций — **где создана переменная, там она и живёт**.

```javascript
function Calculate() {
    let a = 5;  // переменная создана ВНУТРИ функции
    let b = 10;
    return a + b;
}

// Пытаемся обратиться к a снаружи:
console.log(a); // ОШИБКА: a is not defined
console.log(b); // ОШИБКА: b is not defined
```

Переменные внутри функции — **локальные**. Снаружи не видны.

Переменные снаружи — **глобальные**. Видны везде, в том числе внутри функций:

```javascript
let c = 100; // глобальная переменная

function ShowSum(a, b) {
    // a и b — локальные (параметры функции)
    // c — видна, потому что создана снаружи
    return a + b + c; // можно использовать c
}

console.log(c);             // 100 (снаружи тоже видна)
console.log(ShowSum(1, 3)); // 104 (1 + 3 + 100)
```

Визуально: представь луковицу. Внешний слой — глобальная область. Внутренний слой (функция) видит всё снаружи. Но снаружи не видно что внутри.

## Аргументы и параметры — точная разница

Это разные слова для похожих вещей, и их часто путают:

```javascript
// ПАРАМЕТР — переменная в ОБЪЯВЛЕНИИ функции
//                ↓↓↓↓↓↓↓↓↓↓↓↓↓↓↓↓↓↓
function AddNumbers(numberOne, numberTwo) {
    return numberOne + numberTwo;
}

// АРГУМЕНТ — конкретное значение при ВЫЗОВЕ
//            ↓↓↓↓↓↓↓↓↓↓↓
AddNumbers(5,          10);
```

Параметры должны иметь осмысленные имена в camelCase — не `a`, `b`, а `numberOne`, `userEmail`, `productPrice`. Хотя в учебных примерах `a` и `b` допустимы для простоты.

Аргументов может быть сколько угодно, разделяются запятой. Тип данных — любой.

## Опциональные аргументы со значением по умолчанию

Что если нужно передать только часть аргументов? Используй значение по умолчанию через `=`:

```javascript
function AddNumbers(a, b = 10) { // b = 10 по умолчанию
    return a + b;
}

AddNumbers(5, 20); // 25 — b берёт переданное значение 20
AddNumbers(5);     // 15 — b = 10 (значение по умолчанию)
// AddNumbers();   // NaN — a не передан, нет дефолта

// Если передать undefined явно — сработает дефолт:
AddNumbers(5, undefined); // 15 — то же самое что не передавать b
```

Это эквивалент optional параметров из TypeScript. В TS: `b?: number` или `b: number = 10`.

## Переменные снаружи — что видно внутри функции

```javascript
let c = 100;

function Sum(a, b) {
    // c здесь видна — она создана снаружи
    return a + b + c;
}

console.log(c);       // 100 — снаружи
console.log(Sum(1, 3)); // 104 — внутри тоже видна c
```

И наоборот — созданное внутри не видно снаружи:

```javascript
function Sum(a, b) {
    let result = a + b; // локальная переменная
    return result;
}

console.log(result); // ОШИБКА: result is not defined
```

## Присвоение функции переменной

Результат вызова функции можно сохранить в переменную:

```javascript
function Sum(a, b) {
    return a + b;
}

// Функция вызывается, возвращает результат, результат сохраняется в переменной
const total = Sum(1, 2); // total = 3
console.log(total);      // 3

// Или используем результат прямо там где нужен
const doubled = Sum(1, 2) * 2; // 6
```

Обрати внимание: `total` хранит **результат** функции (число 3), а не саму функцию. Если функция ничего не возвращает (void) — переменная будет `undefined`.

## Вложенные функции — функция внутри функции

Функция может содержать другие функции внутри:

```javascript
function Outer() {
    let x = 5;

    function Inner() {
        return x; // Inner видит x из Outer — это замыкание (closure)
    }

    return "Hello " + Inner(); // Outer вызывает Inner
}

console.log(Outer()); // "Hello 5"
// Inner() вызвать снаружи нельзя — она локальна для Outer
```

Зачем это нужно? Когда внутри функции много логики — её можно разбить на маленькие вспомогательные функции. Снаружи видишь только одну точку входа (`Outer`), не зная деталей реализации. Именно так устроены фреймворки Angular, React и другие.

## Рекурсия — функция вызывает себя

Рекурсивная функция — та, которая вызывает саму себя. Это как цикл, но через функцию:

```javascript
// ОПАСНО: бесконечная рекурсия — stackoverflow!
function SquareIt(number) {
    console.log(number);
    let square = number * number;
    SquareIt(square); // вызывает себя снова и снова...
}
SquareIt(2); // 2, 4, 16, 256, 65536... и так до ошибки

// ПРАВИЛЬНО: нужно условие выхода!
function SquareIt(number) {
    if (number > 1000) return; // выходим когда число стало большим
    console.log(number);
    let square = number * number;
    SquareIt(square); // вызываем себя, но теперь есть выход
}
SquareIt(2); // 2, 4, 16, 256 — остановится
```

**Правило рекурсии:** всегда должно быть условие выхода (`if ... return`), иначе функция уйдёт в бесконечность и упадёт с ошибкой `Maximum call stack size exceeded` (или `stackoverflow`).

`return` без значения просто **выходит** из функции, ничего не возвращая. Это нормально и часто используется для ранних выходов.

## Function Declaration vs Function Expression

В JavaScript есть **два принципиально разных** способа создавать функции. Понимание этой разницы — ключ к пониманию как работает JS.

### Function Declaration именованная функция

```javascript
// Function Declaration — традиционный способ
function Print(text) {
    console.log(text);
}

Print("Hello"); // вызов
```

Отличительные черты:

- Обязательное ключевое слово `function`
- Обязательное имя функции
- Точка с запятой в конце **НЕ ставится**

### Hoisting — подъём функций

Самая магическая особенность Function Declaration — **hoisting** (подъём):

```javascript
// Вызываем ДО объявления — и это работает!
Print("Hello"); // ← написано раньше чем объявление функции

function Print(text) {
    console.log(text);
}
```

Почему это работает? JavaScript, прежде чем начать выполнять код, **сканирует весь файл** сверху вниз, ищет все `function` объявления и автоматически переносит их в самое начало файла. Это и есть hoisting — подъём функций.

Процесс выполнения JavaScript файла:

1. **Фаза сканирования** — движок просматривает весь код, ищет `function` объявления, поднимает их наверх, проверяет синтаксис
2. **Фаза выполнения** — только теперь код начинает работать построчно сверху вниз

Именно поэтому если есть синтаксическая ошибка (пропущена скобка) — код не запустится вообще: ошибка обнаруживается на фазе сканирования.

На практике многие разработчики держат все функции внизу файла (или вверху, по алфавиту), а сам исполняемый код — где удобно. Hoisting делает порядок объявления неважным.

### Function Expression — функция в переменной

Второй способ — присвоить функцию переменной:

```javascript
// Function Expression
const sum = function Sum(a, b) {
    return a + b;
};  // ← точка с запятой обязательна! (это присваивание переменной)

// Вызов через имя переменной (с маленькой буквы!)
console.log(sum(1, 2)); // 3

// Не через имя функции!
Sum(1, 2); // работает, но это не стиль Function Expression
```

Ключевые отличия от Function Declaration:

- Функция **присваивается переменной**
- Точка с запятой в конце **обязательна** (это переменная, а не функция-объявление)
- **Нет hoisting!** Вызвать до объявления — ошибка:

```javascript
sum(1, 2); // ОШИБКА: Cannot access 'sum' before initialization

const sum = function Sum(a, b) {
    return a + b;
};
```

### Анонимная функция

В Function Expression имя функции не нужно — переменная уже несёт имя. Поэтому можно убрать имя после `function`:

```javascript
// Function Expression с именем (избыточно)
const sum = function Sum(a, b) {
    return a + b;
};

// Анонимная функция (function expression без имени)
const sum = function(a, b) {
    return a + b;
};

// Вызов в обоих случаях одинаковый
console.log(sum(1, 2)); // 3
```

Анонимная функция — функция без имени. Имя не нужно, потому что мы обращаемся к ней через переменную `sum`.

### Arrow Function — стрелочная функция

Следующий шаг — убрать слово `function` тоже. Вместо него — стрелка (fat arrow, символ равно плюс больше):

```javascript
// Anonymous Function Expression (можно убрать имя)
const sum = function(a, b) {
    return a + b;
};

// Arrow Function — убрали слово function, добавили стрелку
const sum = (a, b) => {
    return a + b;
};
```

Что происходит: `function` убрали — JavaScript не понял бы что это функция. Придумали стрелку (fat arrow) как замену. Стрелка говорит: "то что слева от меня — параметры, то что справа — тело функции".

Arrow function используется в ~80% случаев в современном JS/TS. Почему:

- Короче писать
- Легче читать
- Позволяет упрощения (implicit return)
- Используется везде — не только когда пишешь свои функции, но и в методах массивов, коллбэках и т.д.

### Implicit Return сокращённая запись

Если тело функции — **одна строка**, можно убрать фигурные скобки и слово `return`:

```javascript
// Полная запись
const sum = (a, b) => {
    return a + b;
};

// Сокращённая (implicit return) — только если одна строка!
const sum = (a, b) => a + b;

// Если несколько строк — фигурные скобки и return обязательны
const process = (a, b) => {
    const temp = a * b;   // первая строка
    return temp + 10;      // вторая строка — нельзя убрать
};
```

Ещё одно сокращение: если параметр **один** — круглые скобки вокруг него можно убрать (но не обязательно, лучше оставить для читаемости):

```javascript
// С одним параметром — скобки можно убрать
const double = x => x * 2;
const double = (x) => x * 2; // тоже правильно, более читаемо

// Без параметров — пустые скобки обязательны!
const greet = () => "Hello!";
```

Рекомендация от преподавателя: для новичков лучше **оставлять скобки** даже с одним параметром — читаемость важнее краткости. Убирать скобки стоит только когда ты хорошо понимаешь что делаешь.

### Три вида функций — сравнение

```javascript
// ① Function Declaration — с именем, без точки с запятой
function MyFunction(a, b) {
    return a + b;
}

// ② Function Expression — через переменную, с точкой с запятой, нет hoisting
const myFunction = function(a, b) {
    return a + b;
};

// ③ Arrow Function — современный стандарт, чаще всего используется
const myFunction = (a, b) => {
    return a + b;
};
// ③ с implicit return (если одна строка)
const myFunction = (a, b) => a + b;
```

Все три функции делают **одно и то же**. Разница только в синтаксисе.

Когда что использовать:

- **Arrow Function** — в ~80% случаев, основной выбор для современного кода
- **Function Declaration** — когда хочешь собрать коллекцию функций в отдельном utility-файле; когда нужен hoisting
- **Function Expression** — редко; иногда когда нужна функция-переменная без стрелки по какой-то причине

## Методы — это функции внутри объекта

Метод — это просто функция, которая создана внутри какой-то структуры (объекта, класса, файла). Снаружи — функция, внутри — метод.

Когда ты использовал `console.log()`, `array.push()`, `string.includes()` — ты вызывал методы. Это встроенные функции JavaScript. Тебе не нужно знать как они написаны внутри — достаточно знать что принимают и что возвращают.

```javascript
Math.sin(x);          // метод объекта Math
"hello".toUpperCase(); // метод строки
[1,2,3].length;        // свойство массива
[1,2,3].push(4);       // метод массива
```

## Callback Function — функция как аргумент

Callback — это функция, которую ты **передаёшь как аргумент** другой функции. Та другая функция потом **вызовет** её сама.

```javascript
// Обычный аргумент — число, строка
array.includes(5);

// Callback — функция как аргумент
array.map(function(item) {
    return item * 2;
});
```

Почему называется callback? Потому что ты как бы говоришь: "когда будешь готов — вызови вот эту функцию". Метод массива сам знает когда её вызвать (для каждого элемента).

В Playwright ты используешь callbacks постоянно:

```typescript
test('login', async ({ page }) => { // стрелочная функция — это callback!
    await page.goto('/login');
});

test.beforeEach(async ({ page }) => { // тоже callback
    await page.goto('/');
});
```

## Методы массивов — главный арсенал

Забудь про цикл `for` для работы с массивами. В современном JS/TS есть готовые методы которые делают всё сами и читаются намного лучше.

### map — трансформация каждого элемента

**Что делает:** проходит по каждому элементу массива, применяет функцию, возвращает **новый массив той же длины** с изменёнными элементами.

**Когда использовать:** нужно что-то сделать с каждым элементом и получить новый массив.

```javascript
const numbers = [1, 2, 5, 6];

// Длинный способ (как это работало раньше через цикл)
const doubled = [];
for (let i = 0; i < numbers.length; i++) {
    doubled.push(numbers[i] * 2);
}

// Современный способ через map (делает то же самое!)
const doubled = numbers.map((item) => item * 2);
// [2, 4, 10, 12]
```

Как работает `map` изнутри:

1. Берёт первый элемент (`1`) — передаёт в callback как `item`
2. Callback умножает `1 * 2 = 2` и возвращает
3. `2` становится первым элементом нового массива
4. Повторяет для каждого элемента
5. Возвращает готовый новый массив

**Оригинальный массив не изменяется!** Создаётся новый.

Практический пример — перевод температур:

```javascript
const celsius = [0, 20, 37, 100];
const fahrenheit = celsius.map((c) => (c * 9 / 5) + 32);
// [32, 68, 98.6, 212]
```

Три параметра callback в map (второй и третий — опциональные):

```javascript
array.map((item, index, originalArray) => {
    // item — текущий элемент
    // index — его индекс (0, 1, 2...)
    // originalArray — исходный массив (редко нужен)
});

// Практический пример с индексом:
// Умножить всё, кроме третьего элемента (индекс 2)
const result = numbers.map((item, index) => {
    if (index !== 2) return item * 2; // умножаем
    return item;                        // третий — без изменений
});
```

### forEach — перебор без нового массива

**Что делает:** проходит по каждому элементу и выполняет функцию. **Ничего не возвращает** (void).

**Когда использовать:** нужно что-то _сделать_ с каждым элементом (вывести в консоль, записать в базу, добавить в DOM), но не нужен новый массив.

```javascript
const animals = ["cat", "dog", "bird"];

animals.forEach((item) => {
    console.log(item);
});
// cat
// dog
// bird

// Если попробуешь сохранить результат — получишь undefined
const result = animals.forEach((item) => item);
console.log(result); // undefined — forEach ничего не возвращает!
```

Разница `map` vs `forEach`:

- `map` — возвращает новый массив, используй когда нужен результат
- `forEach` — ничего не возвращает, используй когда нужно просто "пройтись" и что-то сделать

### filter — фильтрация по условию

**Что делает:** проходит по элементам, оставляет только те, для которых callback вернул `true`. Возвращает **новый массив, обычно меньшей длины**.

**Когда использовать:** нужно отобрать элементы по какому-то условию.

```javascript
const words = ["hello", "afternoon", "fun", "parenthesis", "hammer"];

// Оставить только слова длиннее 6 символов
const longWords = words.filter((word) => word.length > 6);
// ["afternoon", "parenthesis"]
```

Как работает: если callback возвращает `true` — элемент попадает в новый массив. Если `false` — пропускается.

Ещё пример — найти все чётные числа:

```javascript
const numbers = [1, 2, 3, 4, 5, 6, 7, 8];
const evens = numbers.filter((n) => n % 2 === 0);
// [2, 4, 6, 8]
```

После `filter` легко узнать количество: `.length` на результате.

**Разница `map` vs `filter`:**

- `map` — каждый элемент трансформируется, длина массива **не меняется**
- `filter` — элементы не трансформируются, длина массива **уменьшается** (или остаётся той же)

### includes — проверка наличия элемента

**Что делает:** проверяет есть ли конкретный элемент в массиве. Возвращает `true` или `false`.

**Когда использовать:** нужно просто проверить — есть ли что-то в массиве.

```javascript
const animals = ["cat", "dog", "bird"];

console.log(animals.includes("dog"));  // true
console.log(animals.includes("fish")); // false

// Строки:
const browsers = ["chrome", "firefox", "safari"];
browsers.includes("chrome");  // true
browsers.includes("edge");    // false
```

Второй параметр (опциональный) — индекс откуда начинать поиск:

```javascript
const arr = ["a", "b", "c", "d"];

arr.includes("b", 1); // true — ищем начиная с индекса 1
arr.includes("b", 2); // false — ищем начиная с индекса 2, b уже пропустили
```

По умолчанию поиск начинается с индекса 0.

`includes` возвращает только `true`/`false`. Если нужно _посчитать_ сколько раз встречается элемент — используй `filter` или `forEach` со счётчиком.

### reduce — схлопывание массива в одно значение

**Что делает:** проходит по всем элементам и накапливает результат в одну переменную (аккумулятор). Возвращает **одно значение** (не массив).

**Когда использовать:** нужно получить одно значение из массива — сумму, произведение, строку, объект.

```javascript
const numbers = [1, 2, 3, 4];

// Сложить все числа
const sum = numbers.reduce((accumulator, currentValue) => {
    return accumulator + currentValue;
}, 0); // 0 — начальное значение (initialValue)

console.log(sum); // 10
```

Как работает шаг за шагом:

```
Начало: accumulator = 0 (initialValue), массив [1, 2, 3, 4]

Шаг 1: acc = 0,  current = 1 → return 0+1=1  → acc становится 1
Шаг 2: acc = 1,  current = 2 → return 1+2=3  → acc становится 3
Шаг 3: acc = 3,  current = 3 → return 3+3=6  → acc становится 6
Шаг 4: acc = 6,  current = 4 → return 6+4=10 → acc становится 10

Результат: 10
```

`reduce` принимает **два аргумента** (в отличие от `map`, `filter`, `forEach` которые принимают один):

1. **Callback функция** — с параметрами `(accumulator, currentValue)`
2. **Initial value** — начальное значение аккумулятора (опционально, но рекомендуется указывать)

Что такое initial value:

- Если указан `0` — аккумулятор начинает с 0, первый элемент массива = first currentValue
- Если не указан — аккумулятор = первый элемент массива, первый currentValue = второй элемент

```javascript
// С initial value 0
[1, 2, 3].reduce((acc, cur) => acc + cur, 0);
// acc=0, cur=1 → 1; acc=1, cur=2 → 3; acc=3, cur=3 → 6

// Без initial value (аккумулятор берёт первый элемент)
[1, 2, 3].reduce((acc, cur) => acc + cur);
// acc=1, cur=2 → 3; acc=3, cur=3 → 6 (тот же результат, но начинаем со второго элемента)
```

Начать сложение со 100:

```javascript
const sum = [1, 2, 3, 4].reduce((acc, cur) => acc + cur, 100);
// 110 (100 + 1 + 2 + 3 + 4)
```

`reduce` может вернуть не только число — любой тип:

```javascript
// Склеить строки
const words = ["Hello", "World", "!"];
const sentence = words.reduce((acc, word) => acc + " " + word, "");
// " Hello World !" (с начальным пробелом — это особенность данного примера)

// Собрать объект из массива
const pairs = [["a", 1], ["b", 2]];
const obj = pairs.reduce((acc, [key, val]) => {
    acc[key] = val;
    return acc;
}, {});
// { a: 1, b: 2 }
```

Аккумулятор часто называют `acc` или `prev` (previous value), текущий элемент — `cur`, `curr`, `item` или `currentValue`. Название не важно — важно понять роль каждого.

## Когда какой метод использовать

Быстрая шпаргалка для выбора метода:

|Задача|Метод|Возвращает|
|---|---|---|
|Изменить каждый элемент|`map`|новый массив той же длины|
|Отобрать элементы по условию|`filter`|новый массив меньшей длины|
|Пройтись и что-то сделать|`forEach`|ничего (void)|
|Проверить есть ли элемент|`includes`|true / false|
|Получить одно значение из массива|`reduce`|одно значение любого типа|

**Примеры задач:**

"Умножить каждое число на 3" → `map`

"Оставить только числа больше 10" → `filter`

"Вывести каждый элемент в консоль" → `forEach`

"Есть ли в списке слово 'error'?" → `includes`

"Сложить все числа в массиве" → `reduce`

"Найти самое длинное слово" → `reduce`

"Получить только email из списка пользователей" → `map`

"Получить список активных пользователей" → `filter`

"Количество элементов удовлетворяющих условию" → `filter` (потом `.length`) или `forEach` со счётчиком

## Частые ошибки и ловушки

**Ошибка 1: Вызвать Function Expression до объявления**

```javascript
sum(1, 2); // ОШИБКА — нет hoisting у Function Expression

const sum = (a, b) => a + b;

// Function Declaration — можно
Print("hello"); // OK благодаря hoisting

function Print(text) {
    console.log(text);
}
```

**Ошибка 2: Забыть console.log для функции с return**

```javascript
function greet() {
    return "Hello!";
}

greet(); // ← вызвали, но ничего не видим — значение вернулось в никуда!
console.log(greet()); // ← правильно
```

**Ошибка 3: Поставить точку с запятой после Function Declaration**

```javascript
function Print() {
    console.log("hello");
}; // ← точка с запятой здесь — ошибка стиля (линтер будет ругаться)

// Точка с запятой только у Function Expression (переменной):
const print = () => console.log("hello"); // ← здесь правильно
```

**Ошибка 4: Ожидать что forEach вернёт массив**

```javascript
const result = [1, 2, 3].forEach((n) => n * 2);
console.log(result); // undefined — forEach ничего не возвращает!

// Нужен map:
const result = [1, 2, 3].map((n) => n * 2);
console.log(result); // [2, 4, 6]
```

**Ошибка 5: Передать в map/filter вызов функции вместо ссылки**

```javascript
function multiply(item) {
    return item * 2;
}

// ОШИБКА: multiply() — вызываем сразу, передаём результат (undefined)
[1, 2, 3].map(multiply()); // undefined передаётся как callback

// ПРАВИЛЬНО: multiply — ссылка на функцию (без скобок)
[1, 2, 3].map(multiply); // map сам её вызовет с каждым элементом
```

**Ошибка 6: Бесконечная рекурсия без условия выхода**

```javascript
// ОПАСНО
function count(n) {
    console.log(n);
    count(n + 1); // никогда не остановится!
}

// ПРАВИЛЬНО: условие выхода
function count(n) {
    if (n > 10) return; // останавливаемся
    console.log(n);
    count(n + 1);
}
count(1); // 1, 2, 3... 10
```

## Шпаргалка — всё в одном месте

```javascript
// ─────────────────────────────────────────────────────────
// FUNCTION DECLARATION
function MyFunction(a, b) {    // с большой буквы
    return a + b;
}                              // без точки с запятой
// Особенность: hoisting! Можно вызывать до объявления.

// ─────────────────────────────────────────────────────────
// FUNCTION EXPRESSION (anonymous)
const myFunction = function(a, b) {
    return a + b;
};                             // точка с запятой обязательна!
// Нет hoisting. Нет имени у функции.

// ─────────────────────────────────────────────────────────
// ARROW FUNCTION (современный стандарт — используется в ~80% случаев)
const myFunction = (a, b) => {
    return a + b;
};

// Implicit return (одна строка — можно убрать {} и return)
const myFunction = (a, b) => a + b;

// Один параметр (скобки можно убрать, но лучше оставить)
const double = x => x * 2;
const double = (x) => x * 2; // рекомендуется для читаемости

// Без параметров (пустые скобки обязательны)
const greet = () => "Hello!";

// ─────────────────────────────────────────────────────────
// SCOPE
let global = "global";        // видна везде
function fn() {
    let local = "local";      // видна только внутри fn
    console.log(global);      // OK — глобальная видна
}
// console.log(local);        // ОШИБКА — local не видна снаружи

// ─────────────────────────────────────────────────────────
// МЕТОДЫ МАССИВОВ

const arr = [1, 2, 3, 4, 5];

// map — трансформация, возвращает новый массив той же длины
const doubled = arr.map((item) => item * 2);             // [2, 4, 6, 8, 10]
const withIndex = arr.map((item, index) => `${index}:${item}`); // ["0:1", "1:2"...]

// forEach — перебор, ничего не возвращает (void)
arr.forEach((item) => console.log(item));                // 1 2 3 4 5

// filter — фильтрация, возвращает новый массив
const evens = arr.filter((item) => item % 2 === 0);     // [2, 4]
const big = arr.filter((item) => item > 3);              // [4, 5]
const count = arr.filter((item) => item > 3).length;     // 2 — количество

// includes — проверка наличия, возвращает boolean
arr.includes(3);          // true
arr.includes(10);         // false
arr.includes(3, 3);       // false — ищем начиная с индекса 3

// reduce — схлопывание в одно значение
const sum = arr.reduce((acc, cur) => acc + cur, 0);      // 15
const max = arr.reduce((acc, cur) => cur > acc ? cur : acc, 0); // 5
const str = arr.reduce((acc, cur) => acc + cur, "");     // "12345"

// ─────────────────────────────────────────────────────────
// CALLBACK FUNCTION
function execute(callback) {
    return callback(42);
}
execute((n) => n * 2); // 84 — передаём arrow function как callback

// В Playwright:
test('test name', async ({ page }) => { // arrow function — это callback
    await page.goto('/');
});
```
