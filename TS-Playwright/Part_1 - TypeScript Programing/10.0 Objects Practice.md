## Как работать с этим файлом

Создай папку `practice/objects/` в проекте. Для каждого блока — отдельный файл. Запускай через `tsx имяФайла.ts`.

Правило: пиши сам, не смотри решения заранее. Даже неправильная попытка полезнее копипаста.

### Задание 1.1 — Первый объект

Создай объект `car` который описывает автомобиль. Объект должен содержать минимум 5 свойств: марка, модель, год выпуска, цвет, цена. Типы данных подбери логично.

Выведи в консоль каждое свойство отдельной строкой с пояснением:

```
Brand: Toyota
Model: Camry
Year: 2022
Color: White
Price: 25000
```

---

### Задание 1.2 — Объект с методом

Возьми объект из задания 1.1 и добавь ему метод `getDescription()`. Метод должен возвращать строку с описанием автомобиля.

Ожидаемый вывод при вызове `car.getDescription()`:

```
2022 Toyota Camry (White) — $25000
```

Используй `this` для доступа к свойствам внутри метода.

---

### Задание 1.3 — Вложенный объект

Создай объект `testEnvironment` для хранения настроек тестовой среды:

```
Структура:

- credentials (вложенный объект):
    - email: string
    - password: string
- viewport (вложенный объект):
    - width: number

```

После создания выведи: baseUrl, email из credentials, ширину viewport.

_Подсказка: к вложенному объекту обращаешься через двойную точку: `obj.credentials.email`_

### Задание 2.2 — Динамический ключ

Дан объект `config` с ключами `"dev"`, `"staging"`, `"prod"`, каждый из которых содержит URL.

Напиши код, который принимает название среды (строка) из переменной и возвращает соответствующий URL. Используй bracket notation с переменной.

```typescript
const config = {
    dev: "http://localhost:3000",
    staging: "https://staging.example.com",
    prod: "https://example.com"
};

const environment = "staging"; // меняй это значение для теста
// Выведи URL для текущей среды — используй переменную environment как ключ
```

---

### Задание 2.3 — Проверка существования свойства

Дан объект теста. Нужно безопасно получить значение `errorMessage` — свойство которого может не существовать.

```typescript
const test1 = { name: "Login Test", status: "passed", duration: 120 };
const test2 = { name: "Cart Test", status: "failed", duration: 340, errorMessage: "Element not found" };
```

Используй оператор `?.` (optional chaining) чтобы безопасно обратиться к `errorMessage`:

```
test1 error: undefined
test2 error: Element not found
```

_Подсказка: синтаксис `obj?.property` вернёт `undefined` если obj или property не существует._

```typescript
class ShoppingCart {
    items: { name: string; price: number; qty: number }[];

    constructor() {
        this.items = [];
    }

    addItem(name: string, price: number, qty: number): void {
        this.items.push({ name, price, qty });
    }

    getSubtotal(): number {
        return this.items.reduce((sum, item) => sum + item.price * item.qty, 0);
    }

    getDiscount(): number {
        return this.getSubtotal() > 100 ? this.getSubtotal() * 0.1 : 0;
    }

    getTotal(): number {
        return this.getSubtotal() - this.getDiscount();
    }

    getReceipt(): string {
        const lines = this.items.map(i => `  ${i.name} x${i.qty} — $${i.price * i.qty}`);
        return [
            "=== Receipt ===",
            ...lines,
            `Subtotal: $${this.getSubtotal()}`,
            `Discount: -$${this.getDiscount()}`,
            `Total: $${this.getTotal()}`
        ].join("\n");
    }
}

const cart = new ShoppingCart();
cart.addItem("Book", 30, 2);
cart.addItem("Pen", 5, 3);
cart.addItem("Notebook", 25, 1);
console.log(cart.getReceipt());
```
