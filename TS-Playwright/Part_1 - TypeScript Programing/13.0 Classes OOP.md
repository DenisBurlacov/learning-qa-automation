## Аналогия из реальной жизни — что такое класс

Представь что ты работаешь на автозаводе. У тебя есть **чертёж автомобиля** — это описание того, каким должен быть каждый автомобиль: сколько колёс, какой двигатель, какие возможные цвета, что умеет делать.

По одному чертежу можно сделать тысячи реальных автомобилей. Каждый автомобиль — это отдельный объект со своим цветом и номером, но все они сделаны по одному чертежу.

В программировании:
- **Класс** — это чертёж (описание, шаблон)
- **Объект** (или экземпляр) — это реальный автомобиль, сделанный по чертежу

```typescript
// Класс — это чертёж
class Car {
    // описание того, что есть у каждой машины
}

// Объекты — реальные машины, созданные по чертежу
const myCar = new Car();
const yourCar = new Car();
const taxiCar = new Car();
```

Три разных машины, один чертёж.

## Класс vs Объект — главное различие

Это нужно понять раз и навсегда.

```typescript
class Dog {
    name: string = "Buddy";
    breed: string = "Labrador";

    bark(): void {
        console.log("Woof!");
    }
}

// Dog — это класс. Он существует только в коде.
// Никакой реальной собаки ещё нет.

const dog1 = new Dog(); // Вот теперь появилась реальная собака
const dog2 = new Dog(); // И вторая собака

dog1.name = "Rex";    // dog1 — Rex
dog2.name = "Buddy";  // dog2 — Buddy, не трогаем dog1

console.log(dog1.name); // Rex
console.log(dog2.name); // Buddy
```

`dog1` и `dog2` — это два разных объекта. У каждого свои данные. Они не влияют друг на друга.

## Базовый синтаксис класса

```typescript
class ИмяКласса {
    // свойства (данные)
    // методы (действия)
}
```

Имя класса всегда с **большой буквы** — это соглашение. `Dog`, `Car`, `LoginPage`, `UserService`.

```typescript
class Person {
    name: string = "";
    age: number = 0;

    greet(): void {
        console.log(`Hi, I'm ${this.name} and I'm ${this.age} years old.`);
    }
}

const anna = new Person();
anna.name = "Anna";
anna.age = 28;
anna.greet(); // Hi, I'm Anna and I'm 28 years old.
```

## Constructor — как рождается объект

Каждый раз когда ты пишешь `new ИмяКласса()` — вызывается специальный метод `constructor`. Он выполняется один раз при создании объекта и устанавливает начальные значения.

**Без constructor:** создаёшь объект и потом вручную заполняешь каждое свойство.

```typescript
const anna = new Person();
anna.name = "Anna";    // долго и неудобно
anna.age = 28;         // особенно когда свойств много
anna.city = "Madrid";
anna.job = "Tester";
```

**С constructor:** передаёшь всё сразу при создании.

```typescript
class Person {
    name: string;
    age: number;
    city: string;

    constructor(name: string, age: number, city: string) {
        this.name = name;   // берём значение из параметра и сохраняем в свойство
        this.age = age;
        this.city = city;
    }

    greet(): void {
        console.log(`Hi, I'm ${this.name}, ${this.age} y.o., from ${this.city}`);
    }
}

// Теперь всё в одну строку:
const anna = new Person("Anna", 28, "Madrid");
const bob = new Person("Bob", 35, "Barcelona");
const carlos = new Person("Carlos", 42, "Valencia");

anna.greet();   // Hi, I'm Anna, 28 y.o., from Madrid
bob.greet();    // Hi, I'm Bob, 35 y.o., from Barcelona
carlos.greet(); // Hi, I'm Carlos, 42 y.o., from Valencia
```

### Сокращённый синтаксис constructor (самый частый в Playwright)

TypeScript позволяет объявить свойства прямо в параметрах конструктора — через `public`, `private` или `protected`:

```typescript
// ДЛИННЫЙ способ — объявляем отдельно
class Dog {
    name: string;
    breed: string;

    constructor(name: string, breed: string) {
        this.name = name;
        this.breed = breed;
    }
}

// КОРОТКИЙ способ — то же самое, но компактнее
class Dog {
    constructor(
        public name: string,
        public breed: string
    ) {}
    // TypeScript автоматически создаёт свойства и присваивает значения
}

const rex = new Dog("Rex", "German Shepherd");
console.log(rex.name);  // Rex
console.log(rex.breed); // German Shepherd
```

## Properties — свойства класса

Свойства — это данные, которые хранит каждый объект. У каждого объекта свои значения.

```typescript
class BankAccount {
    // Способ 1 — с начальным значением
    currency: string = "EUR";
    isActive: boolean = true;

    // Способ 2 — без начального значения (нужен constructor)
    owner: string;
    balance: number;

    constructor(owner: string, initialBalance: number) {
        this.owner = owner;
        this.balance = initialBalance;
    }
}

const acc1 = new BankAccount("Anna", 1000);
const acc2 = new BankAccount("Bob", 500);

console.log(acc1.owner);    // Anna
console.log(acc1.balance);  // 1000
console.log(acc2.owner);    // Bob
console.log(acc2.balance);  // 500
console.log(acc1.currency); // EUR  ← одинаково у обоих, начальное значение
```

## Methods — методы класса

Методы — это функции внутри класса. Они описывают то, **что умеет делать** объект.

```typescript
class BankAccount {
    constructor(
        public owner: string,
        private balance: number,
        public currency: string = "EUR"
    ) {}

    // Метод — просто функция внутри класса (без слова function)
    deposit(amount: number): void {
        this.balance += amount;
        console.log(`${this.owner} deposited ${amount} ${this.currency}. Balance: ${this.balance}`);
    }

    withdraw(amount: number): void {
        if (amount > this.balance) {
            console.log("Insufficient funds!");
            return;
        }
        this.balance -= amount;
        console.log(`${this.owner} withdrew ${amount} ${this.currency}. Balance: ${this.balance}`);
    }

    getBalance(): number {
        return this.balance;
    }

    printStatement(): void {
        console.log(`--- Account of ${this.owner} ---`);
        console.log(`Balance: ${this.balance} ${this.currency}`);
    }
}

const account = new BankAccount("Anna", 1000);
account.deposit(500);      // Anna deposited 500 EUR. Balance: 1500
account.withdraw(200);     // Anna withdrew 200 EUR. Balance: 1300
account.withdraw(2000);    // Insufficient funds!
account.printStatement();  // --- Account of Anna --- Balance: 1300 EUR
```

## this — самое важное ключевое слово

`this` — это **ссылка на текущий объект**. Когда ты внутри метода пишешь `this.name` — ты обращаешься к свойству `name` конкретного объекта, на котором вызван метод.

Аналогия: представь документ с анкетой, где написано "моё имя: ___". Слово "моё" — это как `this`. Когда Анна заполняет анкету, "моё" означает Анну. Когда Боб — означает Боба.

```typescript
class Timer {
    seconds: number = 0;

    tick(): void {
        this.seconds++;  // this.seconds — это секунды ЭТОГО конкретного таймера
        console.log(`Timer: ${this.seconds}s`);
    }

    reset(): void {
        this.seconds = 0;  // сбрасываем секунды ЭТОГО таймера
    }
}

const timer1 = new Timer();
const timer2 = new Timer();

timer1.tick(); // Timer: 1s  ← this = timer1
timer1.tick(); // Timer: 2s  ← this = timer1
timer2.tick(); // Timer: 1s  ← this = timer2 (у него свои секунды!)

console.log(timer1.seconds); // 2
console.log(timer2.seconds); // 1
```

## Access Modifiers — кто что видит

Access Modifiers (модификаторы доступа) контролируют, кто может читать и изменять свойства и методы класса.

Аналогия: в больнице есть разные уровни доступа. Пациент может видеть своё имя в базе, но не может видеть секретные медицинские данные других пациентов. Медсестра видит больше. Главврач — всё.

### public — открыто для всех (по умолчанию)

```typescript
class User {
    public name: string;  // public можно не писать — это значение по умолчанию

    constructor(name: string) {
        this.name = name;
    }
}

const user = new User("Anna");
console.log(user.name);  // OK — можно читать снаружи
user.name = "Bob";       // OK — можно менять снаружи
```

### private — только внутри класса

```typescript
class User {
    public name: string;
    private password: string;    // пароль — только для внутреннего использования
    private loginAttempts: number = 0;

    constructor(name: string, password: string) {
        this.name = name;
        this.password = password;
    }

    checkPassword(input: string): boolean {
        this.loginAttempts++;                       // OK — внутри класса
        return this.password === input;             // OK — внутри класса
    }
}

const user = new User("Anna", "secret123");

console.log(user.name);         // OK — public
console.log(user.password);     // ОШИБКА — Property 'password' is private
user.password = "hack";         // ОШИБКА — нельзя менять снаружи

console.log(user.checkPassword("secret123")); // true — через метод OK
```

### protected — только внутри класса и в дочерних классах

```typescript
class Animal {
    public name: string;
    protected sound: string;     // видно в этом классе и в дочерних

    constructor(name: string, sound: string) {
        this.name = name;
        this.sound = sound;
    }
}

class Dog extends Animal {
    bark(): void {
        // OK — protected видно в дочернем классе
        console.log(`${this.name} says: ${this.sound}!`);
    }
}

const dog = new Dog("Rex", "Woof");
dog.bark();          // Rex says: Woof!
console.log(dog.sound); // ОШИБКА — protected не видно снаружи классов
```

### Итог — правило трёх уровней

| Модификатор | Внутри класса | Дочерний класс | Снаружи |
|-------------|:---:|:---:|:---:|
| `public` | ✅ | ✅ | ✅ |
| `protected` | ✅ | ✅ | ❌ |
| `private` | ✅ | ❌ | ❌ |

**Правило большого пальца:** делай всё `private` по умолчанию. Открывай через `public` только то, что действительно нужно снаружи. Это называется **инкапсуляция** — одно из ключевых понятий ООП.

## Readonly свойства в классе

`readonly` — можно задать только один раз (в constructor или при объявлении). После — нельзя изменить.

```typescript
class DatabaseConnection {
    public readonly host: string;
    public readonly port: number;
    private isConnected: boolean = false;

    constructor(host: string, port: number) {
        this.host = host;
        this.port = port;
    }

    connect(): void {
        this.isConnected = true;
        // this.host = "other"; — ОШИБКА, readonly
    }
}

const db = new DatabaseConnection("localhost", 5432);
console.log(db.host);  // localhost
db.host = "other";     // ОШИБКА — Cannot assign to 'host' because it is a read-only property
```

## Getters и Setters

Геттер — специальный метод для **чтения** свойства. Сеттер — для **записи**. Они выглядят как свойства снаружи, но внутри — это функции.

**Зачем?** Чтобы добавить логику при чтении или записи — валидацию, форматирование, логирование.

```typescript
class Temperature {
    private _celsius: number;

    constructor(celsius: number) {
        this._celsius = celsius;
    }

    // GETTER — вызывается как свойство: temp.celsius
    get celsius(): number {
        return this._celsius;
    }

    // SETTER — вызывается как присваивание: temp.celsius = 25
    set celsius(value: number) {
        if (value < -273.15) {
            throw new Error("Temperature below absolute zero!");
        }
        this._celsius = value;
    }

    // Геттер без сеттера — вычисляемое свойство только для чтения
    get fahrenheit(): number {
        return (this._celsius * 9 / 5) + 32;
    }

    get kelvin(): number {
        return this._celsius + 273.15;
    }
}

const temp = new Temperature(25);

console.log(temp.celsius);    // 25   ← вызывает геттер
console.log(temp.fahrenheit); // 77   ← вычисляется автоматически
console.log(temp.kelvin);     // 298.15

temp.celsius = 100;   // вызывает сеттер
console.log(temp.fahrenheit); // 212

temp.celsius = -300;  // ОШИБКА — Temperature below absolute zero!
```

### Геттеры в Playwright — очень частый паттерн

```typescript
class LoginPage {
    constructor(private page: Page) {}

    // Геттеры для локаторов — красиво и переиспользуемо

    }
}
```

## Static — общее для всех

`static` свойства и методы принадлежат **классу**, а не конкретному объекту. Они одни на всех — и доступны без создания объекта.

Аналогия: в компании есть общий счётчик — сколько всего сотрудников нанято. Этот счётчик не у конкретного сотрудника, он у самой компании (класса).

```typescript
class Counter {
    private static count: number = 0;  // одно значение на всех

    constructor(public name: string) {
        Counter.count++;  // при создании нового объекта — увеличиваем общий счётчик
        console.log(`Created: ${this.name}. Total: ${Counter.count}`);
    }

    static getCount(): number {
        return Counter.count;
    }

    static reset(): void {
        Counter.count = 0;
    }
}

const a = new Counter("Alpha");   // Created: Alpha. Total: 1
const b = new Counter("Beta");    // Created: Beta. Total: 2
const c = new Counter("Gamma");   // Created: Gamma. Total: 3

console.log(Counter.getCount()); // 3 — вызываем на классе, не на объекте!
Counter.reset();
console.log(Counter.getCount()); // 0
```

### Static в Playwright — фабричные методы

```typescript
class TestUser {
    private constructor(
        public email: string,
        public password: string,
        public role: string
    ) {}

    // Статические фабричные методы — создают объект с нужными данными
    static createAdmin(): TestUser {
        return new TestUser("admin@test.com", "Admin123!", "admin");
    }

    static createTester(): TestUser {
        return new TestUser("tester@test.com", "Test123!", "tester");
    }

    static createViewer(): TestUser {
        return new TestUser("viewer@test.com", "View123!", "viewer");
    }
}

// Используем в тестах:
const admin = TestUser.createAdmin();
const tester = TestUser.createTester();
```

## Inheritance — наследование через extends

Наследование позволяет создать новый класс на основе существующего. Новый класс получает **всё** от родителя и может добавить своё.

Аналогия: у тебя есть базовый рецепт пиццы (тесто, соус, сыр). Маргарита — это базовый рецепт + томаты. Пепперони — базовый рецепт + колбаса. Четыре сыра — базовый рецепт + четыре вида сыра. Каждая — своя пицца, но основа одна.

```typescript
// Родительский класс (базовый, суперкласс)
class Animal {
    constructor(
        public name: string,
        public age: number
    ) {}

    breathe(): void {
        console.log(`${this.name} is breathing`);
    }

    eat(food: string): void {
        console.log(`${this.name} is eating ${food}`);
    }

    describe(): string {
        return `${this.name}, ${this.age} years old`;
    }
}

// Дочерний класс — получает всё от Animal + добавляет своё
class Dog extends Animal {
    constructor(name: string, age: number, public breed: string) {
        super(name, age);  // обязательно вызываем constructor родителя
    }

    bark(): void {
        console.log(`${this.name} says: Woof!`);
    }

    fetch(): void {
        console.log(`${this.name} fetches the ball!`);
    }
}

class Cat extends Animal {
    constructor(name: string, age: number, public isIndoor: boolean) {
        super(name, age);
    }

    meow(): void {
        console.log(`${this.name} says: Meow!`);
    }

    purr(): void {
        console.log(`${this.name} is purring...`);
    }
}

const rex = new Dog("Rex", 3, "Labrador");
const whiskers = new Cat("Whiskers", 5, true);

rex.breathe();   // Rex is breathing       ← унаследованный метод
rex.eat("bone"); // Rex is eating bone     ← унаследованный метод
rex.bark();      // Rex says: Woof!        ← собственный метод Dog

whiskers.eat("fish"); // Whiskers is eating fish  ← унаследованный
whiskers.meow();      // Whiskers says: Meow!     ← собственный метод Cat
```

## super — обращение к родителю

`super` — это ссылка на **родительский класс**. Используется в двух ситуациях:

### 1. `super()` в конструкторе — вызов конструктора родителя

Это **обязательно** в дочернем классе если у родителя есть constructor с параметрами.

```typescript
class Vehicle {
    constructor(
        public brand: string,
        public speed: number
    ) {
        console.log(`Vehicle created: ${brand}`);
    }
}

class Car extends Vehicle {
    constructor(brand: string, speed: number, public doors: number) {
        super(brand, speed);  // ОБЯЗАТЕЛЬНО — передаём данные родителю
        // только после super() можно использовать this
        console.log(`Car created with ${doors} doors`);
    }
}

const car = new Car("Toyota", 180, 4);
// Vehicle created: Toyota
// Car created with 4 doors
```

### 2. `super.метод()` — вызов метода родителя

```typescript
class Animal {
    describe(): string {
        return `Animal: ${this.name}`;
    }
}

class Dog extends Animal {
    describe(): string {
        const parentDescription = super.describe(); // вызываем метод родителя
        return `${parentDescription}, Breed: ${this.breed}`; // добавляем своё
    }
}
```

## Method Overriding — переопределение метода

Дочерний класс может **переопределить** метод родителя — написать свою версию с тем же именем.

```typescript
class Shape {
    color: string;

    constructor(color: string) {
        this.color = color;
    }

    area(): number {
        return 0;  // базовая реализация — ничего не считает
    }

    describe(): string {
        return `${this.color} shape, area: ${this.area()}`;
    }
}

class Circle extends Shape {
    constructor(color: string, private radius: number) {
        super(color);
    }

    // Переопределяем area() — своя формула для круга
    area(): number {
        return Math.PI * this.radius * this.radius;
    }
}

class Rectangle extends Shape {
    constructor(color: string, private width: number, private height: number) {
        super(color);
    }

    // Переопределяем area() — своя формула для прямоугольника
    area(): number {
        return this.width * this.height;
    }
}

const shapes: Shape[] = [
    new Circle("red", 5),
    new Rectangle("blue", 4, 6),
    new Circle("green", 3),
];

// describe() у каждого вызовет ПРАВИЛЬНЫЙ area() — свой для каждой фигуры
shapes.forEach(shape => console.log(shape.describe()));
// red shape, area: 78.54
// blue shape, area: 24
// green shape, area: 28.27
```

Это называется **полиморфизм** — разные объекты реагируют на одно и то же сообщение по-разному.

## Abstract класс — шаблон без реализации

`abstract` класс — это класс который **нельзя создать напрямую** (`new AbstractClass()` — ошибка). Он только шаблон для дочерних классов. Абстрактные методы не имеют реализации — дочерние классы обязаны их реализовать.

```typescript
abstract class PageBase {
    constructor(protected page: Page) {}

    // Обычный метод — есть реализация, наследуют все
    async goto(url: string): Promise<void> {
        await this.page.goto(url);
    }

    async getTitle(): Promise<string> {
        return this.page.title();
    }

    // Абстрактный метод — нет реализации, каждый дочерний пишет свой
    abstract waitForLoad(): Promise<void>;
    abstract getUrl(): string;
}

class LoginPage extends PageBase {
    // ОБЯЗАНЫ реализовать все abstract методы
    async waitForLoad(): Promise<void> {
        await this.page.waitForSelector("#email");
    }

    getUrl(): string {
        return "/login";
    }

    async login(email: string, password: string): Promise<void> {
        await this.page.fill("#email", email);
        await this.page.fill("#password", password);
        await this.page.click("[type='submit']");
    }
}

class DashboardPage extends PageBase {
    async waitForLoad(): Promise<void> {
        await this.page.waitForSelector(".dashboard-content");
    }

    getUrl(): string {
        return "/dashboard";
    }
}

// const base = new PageBase(page);  // ОШИБКА — Cannot create an instance of an abstract class
const login = new LoginPage(page);   // OK
```

## Классы в Playwright — Page Object Model

Page Object Model (POM) — это паттерн где каждая страница браузера представлена отдельным классом. Это **главная причина** почему классы нужны в Playwright.

### Без POM — хаос

```typescript
test("add to cart", async ({ page }) => {
    await page.goto("/login");
    await page.fill("#email", "user@test.com");
    await page.fill("#password", "pass123");
    await page.click("[type='submit']");
    await page.waitForURL("/dashboard");
    await page.click(".products-link");
    await page.waitForSelector(".product-list");
    await page.locator(".product-item").first().click();
    await page.click("#add-to-cart");
    await expect(page.locator(".cart-count")).toHaveText("1");
});
```

Если изменится селектор `#email` — нужно менять его в каждом тесте где есть логин.

### С POM — порядок

```typescript
// pages/LoginPage.ts
class LoginPage {
    constructor(private page: Page) {}

    get emailInput() { return this.page.locator("#email"); }
    get passwordInput() { return this.page.locator("#password"); }
    get submitBtn() { return this.page.locator("[type='submit']"); }

    async goto(): Promise<void> {
        await this.page.goto("/login");
    }

    async login(email: string, password: string): Promise<void> {
        await this.emailInput.fill(email);
        await this.passwordInput.fill(password);
        await this.submitBtn.click();
        await this.page.waitForURL("/dashboard");
    }
}

// pages/ProductPage.ts
class ProductPage {
    constructor(private page: Page) {}

    get productList() { return this.page.locator(".product-item"); }
    get cartCount() { return this.page.locator(".cart-count"); }

    async addFirstToCart(): Promise<void> {
        await this.productList.first().click();
        await this.page.click("#add-to-cart");
    }
}

// tests/cart.spec.ts — тест стал читабельным
test("add to cart", async ({ page }) => {
    const loginPage = new LoginPage(page);
    const productPage = new ProductPage(page);

    await loginPage.goto();
    await loginPage.login("user@test.com", "pass123");

    await productPage.addFirstToCart();

    await expect(productPage.cartCount).toHaveText("1");
});
```

Если изменился селектор `#email` — меняем только в `LoginPage.ts`. Все тесты автоматически получают обновление.

## Частые ошибки и как их избежать

### Ошибка 1 — забыть super() в дочернем классе

```typescript
class Animal {
    constructor(public name: string) {}
}

class Dog extends Animal {
    constructor(name: string, public breed: string) {
        // ОШИБКА — Constructors for derived classes must contain a 'super' call
        this.breed = breed;
    }
}

// ВЕРНО:
class Dog extends Animal {
    constructor(name: string, public breed: string) {
        super(name); // сначала super, потом всё остальное
    }
}
```

### Ошибка 2 — использовать this до super()

```typescript
class Dog extends Animal {
    constructor(name: string, public breed: string) {
        this.breed = breed;  // ОШИБКА — 'super' must be called before accessing 'this'
        super(name);
    }
}
```

### Ошибка 3 — обратиться к private снаружи

```typescript
class User {
    private password: string = "secret";
}

const user = new User();
console.log(user.password); // ОШИБКА — private доступен только внутри класса
```

### Ошибка 4 — забыть что static вызывается на классе

```typescript
class Counter {
    static count: number = 0;
}

const c = new Counter();
console.log(c.count);       // ОШИБКА — Property 'count' does not exist on instance
console.log(Counter.count); // ВЕРНО — вызываем на классе
```

## Шпаргалка — всё в одном месте

```typescript
// Базовый класс
class MyClass {
    // Свойства
    public name: string;          // доступно везде
    private secret: string;       // только внутри класса
    protected shared: string;     // внутри + в дочерних
    readonly id: number;          // только чтение
    static count: number = 0;     // принадлежит классу, не объекту

    // Constructor
    constructor(name: string) {
        this.name = name;
        this.secret = "hidden";
        this.shared = "shared";
        this.id = Date.now();
        MyClass.count++;
    }

    // Короткий синтаксис constructor
    // constructor(public name: string, private secret: string) {}

    // Методы
    publicMethod(): void {}
    private privateMethod(): void {}
    protected protectedMethod(): void {}
    static staticMethod(): void {}

    // Getter / Setter
    get displayName(): string { return this.name.toUpperCase(); }
    set displayName(value: string) { this.name = value; }
}

// Наследование
class ChildClass extends MyClass {
    constructor(name: string, public extra: string) {
        super(name);  // обязательно первым
    }

    // Переопределение метода
    publicMethod(): void {
        super.publicMethod(); // вызов родительского метода
        // добавляем своё
    }
}

// Abstract
abstract class AbstractBase {
    abstract mustImplement(): void;  // без реализации
    normalMethod(): void {}          // с реализацией
}

// Создание объектов
const obj = new MyClass("test");
const child = new ChildClass("test", "extra");
```
