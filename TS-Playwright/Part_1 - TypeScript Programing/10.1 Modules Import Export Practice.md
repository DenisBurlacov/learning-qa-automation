## Как работать с этим файлом

Создай папку `practice/modules/` в своём проекте. Каждый блок — это несколько связанных файлов. Запускай через `tsx entrypoint.ts`.

Алгоритм:

1. Прочитай условие и создай нужные файлы
2. Попробуй написать решение сам
3. Запусти — убедись что ошибок нет
4. Сравни со своим решением в разделе РЕШЕНИЯ

## БЛОК 1 — Named Export и Import базовый уровень

Папка: `block1/`

---

### Задание 1.1 — Первый экспорт

Создай файл `constants.ts` который экспортирует:

- `BASE_URL` — строка `"https://myapp.com"`
- `TIMEOUT` — число `30000`
- `MAX_RETRIES` — число `3`
- `DEFAULT_USER_ROLE` — строка `"tester"`

Создай файл `main.ts` который импортирует все четыре константы и выводит каждую в консоль.

Ожидаемый вывод:

```
Base URL: https://myapp.com
Timeout: 30000ms
Max retries: 3
Default role: tester
```

---

### Задание 1.2 — Экспорт функций

Создай файл `math.ts` который экспортирует:

- `add(a, b)` — сложение
- `subtract(a, b)` — вычитание
- `multiply(a, b)` — умножение
- `divide(a, b)` — деление (бросает ошибку если b равно нулю)

Создай `calculator.ts` который импортирует все функции и демонстрирует их работу.

---

### Задание 1.3 — Переименование при импорте

В задании 1.2 ты создал `add`, `subtract`, `multiply`, `divide`. Теперь создай файл `aliased.ts` который импортирует те же функции, но переименовывает их при импорте:

- `add` → `sum`
- `subtract` → `minus`
- `multiply` → `times`
- `divide` → `quotient`

Используй переименованные варианты в коде.

---

### Задание 1.4 — Namespace import

Создай файл `namespace.ts` который импортирует весь модуль `math.ts` как `MathUtils` и вызывает функции через `MathUtils.add()`, `MathUtils.divide()`.

Когда это удобнее чем импорт по имени? Напиши ответ в комментарии.

### Задание 5.1 — Создай структуру проекта

Создай следующую структуру файлов (без Playwright тестов — просто файлы с нужными экспортами):

```
playwright-project/
├── types/
│   └── index.ts          ← User, Product, CartItem, OrderStatus
├── config/
│   └── index.ts          ← baseUrl, credentials, timeouts
├── helpers/
│   ├── auth.ts           ← login(), logout(), getAuthToken()
│   ├── data.ts           ← generateUser(), generateProduct()
│   └── index.ts          ← реэкспорт из auth и data
├── pages/
│   ├── LoginPage.ts      ← class LoginPage (default export)
│   ├── CartPage.ts       ← class CartPage (default export)
│   └── index.ts          ← реэкспорт обоих классов
└── main.ts               ← точка входа — использует всё выше
```

В `main.ts` импортируй только из `types/index.ts`, `config/index.ts`, `helpers/index.ts`, `pages/index.ts` — не из отдельных файлов внутри папок.

### Решения БЛОК 3

**3.1**

```typescript
// string-utils.ts
export const capitalize = (str: string): string =>
    str.charAt(0).toUpperCase() + str.slice(1);

export const truncate = (str: string, max: number): string =>
    str.length > max ? str.slice(0, max) + "..." : str;

export const slugify = (str: string): string =>
    str.toLowerCase().replace(/\s+/g, "-").replace(/[^a-z0-9-]/g, "");

// index.ts
export * from "./string-utils";
export * from "./number-utils";
export * from "./date-utils";

// main.ts
import { capitalize, clamp, formatDate } from "./utils";
```

### Решения БЛОК 4

**4.3**

```typescript
import type { User } from "./types";       // только тип — не попадает в JS
import { createUser } from "./user-factory"; // функция — нужна в runtime

// import type нужен для типов потому что:
// 1. TypeScript удаляет типы при компиляции в JS
// 2. import type явно сигнализирует: это только для проверки типов
// 3. Предотвращает circular import проблемы
// 4. Bundler'ы могут лучше оптимизировать tree-shaking
```
