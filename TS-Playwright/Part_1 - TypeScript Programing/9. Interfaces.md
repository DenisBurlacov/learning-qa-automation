## Что такое Interface и зачем он нужен

**Interface** — это контракт. Он описывает какие свойства и методы должен иметь объект, но не описывает как они реализованы.

Аналогия: в компании есть должностная инструкция для тестировщика — список обязанностей. Любой сотрудник с этой инструкцией обязан уметь: писать тест-кейсы, выполнять регрессионное тестирование, оформлять баг-репорты. Конкретный человек реализует эти обязанности по-своему, но интерфейс (набор требований) у всех тестировщиков одинаковый.

### Проблема без Interface

```typescript
// Без интерфейса — ошибку найдёшь только во время выполнения
function displayUser(user: any) {
    console.log(user.name.toUpperCase());  // а вдруг name — undefined?
    console.log(user.email);               // а вдруг поле называется e_mail?
}

displayUser({ name: "Anna", email: "anna@test.com" });  // OK
displayUser({ userName: "Bob", email: "bob@test.com" }); // Runtime Error!
```

### Решение с Interface

```typescript
interface User {
    name: string;
    email: string;
}

function displayUser(user: User) {
    console.log(user.name.toUpperCase());
    console.log(user.email);
}

displayUser({ name: "Anna", email: "anna@test.com" });
displayUser({ userName: "Bob", email: "bob@test.com" }); // ОШИБКА НА ЭТАПЕ КОМПИЛЯЦИИ
// Argument of type '{ userName: string; email: string; }' is not
// assignable to parameter of type 'User'. Object literal may only
// specify known properties, and 'userName' does not exist in type 'User'.
```

TypeScript находит ошибку до запуска — это и есть главная ценность интерфейсов.

## Базовый синтаксис Interface

```typescript
interface InterfaceName {
    property: Type;
}
```

### Примеры

```typescript
interface Product {
    id: number;
    name: string;
    price: number;
    inStock: boolean;
}

interface ApiResponse {
    status: number;
    message: string;
    data: unknown;
}

interface Coordinates {
    x: number;
    y: number;
}
```

### Использование интерфейса

```typescript
// Как тип переменной
const product: Product = {
    id: 1,
    name: "Laptop",
    price: 999.99,
    inStock: true
};

// Как тип параметра функции
function getProductInfo(p: Product): string {
    return `${p.name} — $${p.price}`;
}

// Как тип возвращаемого значения
function createProduct(name: string, price: number): Product {
    return { id: Date.now(), name, price, inStock: true };
}

// Как тип массива
const products: Product[] = [];
```

## Optional свойства

Некоторые свойства не обязательны. Помечаются `?` после имени.

```typescript
interface User {
    id: number;
    email: string;
    firstName: string;
    lastName?: string;    // необязательно
    phone?: string;       // необязательно
    avatarUrl?: string;   // необязательно
}

// Минимально валидный объект:
const user1: User = {
    id: 1,
    email: "anna@test.com",
    firstName: "Anna"
};

// Полный объект:
const user2: User = {
    id: 2,
    email: "bob@test.com",
    firstName: "Bob",
    lastName: "Smith",
    phone: "+34600000000"
};
```

### Работа с optional свойствами

```typescript
function getFullName(user: User): string {
    // Нужно проверять наличие перед использованием
    if (user.lastName !== undefined) {
        return `${user.firstName} ${user.lastName}`;
    }
    return user.firstName;
}

// Или через optional chaining
const upperLastName = user.lastName?.toUpperCase();
// undefined если lastName нет, строка если есть
```

## Readonly свойства

Свойство нельзя изменить после создания объекта. Аналог `const` для свойств.

```typescript
interface Config {
    readonly apiKey: string;
    readonly baseUrl: string;
    timeout: number;         // это можно менять
}

const config: Config = {
    apiKey: "abc123",
    baseUrl: "https://api.myapp.com",
    timeout: 5000
};

config.timeout = 10000;  // OK — это обычное свойство
config.apiKey = "xyz";   // ОШИБКА — Cannot assign to 'apiKey' because it is a read-only property
```

### Когда использовать readonly

- Идентификаторы (id, apiKey, createdAt)
- Конфигурация которая не должна меняться в runtime
- Данные пришедшие от сервера (не мутируем — создаём новый объект)

```typescript
interface TestCase {
    readonly id: string;
    readonly name: string;
    readonly createdAt: Date;
    status: "pending" | "running" | "passed" | "failed";  // меняется
    result?: string;  // появляется после выполнения
}
```

## Методы в Interface

Интерфейс может описывать не только свойства, но и методы.

```typescript
interface Logger {
    log(message: string): void;
    warn(message: string): void;
    error(message: string): void;
    clear(): void;
}

interface Calculator {
    add(a: number, b: number): number;
    subtract(a: number, b: number): number;
    reset(): void;
}
```

### Объект реализующий интерфейс

```typescript
const consoleLogger: Logger = {
    log: (message) => console.log(`[LOG] ${message}`),
    warn: (message) => console.warn(`[WARN] ${message}`),
    error: (message) => console.error(`[ERROR] ${message}`),
    clear: () => console.clear()
};

const basicCalc: Calculator = {
    add: (a, b) => a + b,
    subtract: (a, b) => a - b,
    reset: () => console.log("Calculator reset")
};
```

## Interface для функций

Interface может описывать форму функции — её сигнатуру.

```typescript
// Описываем: функция принимает два числа и возвращает число
interface MathOperation {
    (a: number, b: number): number;
}

// Все эти функции соответствуют интерфейсу MathOperation:
const add: MathOperation = (a, b) => a + b;
const subtract: MathOperation = (a, b) => a - b;
const multiply: MathOperation = (a, b) => a * b;

// Полезно для callback параметров:
interface ClickHandler {
    (event: MouseEvent): void;
}

interface AsyncOperation {
    (url: string, options?: RequestInit): Promise<Response>;
}
```

## Extending — расширение Interface

Интерфейс может наследовать другой (или несколько) — получает все его свойства плюс добавляет свои.

```typescript
interface Base {
    id: number;
    createdAt: Date;
    updatedAt: Date;
}

// User расширяет Base — получает id, createdAt, updatedAt + свои поля
interface User extends Base {
    email: string;
    role: "admin" | "tester" | "viewer";
}

// AdminUser расширяет User — получает всё из User (и Base) + свои поля
interface AdminUser extends User {
    permissions: string[];
    canDeleteUsers: boolean;
}

const admin: AdminUser = {
    id: 1,
    createdAt: new Date(),
    updatedAt: new Date(),
    email: "admin@test.com",
    role: "admin",
    permissions: ["read", "write", "delete"],
    canDeleteUsers: true
};
```

### Множественное расширение

```typescript
interface Timestamps {
    createdAt: Date;
    updatedAt: Date;
}

interface Identifiable {
    id: number;
}

interface Auditable {
    createdBy: string;
    updatedBy: string;
}

// Расширяем несколько интерфейсов сразу
interface Entity extends Identifiable, Timestamps, Auditable {
    name: string;
}
```

## Interface vs Type Alias — когда что использовать

Это один из самых частых вопросов. Они похожи, но есть различия.

```typescript
// Interface
interface UserI {
    name: string;
    email: string;
}

// Type Alias
type UserT = {
    name: string;
    email: string;
};
```

### Чем отличаются

| | Interface | Type Alias |
|--|-----------|-----------|
| Расширение | `extends` | `&` (intersection) |
| Объединение типов | Нет | Да (`string \| number`) |
| Примитивы | Нет | Да (`type ID = string`) |
| Declaration merging | Да | Нет |
| Ошибки компилятора | Понятнее | Иногда сложнее читать |

### Declaration merging (уникальная фича Interface)

```typescript
// Можно объявить один interface в двух местах — они сольются
interface Window {
    myCustomProp: string;
}
interface Window {
    anotherProp: number;
}
// Итоговый Window имеет ОБА свойства
```

### Правило когда что использовать

**Interface** — для объектов и классов: `User`, `Product`, `LoginPage`, `ApiResponse`

**Type Alias** — для всего остального: примитивы, union, intersection, tuple

```typescript
// Interface — объект
interface PageConfig {
    url: string;
    timeout: number;
}

// Type — union
type Status = "active" | "inactive" | "banned";

// Type — примитив с алиасом
type UserId = number;
type Email = string;

// Type — tuple
type Coordinates = [number, number];

// Type — intersection (когда нельзя использовать extends)
type AdminConfig = PageConfig & { adminKey: string };
```

## Implements — класс реализует Interface

Класс может объявить что он реализует интерфейс. TypeScript проверит что все обязательные свойства и методы присутствуют.

```typescript
interface Printable {
    print(): void;
    getContent(): string;
}

interface Saveable {
    save(): Promise<void>;
    load(): Promise<void>;
}

// Класс реализует оба интерфейса
class Document implements Printable, Saveable {
    private content: string;

    constructor(content: string) {
        this.content = content;
    }

    // Обязан реализовать все методы из Printable
    print(): void {
        console.log(this.content);
    }

    getContent(): string {
        return this.content;
    }

    // Обязан реализовать все методы из Saveable
    async save(): Promise<void> {
        console.log("Saving...");
    }

    async load(): Promise<void> {
        console.log("Loading...");
    }
}
```

## Interfaces в Playwright — Page Object Model

Page Object Model (POM) — это паттерн где каждая страница браузера представлена классом. Интерфейсы делают POM более структурированным и предсказуемым.

```typescript
import { Page } from "@playwright/test";

// Интерфейс — описывает что ДОЛЖНА уметь любая страница
interface BasePage {
    goto(): Promise<void>;
    waitForLoad(): Promise<void>;
    getTitle(): Promise<string>;
}

// Интерфейс для страниц с формами
interface FormPage extends BasePage {
    fillForm(data: Record<string, string>): Promise<void>;
    submitForm(): Promise<void>;
    getFormError(): Promise<string | null>;
}

// Конкретная реализация
class LoginPage implements FormPage {
    private readonly url = "/login";

    constructor(private page: Page) {}

    async goto(): Promise<void> {
        await this.page.goto(this.url);
    }

    async waitForLoad(): Promise<void> {
        await this.page.waitForSelector("#email");
    }

    async getTitle(): Promise<string> {
        return this.page.title();
    }

    async fillForm(data: Record<string, string>): Promise<void> {
        for (const [selector, value] of Object.entries(data)) {
            await this.page.fill(selector, value);
        }
    }

    async submitForm(): Promise<void> {
        await this.page.click("[type='submit']");
    }

    async getFormError(): Promise<string | null> {
        const error = this.page.locator(".error-message");
        const count = await error.count();
        if (count === 0) return null;
        return error.textContent();
    }
}
```

## Частые ошибки и как их избежать

### Ошибка 1 — лишние свойства при литеральном присваивании

```typescript
interface User {
    name: string;
    email: string;
}

// ОШИБКА при прямом присваивании объектного литерала
const user: User = {
    name: "Anna",
    email: "anna@test.com",
    age: 28  // Object literal may only specify known properties
};

// OK — через промежуточную переменную (excess property check не срабатывает)
const data = { name: "Anna", email: "anna@test.com", age: 28 };
const user2: User = data;  // работает — data совместима с User
```

### Ошибка 2 — забыть реализовать метод в классе

```typescript
interface Animal {
    sound(): string;
    move(): void;
}

class Dog implements Animal {
    sound(): string { return "Woof"; }
    // ОШИБКА — Class 'Dog' incorrectly implements interface 'Animal'.
    // Property 'move' is missing in type 'Dog' but required in type 'Animal'.
}
```

### Ошибка 3 — пытаться добавить в interface union type

```typescript
// НЕВЕРНО — interface не может быть union
interface StringOrNumber = string | number;  // SyntaxError

// ВЕРНО — для union используй type alias
type StringOrNumber = string | number;
```

## Шпаргалка — всё в одном месте

```typescript
// Базовый interface
interface Product {
    id: number;
    name: string;
    price?: number;       // optional
    readonly sku: string; // readonly
}

// Interface с методами
interface Service {
    fetch(url: string): Promise<unknown>;
    process(data: unknown): string;
}

// Interface для функции
interface Validator {
    (value: string): boolean;
}

// Extends — наследование
interface DetailedProduct extends Product {
    description: string;
    images: string[];
}

// Extends нескольких
interface SuperProduct extends Product, Service {
    rating: number;
}

// Implements — класс реализует interface
class ProductService implements Service {
    async fetch(url: string): Promise<unknown> { /* ... */ return {}; }
    process(data: unknown): string { return String(data); }
}

// В Playwright
interface PageObject {
    goto(): Promise<void>;
    waitForLoad(): Promise<void>;
}

class MyPage implements PageObject {
    constructor(private page: Page) {}
    async goto(): Promise<void> { await this.page.goto("/"); }
    async waitForLoad(): Promise<void> { await this.page.waitForLoadState(); }
}
```
