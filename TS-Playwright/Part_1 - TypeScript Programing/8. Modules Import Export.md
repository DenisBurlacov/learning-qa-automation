## Зачем нужны модули

Представь реальный Playwright проект. Без модулей весь код — в одном файле:

```typescript
// один огромный test.ts — 2000 строк...
// сверху — константы
const BASE_URL = "https://myapp.com";
const TIMEOUT = 30000;

// затем типы
type UserRole = "admin" | "tester" | "viewer";
interface LoginData { email: string; password: string; }

// затем helper функции
async function login(page, email, password) { /* 20 строк */ }
async function addToCart(page, productId) { /* 15 строк */ }
async function checkout(page, address) { /* 30 строк */ }

// потом Page Objects — ещё 300 строк
class LoginPage { /* ... */ }
class ProductPage { /* ... */ }

// и наконец сами тесты — ещё 500 строк
test("login works", async ({ page }) => { /* ... */ });
test("add to cart", async ({ page }) => { /* ... */ });
```

Когда проект растёт — поддерживать это невозможно. Модули позволяют разбить код на логические файлы и подключать только то что нужно.

### Что даёт разбивка на модули

- **Читаемость** — каждый файл отвечает за одну вещь
- **Переиспользование** — функцию из `helpers.ts` используешь в 10 тестах
- **Командная работа** — разные люди работают в разных файлах без конфликтов
- **Тестируемость** — отдельные модули легче тестировать изолированно

## Что такое модуль

В TypeScript/JavaScript **каждый файл — это модуль** если в нём есть хотя бы один `import` или `export`. Код в модуле **изолирован** — переменные из одного файла не видны в другом, пока явно не экспортированы.

```typescript
// file-a.ts
const secret = "hidden";      // недоступно снаружи
export const visible = "hello"; // доступно после import

// file-b.ts
import { visible } from "./file-a";
console.log(visible);  // "hello"
console.log(secret);   // ОШИБКА — secret не экспортирован
```

## Named Export — именованный экспорт

Можно экспортировать несколько значений из одного файла. Каждое имеет своё имя.

### Способ 1 — export при объявлении

```typescript
// utils.ts
export const BASE_URL = "https://myapp.com";
export const TIMEOUT = 30000;

export function formatDate(date: Date): string {
    return date.toISOString().split("T")[0];
}

export const delay = async (ms: number): Promise<void> => {
    return new Promise(resolve => setTimeout(resolve, ms));
};

export type UserRole = "admin" | "tester" | "viewer";
```

### Способ 2 — export списком в конце файла

```typescript
// helpers.ts
const login = async (page: Page, email: string, password: string): Promise<void> => {
    // ...
};

const logout = async (page: Page): Promise<void> => {
    // ...
};

const formatPrice = (price: number): string => `$${price.toFixed(2)}`;

// Экспортируем всё в одном месте — удобно для чтения файла
export { login, logout, formatPrice };
```

### Импорт Named Exports

```typescript
// test.ts
import { BASE_URL, TIMEOUT, formatDate } from "./utils";
import { login, logout } from "./helpers";

console.log(BASE_URL);  // "https://myapp.com"
```

## Default Export — экспорт по умолчанию

Один файл — один default export. Имя при импорте можно задать любое.

```typescript
// LoginPage.ts
class LoginPage {
    constructor(private page: Page) {}

    async goto(): Promise<void> {
        await this.page.goto("/login");
    }

    async login(email: string, password: string): Promise<void> {
        await this.page.fill("#email", email);
        await this.page.fill("#password", password);
        await this.page.click("[type='submit']");
    }
}

export default LoginPage;
```

```typescript
// test.ts — имя можно задать любое при импорте
import LoginPage from "./LoginPage";       // стандартное имя
import LP from "./LoginPage";              // тоже работает
import MyLoginPage from "./LoginPage";     // тоже работает
```

### Default export для функций и объектов

```typescript
// config.ts — default export для конфиг объекта
const config = {
    baseUrl: "https://myapp.com",
    timeout: 30000,
    credentials: {
        admin: { email: "admin@test.com", password: "secret" },
        tester: { email: "tester@test.com", password: "secret" },
    }
};

export default config;
```

```typescript
// test.ts
import config from "./config";
console.log(config.baseUrl);
```

## Named vs Default — когда что использовать

| | Named Export | Default Export |
|--|-------------|----------------|
| Сколько можно | Много | Один |
| Имя при импорте | Фиксированное (можно менять через `as`) | Любое |
| Когда использовать | Утилиты, константы, типы | Классы, главный объект файла |
| Автодополнение IDE | Лучше | Хуже |

**Правило большого пальца:** классы и Page Objects — default, всё остальное — named.

```typescript
// ХОРОШО — класс как default
export default class CartPage { /* ... */ }

// ХОРОШО — утилиты как named
export const addToCart = async () => { /* ... */ };
export const removeFromCart = async () => { /* ... */ };
export type CartItem = { id: number; qty: number };
```

## Import синтаксис — все варианты

```typescript
// 1. Named import
import { login, logout } from "./helpers";

// 2. Named import с переименованием (as)
import { login as doLogin, logout as doLogout } from "./helpers";

// 3. Default import
import LoginPage from "./LoginPage";

// 4. Default + named вместе
import LoginPage, { SELECTORS, fillForm } from "./LoginPage";

// 5. Всё под одним именем — namespace import
import * as helpers from "./helpers";
helpers.login(page, email, password);
helpers.logout(page);

// 6. Только для side effects (редко в Playwright)
import "./setup";  // файл выполнится, но ничего не импортируется

// 7. Импорт типов (только для типов, не попадает в скомпилированный JS)
import type { UserRole, LoginData } from "./types";
```

## Re-export — переэкспорт

Позволяет объединить экспорты из нескольких файлов в одном — удобно для создания "входной точки" (`index.ts`).

```typescript
// pages/LoginPage.ts
export default class LoginPage { /* ... */ }

// pages/CartPage.ts
export default class CartPage { /* ... */ }

// pages/CheckoutPage.ts
export default class CheckoutPage { /* ... */ }

// pages/index.ts — собираем всё в одном месте
export { default as LoginPage } from "./LoginPage";
export { default as CartPage } from "./CartPage";
export { default as CheckoutPage } from "./CheckoutPage";
```

```typescript
// test.ts — один импорт вместо трёх
import { LoginPage, CartPage, CheckoutPage } from "./pages";
```

## Типы и интерфейсы в модулях

Типы и интерфейсы тоже экспортируются. Хорошая практика — держать все типы в отдельном файле.

```typescript
// types.ts
export type UserRole = "admin" | "tester" | "viewer";

export interface User {
    id: number;
    email: string;
    role: UserRole;
}

export interface LoginCredentials {
    email: string;
    password: string;
    rememberMe?: boolean;
}

export interface TestConfig {
    baseUrl: string;
    timeout: number;
    headless: boolean;
}
```

```typescript
// helpers.ts — импортируем типы
import type { User, LoginCredentials } from "./types";

export const createUser = (email: string, role: UserRole): User => ({
    id: Date.now(),
    email,
    role
});
```

## Структура Playwright проекта с модулями

Вот как выглядит типичная структура реального Playwright проекта:

```
my-playwright-project/
├── tests/
│   ├── login.spec.ts
│   ├── cart.spec.ts
│   └── checkout.spec.ts
├── pages/                    ← Page Object классы
│   ├── LoginPage.ts
│   ├── CartPage.ts
│   ├── CheckoutPage.ts
│   └── index.ts              ← Re-export всех страниц
├── helpers/                  ← Вспомогательные функции
│   ├── auth.ts
│   ├── data.ts
│   └── index.ts
├── fixtures/                 ← Playwright fixtures
│   └── base.ts
├── types/                    ← Все TypeScript типы
│   └── index.ts
└── playwright.config.ts
```

### Пример связи файлов

```typescript
// types/index.ts
export interface User { id: number; email: string; role: "admin" | "tester"; }
export interface Product { id: number; name: string; price: number; }

// helpers/auth.ts
import type { User } from "../types";
export const createTestUser = (role: User["role"]): User => ({
    id: Date.now(), email: `${role}@test.com`, role
});

// pages/LoginPage.ts
import type { User } from "../types";
export default class LoginPage {
    constructor(private page: Page) {}
    async loginAs(user: User): Promise<void> {
        await this.page.fill("#email", user.email);
        await this.page.fill("#password", "secret");
        await this.page.click("[type='submit']");
    }
}

// tests/login.spec.ts
import { test, expect } from "@playwright/test";
import LoginPage from "../pages/LoginPage";
import { createTestUser } from "../helpers/auth";

test("admin can login", async ({ page }) => {
    const loginPage = new LoginPage(page);
    const admin = createTestUser("admin");
    await loginPage.loginAs(admin);
    await expect(page).toHaveURL("/admin");
});
```

## Частые ошибки и как их избежать

### Ошибка 1 — неверный путь

```typescript
// НЕВЕРНО — забыли "./"
import { login } from "helpers";  // ищет в node_modules!

// ВЕРНО — относительный путь
import { login } from "./helpers";
import { login } from "../helpers";
```

### Ошибка 2 — экспорт и default вперемешку в одном имени

```typescript
// НЕВЕРНО
export default function login() {}
export default function logout() {}  // нельзя два default!

// ВЕРНО — только один default
export default function login() {}
export function logout() {}  // остальные — named
```

### Ошибка 3 — circular imports (циклический импорт)

```typescript
// a.ts импортирует из b.ts, а b.ts импортирует из a.ts — плохо!
// Решение: вынести общий код в третий файл c.ts
```

### Ошибка 4 — импорт значения вместо типа

```typescript
// В runtime TypeScript типы удаляются. Лучше явно пометить:
import type { User } from "./types";  // import type — только для типов
```

## Шпаргалка — всё в одном месте

```typescript
// ===== EXPORT =====

// Named export
export const PI = 3.14;
export function greet(name: string): void {}
export class MyClass {}
export type MyType = string | number;
export interface MyInterface { id: number; }

// Named export списком
export { PI, greet, MyClass };

// Default export
export default class MainClass {}
export default function mainFn() {}
export default { key: "value" };

// ===== IMPORT =====

// Named
import { PI, greet } from "./module";

// Named с переименованием
import { greet as sayHello } from "./module";

// Default
import MainClass from "./module";

// Default + named
import MainClass, { PI, greet } from "./module";

// Всё как namespace
import * as mod from "./module";
mod.greet("world");

// Только типы
import type { MyType } from "./module";

// ===== RE-EXPORT =====
export { default as MyClass } from "./MyClass";
export { greet, PI } from "./utils";
export * from "./helpers";
```
