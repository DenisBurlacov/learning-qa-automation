## Зачем вообще нужны функции

Представь, что ты тестируешь интернет-магазин. В каждом тесте нужно сначала залогиниться. Без функций ты пишешь одно и то же снова и снова:

```typescript
// БЕЗ функций — кошмар поддержки
test('test 1', async ({ page }) => {
    await page.goto('/login');
    await page.fill('#email', 'user@test.com');
    await page.fill('#password', '123456');
    await page.click('#submit');
    await page.waitForURL('/dashboard');
    // ... сам тест
});

test('test 2', async ({ page }) => {
    await page.goto('/login');          // снова то же самое
    await page.fill('#email', 'user@test.com');  // снова
    await page.fill('#password', '123456');      // снова
    await page.click('#submit');                 // снова
    await page.waitForURL('/dashboard');         // снова
    // ... сам тест
});
```

Что случится, если поменяется селектор `#email` на `#user-email`? Ты будешь исправлять его в 20 тестах. С функцией — только в одном месте:

```typescript
// С функцией — красота и порядок
const login = async (page, email, password) => {
    await page.goto('/login');
    await page.fill('#email', email);
    await page.fill('#password', password);
    await page.click('#submit');
    await page.waitForURL('/dashboard');
};

test('test 1', async ({ page }) => {
    await login(page, 'user@test.com', '123456');
    // ... сам тест
});

test('test 2', async ({ page }) => {
    await login(page, 'admin@test.com', 'admin123');
    // ... сам тест
});
```

**Три главных причины писать функции:**

Первая — **Reusability (переиспользование)**. Написал один раз — используешь везде. Поменял в одном месте — изменилось везде.

Вторая — **Readability (читаемость)**. `await login(page, email, pass)` читается как человеческий язык. Понятно что делает без объяснений.

Третья — **Maintainability (поддержка)**. Когда приложение меняется — ты обновляешь только функцию, а не каждый тест.

## Параметр vs Аргумент — важное различие

Это два разных слова для похожих вещей, и их часто путают. На интервью спросят обязательно.

```typescript
// ПАРАМЕТР — это переменная в ОБЪЯВЛЕНИИ функции
//            ↓↓↓↓↓↓↓↓↓↓↓↓↓↓↓↓↓↓↓↓
function greet(name: string, age: number): void {
    console.log(`Hello ${name}, you are ${age}`);
}

// АРГУМЕНТ — это конкретное значение при ВЫЗОВЕ функции
//             ↓↓↓↓↓↓↓↓↓↓↓↓↓↓↓↓
greet("Anna",        30);
```

Простая аналогия: параметр — это как пустой стакан (`name: string`). Аргумент — это то, что в него наливают (`"Anna"`). Стакан объявляется один раз, а налить в него можно что угодно при каждом вызове.

## Три типа функций — карта местности

```
TypeScript Functions
├── Named Function       → function имя(params): тип { }
├── Anonymous Function   → let x = function(params): тип { }
└── Arrow Function       → let x = (params): тип => { }
         ↑
         В Playwright используешь КАЖДЫЙ ДЕНЬ
```

Историческая справка: Named и Anonymous — это старый стиль из ранних версий JavaScript. Arrow Functions появились в ES6 (2015) и с тех пор являются стандартом. В Playwright и современном TypeScript ты будешь видеть и писать почти исключительно Arrow Functions.

Но Named и Anonymous нужно понимать, потому что:

- они встречаются в старом коде и библиотеках
- на интервью спросят про все три
- Arrow Function — это просто более короткая запись Anonymous Function, и нужно понимать от чего она произошла

## NAMED FUNCTIONS — Именованные функции

### Синтаксис named function

```typescript
function имяФункции(параметр1: тип1, параметр2: тип2): типВозврата {
    // тело функции
    return значение; // если есть возврат
}

// Вызов (invocation / calling)
имяФункции(аргумент1, аргумент2);
```

Что здесь обязательно, а что нет:

- `function` — обязательно
- имя функции — обязательно (это и есть "named")
- параметры — опционально, зависит от задачи
- `: типВозврата` — опционально (TypeScript угадает через type inference)
- `return` — обязательно только если указан тип возврата не `void`

---

### Вариант 1 — без параметров без возврата

Самая простая функция — просто делает что-то и ничего не возвращает.

```typescript
function display(): void {
    console.log("Welcome to TypeScript");
}

// Вызов
display();
// Вывод: Welcome to TypeScript
```

Важный момент: функция не запустится сама по себе. Пока ты не напишешь `display()` — код внутри мёртв. Это не Python-скрипт, который выполняется сверху вниз построчно. Код внутри функции выполняется только в момент вызова.

Проверь сам — если закомментировать вызов:

```typescript
function display(): void {
    console.log("Welcome to TypeScript");
}

// display(); // закомментировали
// Запускаем — тишина. Ничего не выведется.
```

---

### Вариант 2 — с параметрами и возвратом значения

```typescript
function addNumbers(x: number, y: number): number {
    return x + y;
}
```

Три способа использовать возвращаемое значение:

```typescript
// Способ 1: сохранить в переменную, потом вывести
let result: number = addNumbers(2, 3);
console.log(result); // 5

// Способ 2: прямо в console.log (удобнее, короче)
console.log(addNumbers(2, 3)); // 5

// Способ 3: использовать в другом выражении
let doubled = addNumbers(2, 3) * 2;
console.log(doubled); // 10
```

TypeScript строго следит за типами. Если функция объявлена как возвращающая `number`, а ты попытаешься вернуть строку — получишь ошибку ещё до запуска:

```typescript
function addNumbers(x: number, y: number): number {
    return "hello"; // ОШИБКА компилятора: Type 'string' is not assignable to type 'number'
}
```

Это и есть главное преимущество TypeScript перед JavaScript — ошибки видно сразу в редакторе, а не во время выполнения теста в 3 часа ночи.

### Вариант 3 — Rest параметры безлимитные аргументы

**Проблема:** что если ты не знаешь заранее, сколько аргументов передашь?

```typescript
// Фиксированные параметры — строгий счёт
function addTwoNumbers(x: number, y: number): number {
    return x + y;
}

addTwoNumbers(1, 2);       // OK
addTwoNumbers(1, 2, 3);    // ОШИБКА: Expected 2 arguments, but got 3
addTwoNumbers(1);           // ОШИБКА: Expected 2 arguments, but got 1
```

**Решение: Rest параметры** через три точки `...`

```typescript
function addNumbers(...nums: number[]): void {
    let sum = 0;
    for (let n of nums) {
        sum += n;
    }
    console.log("Sum:", sum);
}

addNumbers(1, 2);               // Sum: 3
addNumbers(1, 2, 3);            // Sum: 6
addNumbers(10, 20, 30, 40, 50); // Sum: 150
addNumbers(100);                // Sum: 100
```

Как это работает внутри: три точки `...nums` говорят TypeScript — "собери все переданные аргументы в массив с именем `nums`". Поэтому тип указывается как `number[]` (массив чисел), а не просто `number`.

Аналогия: параметры как чашки — обычные параметры это отдельные чашки (`x`, `y`, `z`), а Rest параметр — это поднос, на который кладёшь сколько угодно чашек.

**Ограничение:** Rest параметр всегда должен быть последним:

```typescript
// ПРАВИЛЬНО — rest последний
function log(prefix: string, ...messages: string[]): void {
    messages.forEach(msg => console.log(`[${prefix}] ${msg}`));
}

// ОШИБКА — rest не может быть в середине
function wrong(...messages: string[], suffix: string): void { } // Компилятор не позволит
```

### Вариант 4 — Rest с несколькими типами Union

Что если нужно принимать и числа, и строки вперемешку?

```typescript
// Union тип в Rest параметре
function findElements(...elements: (number | string)[]): number {
    return elements.length;
}

// Можно мешать числа и строки
console.log(findElements(3, "John", 42, "Ana", 7)); // 5

// Можно только числа
console.log(findElements(10, 20, 30, 40, 50, 60, 70)); // 7

// Можно только строки
console.log(findElements("abc", "xyz")); // 2

// НЕЛЬЗЯ — boolean не указан в union
findElements(true, 1, "test"); // ОШИБКА компилятора
```

Запись `(number | string)[]` читается как: "массив элементов, каждый из которых может быть либо number, либо string". Скобки вокруг union типа обязательны — без них `number | string[]` означало бы "number ИЛИ массив строк", что не то что нам нужно.

### Вариант 5 — Optional параметры необязательные

Знак вопроса `?` делает параметр необязательным. Можешь передать — передай. Не передал — функция всё равно работает.

```typescript
function displayDetails(id: number, name: string, mailId?: string): void {
    console.log("ID:", id);
    console.log("Name:", name);
    if (mailId !== undefined) {
        console.log("Email:", mailId);
    }
}

// Три аргумента
displayDetails(123, "Scott", "scott@gmail.com");
// ID: 123
// Name: Scott
// Email: scott@gmail.com

// Два аргумента — тоже работает
displayDetails(123, "Scott");
// ID: 123
// Name: Scott
// (email не выведется)
```

Что происходит, если НЕ передать optional параметр? Он получает значение `undefined` — не `null`, не `0`, не пустую строку — именно `undefined`. Поэтому проверяем через `!== undefined`.

**Железное правило:** optional параметры идут строго ПОСЛЕ обязательных. Это логично — нельзя пропустить обязательный параметр посередине.

```typescript
// ОШИБКА: обязательный параметр после optional
function bad(id?: number, name: string): void { }
// A required parameter cannot follow an optional parameter

// ПРАВИЛЬНО: optional всегда в конце
function good(name: string, id?: number, email?: string): void { }

// ПРАВИЛЬНО: все optional (тоже можно)
function allOptional(name?: string, id?: number): void { }
```

### Вариант 6 — Default параметры значение по умолчанию

Default параметр — это параметр с заготовленным значением. Если передашь — берётся переданное. Если нет — берётся заготовленное.

```typescript
function calculateDiscount(price: number, rate: number = 0.50): void {
    let discount = price * rate;
    console.log("Discount:", discount);
}

// Передали оба параметра — rate = 0.30
calculateDiscount(1000, 0.30); // Discount: 300

// Передали только price — rate = 0.50 (default)
calculateDiscount(1000);       // Discount: 500

// Передали оба опять — снова берётся переданный
calculateDiscount(2000, 0.10); // Discount: 200
```

**Чем default отличается от optional?**

Optional (`?`) — параметр может быть не передан, внутри будет `undefined`, логику обрабатываешь сам.

Default (`= значение`) — параметр может быть не передан, но он никогда не будет `undefined` — всегда будет либо переданное значение, либо значение по умолчанию. Это удобнее.

```typescript
// Optional — нужно проверять undefined
function withOptional(timeout?: number): void {
    const t = timeout !== undefined ? timeout : 5000; // приходится писать проверку
    console.log(t);
}

// Default — значение уже гарантировано
function withDefault(timeout: number = 5000): void {
    console.log(timeout); // 5000 или то, что передали
}
```

В Playwright default параметры очень удобны для таймаутов и флагов конфигурации.

## ANONYMOUS FUNCTIONS — Анонимные функции

### Что такое анонимная функция и зачем она

"Анонимная" = без имени. Вместо имени — переменная, которая хранит функцию.

Зачем? Иногда функция нужна как одноразовая вещь — передать куда-то, сохранить в переменную, использовать в конкретном контексте. Давать ей имя нет смысла.

### Синтаксис anonymous function

```typescript
let переменная = function(параметры): типВозврата {
    // тело
};

// Вызов — используем имя переменной
переменная(аргументы);
```

Переменная в данном случае — не просто контейнер для числа или строки. Это контейнер для ФУНКЦИИ. Переменная хранит саму функцию как объект, и вызов через `()` запускает эту функцию.

### Примеры анонимных функций

```typescript
// Без параметров, с возвратом
let msg = function(): string {
    return "Hello TypeScript";
};
console.log(msg()); // Hello TypeScript

// С параметрами
let multiply = function(a: number, b: number): number {
    return a * b;
};
console.log(multiply(10, 20)); // 200

// С optional параметром
let greetUser = function(name: string, title?: string): string {
    if (title !== undefined) {
        return `Hello, ${title} ${name}`;
    }
    return `Hello, ${name}`;
};
console.log(greetUser("Scott", "Mr.")); // Hello, Mr. Scott
console.log(greetUser("Anna"));          // Hello, Anna
```

**Ключевое отличие от named function:** только одно — нет имени после `function`. Всё остальное (параметры, тип возврата, optional, default, rest) работает абсолютно так же.

## ARROW FUNCTIONS — Стрелочные функции

### Почему именно arrow functions в Playwright

Arrow functions — это буквально язык Playwright. Открой любой пример в документации Playwright:

```typescript
test('basic test', async ({ page }) => {     // arrow function!
    await page.goto('https://example.com');
    await expect(page).toHaveTitle(/Example/);
});

test.beforeEach(async ({ page }) => {        // arrow function!
    await page.goto('/login');
});

const login = async (page: Page) => {       // arrow function!
    await page.fill('#email', 'test@test.com');
};
```

Стрелка (fat arrow) везде. Это современный стандарт JavaScript/TypeScript.

### Синтаксис arrow function

```typescript
let переменная = (параметры): типВозврата => {
    // тело
};
```

Что изменилось по сравнению с anonymous function:

- слово `function` убрали полностью
- после параметров и типа возврата добавили стрелку (fat arrow, символ равно плюс больше)
- переменная (имя) осталась — функция всё ещё anonymous, имя только у переменной

```typescript
// Anonymous (старый стиль)
let add = function(a: number, b: number): number {
    return a + b;
};

// Arrow (современный стиль) — то же самое, короче
let add = (a: number, b: number): number => {
    return a + b;
};
```

Примеры:

```typescript
// Без параметров, без возврата
let greeter = (): void => {
    console.log("Hello TypeScript");
};
greeter();

// С параметрами, с возвратом
let add = (a: number, b: number): number => {
    return a + b;
};
console.log(add(10, 20)); // 30

// С optional параметром
let displayInfo = (name: string, age?: number): void => {
    console.log(name);
    if (age !== undefined) console.log(age);
};
```

### Implicit Return — магия однострочных функций

Это особенность arrow functions которой нет у named и anonymous.

Если тело функции содержит **ровно одно выражение** — можно убрать фигурные скобки и слово `return`. Функция вернёт это выражение автоматически (неявно, implicit).

```typescript
// Полная запись (explicit return)
let add = (a: number, b: number): number => {
    return a + b;
};

// Сокращённая запись (implicit return)
let add = (a: number, b: number): number => a + b;

// Ещё примеры
let multiply = (a: number, b: number): number => a * b;
let isAdult = (age: number): boolean => age >= 18;
let greet = (name: string): string => `Hello, ${name}!`;
let double = (n: number): number => n * 2;
```

Это не просто синтаксический сахар — в реальных тестах ты будешь это видеть постоянно:

```typescript
// Playwright — фильтрация локаторов через implicit return
const visibleItems = await page.locator('.item').filter({
    has: page.locator('.badge')
});

// Array методы с implicit return — очень часто в тестах
const prices = ['$10', '$25', '$5'];
const numbers = prices.map(p => parseInt(p.replace('$', '')));
// [10, 25, 5]

const expensive = numbers.filter(n => n > 10);
// [25]
```

**Когда implicit return работает:** только когда одно выражение. Как только логики больше — фигурные скобки обязательны:

```typescript
// НЕ РАБОТАЕТ как implicit return — два выражения
let process = (x: number): number => {
    console.log("processing:", x); // первое выражение
    return x * 2;                  // второе выражение — нужны фигурные скобки!
};
```

### Arrow функции со всеми типами параметров

Optional, Default и Rest — всё работает точно так же, как в named functions:

```typescript
// Optional
let displayDetails = (id: number, name: string, mailId?: string): void => {
    console.log("ID:", id);
    console.log("Name:", name);
    if (mailId !== undefined) console.log("Email:", mailId);
};
displayDetails(1, "Scott");
displayDetails(1, "Scott", "scott@gmail.com");

// Default
let calculateDiscount = (price: number, rate: number = 0.50): void => {
    console.log("Discount:", price * rate);
};
calculateDiscount(1000);       // 500 (default rate)
calculateDiscount(1000, 0.30); // 300 (переданный rate)

// Rest
let findElements = (...elements: (number | string)[]): number => {
    return elements.length;
};
console.log(findElements(1, "a", 2, "b", 3)); // 5
```

### Эволюция от named к arrow

Полезно видеть как одна и та же функция выглядит в трёх разных стилях:

```typescript
// Named — старый стиль, явное имя
function calculateDiscount(price: number, rate: number = 0.50): void {
    let discount = price * rate;
    console.log("Discount:", discount);
}

// Anonymous — убрали имя, добавили переменную
let calculateDiscount = function(price: number, rate: number = 0.50): void {
    let discount = price * rate;
    console.log("Discount:", discount);
};

// Arrow — убрали 'function', добавили стрелку
let calculateDiscount = (price: number, rate: number = 0.50): void => {
    let discount = price * rate;
    console.log("Discount:", discount);
};

// Arrow с implicit return (если одна строка)
let getDiscount = (price: number, rate: number = 0.50): number => price * rate;
```

Все четыре — функционально идентичны. Разница только в синтаксисе.

## CALLBACK FUNCTIONS — Функции обратного вызова

### Проблема которую решает callback

JavaScript и TypeScript асинхронны по своей природе. Это значит, что если у тебя есть три операции, они могут выполниться не по порядку — браузер не ждёт первую операцию чтобы начать вторую.

Представь такую ситуацию:

```typescript
// Ожидаемый порядок: 1, 2, 3
// Реальный порядок в JS без управления: 1, 3, 2 (операция 2 асинхронная)

function fetchData(): void {
    // Эта операция занимает время (запрос к серверу)
    setTimeout(() => console.log("2. Данные получены"), 1000);
}

console.log("1. Начало");
fetchData();
console.log("3. Конец"); // Выполнится РАНЬШЕ чем fetchData завершится!
```

Callback — один из способов сказать: "Эй, когда закончишь своё дело — вызови вот эту функцию".

**Важная оговорка от лектора:** в Playwright эта проблема решается через `async/await`, который ты уже видел в тестах. Callback функции — это скорее для общего понимания JavaScript и для интервью. В реальных тестах ты будешь использовать `await`.

### Аналогия из реальной жизни

Ты звонишь в пиццерию и заказываешь пиццу. Ты не стоишь у телефона ожидая когда они приготовят. Ты даёшь им свой номер (callback) и говоришь: "Когда будет готово — позвоните мне". Пиццерия (главная функция) делает своё дело, а когда готово — вызывает твой номер (callback).

### Синтаксис callback в TypeScript подробно

```typescript
// Шаг 1: Создаём функцию, которая будет callback-ом
function showMessage(message: string): void {
    console.log(message);
}

// Шаг 2: Создаём функцию, которая ПРИНИМАЕТ callback как параметр
//                                      ↓↓↓↓↓↓↓↓↓↓↓↓↓↓↓↓↓↓↓↓↓↓↓↓↓↓↓↓
function greet(name: string, smg: (message: string) => void): void {
    console.log(name);
    smg("Hello!"); // вызываем callback внутри функции
}

// Шаг 3: Вызываем greet, передавая showMessage как аргумент
//                          ↓↓↓↓↓↓↓↓↓↓↓↓
greet("John", showMessage); // передаём БЕЗ скобок — это ссылка на функцию!
// John
// Hello!
```

Синтаксис типа callback параметра в TypeScript:

```typescript
// Как описать тип функции-параметра:
// имяПараметра: (параметрыФункции) => типВозврата

// Функция без параметров, ничего не возвращает:
callback: () => void

// Функция с одним параметром, ничего не возвращает:
callback: (message: string) => void

// Функция с параметром, возвращает число:
callback: (result: number) => number

// Функция с двумя параметрами:
callback: (a: number, b: string) => boolean
```

Полный пример с числами:

```typescript
// Callback функция — будет вызвана с результатом
function displayResult(result: number): void {
    console.log("Result:", result);
}

// Главная функция принимает callback
function sum(a: number, b: number, callback: (result: number) => void): void {
    let result = a + b;
    callback(result); // вызываем callback с результатом
}

// Вызов: передаём числа и функцию
sum(10, 20, displayResult);
// Result: 30

// Можно передать стрелочную функцию прямо при вызове
sum(5, 7, (result) => {
    console.log("Сумма равна:", result);
});
// Сумма равна: 12
```

### Callback и асинхронность — связь с Playwright

В Playwright ты каждый день видишь callback в таком виде:

```typescript
// test() принимает callback функцию
test('login test', async ({ page }) => {  // эта стрелочная функция — callback!
    await page.goto('/login');
    await page.fill('#email', 'test@test.com');
});

// beforeEach() тоже принимает callback
test.beforeEach(async ({ page }) => {     // тоже callback!
    await page.goto('/');
});

// Методы массивов используют callback
const items = ['apple', 'banana', 'cherry'];
items.forEach((item) => {                 // callback для каждого элемента
    console.log(item);
});

const lengths = items.map((item) => item.length); // [5, 6, 6]
```

Ключевое слово `async` перед параметрами стрелочной функции — это тоже часть системы асинхронности. `await` внутри говорит: "Подожди пока эта операция завершится, только потом иди дальше". Это более современный и удобный способ по сравнению с callback-цепочками.

## FUNCTION OVERLOADING — Перегрузка функций

### Идея перегрузки зачем это нужно

Представь функцию `getInfo`. Ей можно передать ID пользователя (число) или имя пользователя (строка). В обоих случаях она возвращает информацию о пользователе, но логика внутри разная.

Без перегрузки ты создал бы две разные функции: `getInfoById` и `getInfoByName`. С перегрузкой — одна функция `getInfo`, которая умеет работать с разными типами данных.

```typescript
getInfo(101);      // передаём ID — число
getInfo("Anna");   // передаём имя — строку
// Обе строки вызывают ОДНУ функцию с разным поведением
```

### Три шага перегрузки — пошаговый разбор

TypeScript требует три чётких шага. Нарушишь порядок — получишь ошибку.

**Шаг 1: Написать сигнатуры** (только объявления, без тела)

```typescript
// Это ТОЛЬКО сигнатуры — без реализации!
function getInfo(id: number): string;
function getInfo(name: string): string;
// Тут нет тела { } — это намеренно
```

**Шаг 2: Написать ОДНУ реализацию** (которая удовлетворяет всем сигнатурам)

```typescript
// Реализация — одна функция для всех сигнатур
function getInfo(param: number | string): string {
    if (typeof param === "number") {
        return `User ID is ${param}`;
    } else {
        return `Username is ${param}`;
    }
}
```

**Шаг 3: Вызвать функцию**

```typescript
console.log(getInfo(101));    // User ID is 101
console.log(getInfo("Anna")); // Username is Anna
```

### Пример 1 — разные типы параметров

```typescript
// Шаг 1: сигнатуры
function getInfo(id: number): string;
function getInfo(name: string): string;

// Шаг 2: реализация
function getInfo(param: number | string): string {
    if (typeof param === "number") {
        return `User ID is ${param}`;
    } else {
        return `Username is ${param}`;
    }
}

// Шаг 3: вызовы
console.log(getInfo(101));    // User ID is 101
console.log(getInfo("Anna")); // Username is Anna
```

Обрати внимание: `typeof param` — это встроенный оператор JavaScript/TypeScript, который возвращает тип переменной в виде строки: `"number"`, `"string"`, `"boolean"`, `"undefined"`, `"object"`, `"function"`.

### Пример 2 — разное количество параметров

```typescript
// Шаг 1: сигнатуры — разное КОЛИЧЕСТВО параметров
function add(a: number, b: number): number;
function add(a: number, b: number, c: number): number;

// Шаг 2: реализация
// c делаем optional — так реализация удовлетворяет обеим сигнатурам
function add(a: number, b: number, c?: number): number {
    if (c !== undefined) {
        return a + b + c;
    }
    return a + b;
}

// Шаг 3: вызовы
console.log(add(10, 20));     // 30  — первая сигнатура
console.log(add(10, 20, 30)); // 60  — вторая сигнатура
```

Почему `c` в реализации optional? Потому что первая сигнатура принимает 2 параметра. Если бы `c` был обязательным, реализация не совпадала бы с первой сигнатурой.

### Пример 3 — разные типы возврата

```typescript
// Шаг 1: сигнатуры — РАЗНЫЕ типы возврата!
function processInput(str: string): string;
function processInput(num: number): number;

// Шаг 2: реализация — union и для параметра, и для возврата
function processInput(input: string | number): string | number {
    if (typeof input === "string") {
        return input.toUpperCase(); // строку конвертируем в верхний регистр
    } else {
        return input * input; // число возводим в квадрат
    }
}

// Шаг 3: вызовы
console.log(processInput("hello")); // HELLO
console.log(processInput(5));       // 25
console.log(processInput(10));      // 100
```

### Пример 4 — три сигнатуры с boolean

```typescript
// Шаг 1: три сигнатуры
function greet(name: string): string;
function greet(age: number): string;
function greet(isMarried: boolean): string;

// Шаг 2: реализация удовлетворяет всем трём
function greet(value: string | number | boolean): string {
    if (typeof value === "string") {
        return `Hello, ${value}`;
    } else if (typeof value === "number") {
        return `You are ${value} years old`;
    } else {
        // boolean: ternary operator value ? "married" : "single"
        return value ? "married" : "single";
    }
}

// Шаг 3: вызовы с разными типами
console.log(greet("John")); // Hello, John
console.log(greet(30));     // You are 30 years old
console.log(greet(true));   // married
console.log(greet(false));  // single
```

**Бонус — тернарный оператор:** `value ? "married" : "single"` — это краткая форма if/else. Читается как: "если value истинно — вернуть 'married', иначе — 'single'". Очень часто встречается в TypeScript коде.

### Правила валидных и невалидных перегрузок

**МОЖНО:**

```typescript
// Разные типы параметров — МОЖНО
function test(a: number): string;
function test(a: string): string;

// Разное количество параметров — МОЖНО
function test(a: number): string;
function test(a: number, b: number): string;

// Разные типы возврата — МОЖНО
function test(a: string): string;
function test(a: number): number;
```

**НЕЛЬЗЯ:**

```typescript
// НЕЛЬЗЯ: одинаковые сигнатуры — конфликт
function test(a: number): string;
function test(a: number): string; // дубликат — ОШИБКА

// НЕЛЬЗЯ: реализация есть но возврат не совпадает с сигнатурами
function calculate(a: number): string;
function calculate(a: string): string;
function calculate(a: number | string): boolean { // boolean не то что ожидают сигнатуры!
    return true; // ОШИБКА
}

// НЕЛЬЗЯ: сигнатуры без реализации
function test(a: number): string;
function test(a: string): string;
// нет тела — ОШИБКА компилятора

// НЕЛЬЗЯ: конфликт в optional параметрах
function test(a: number, b?: number): string;
function test(a: number, b: number, c: number): string;
// В первой b — optional, во второй — обязательный. Конфликт.
```

**Главное правило:** реализация должна быть совместима со ВСЕМИ сигнатурами — и по параметрам, и по типу возврата.

## Сравнение всех трёх типов функций

```typescript
// Одна и та же функция в трёх стилях:

// Named
function addNumbers(a: number, b: number): number {
    return a + b;
}

// Anonymous
let addNumbers = function(a: number, b: number): number {
    return a + b;
};

// Arrow
let addNumbers = (a: number, b: number): number => {
    return a + b;
};

// Arrow с implicit return (самый короткий)
let addNumbers = (a: number, b: number): number => a + b;
```

|Характеристика|Named|Anonymous|Arrow|
|---|---|---|---|
|Ключевое слово `function`|да|да|нет|
|Имя функции|да (обязательно)|нет|нет|
|Нужна переменная|нет|да|да|
|Implicit return|нет|нет|да|
|Optional параметры|да|да|да|
|Default параметры|да|да|да|
|Rest параметры|да|да|да|
|Поддержка `this` (OOP)|да|да|нет (у arrow нет своего this)|
|Использование в Playwright|редко|редко|**постоянно**|
|Современный стандарт|нет|нет|**да**|

Особенность стрелочных функций: у них нет своего `this`. Это важно при работе с классами (OOP), но в контексте Playwright это почти никогда не мешает.

### Page Object с методами — функции внутри класса

В реальных проектах функции живут внутри классов (Page Object Model). Функции внутри класса называются методами, но синтаксис тот же:

```typescript
import { Page, Locator } from '@playwright/test';

export class LoginPage {
    private page: Page;
    private emailInput: Locator;
    private passwordInput: Locator;
    private submitButton: Locator;

    constructor(page: Page) {
        this.page = page;
        this.emailInput = page.locator('#email');
        this.passwordInput = page.locator('#password');
        this.submitButton = page.locator('[type="submit"]');
    }

    // Методы — это функции внутри класса
    async goto(): Promise<void> {
        await this.page.goto('/login');
    }

    async fillEmail(email: string): Promise<void> {
        await this.emailInput.fill(email);
    }

    async fillPassword(password: string): Promise<void> {
        await this.passwordInput.fill(password);
    }

    async submit(): Promise<void> {
        await this.submitButton.click();
    }

    // Функция которая объединяет все шаги
    async login(email: string, password: string): Promise<void> {
        await this.goto();
        await this.fillEmail(email);
        await this.fillPassword(password);
        await this.submit();
        await this.page.waitForURL('/dashboard');
    }
}
```

### Helper функции — твой арсенал

Собери библиотеку функций которые пригодятся в любом проекте:

```typescript

    selector: string,

    page: Page,

// Rest параметр — проверяем сколько угодно элементов
const verifyAllVisible = async (page: Page, ...selectors: string[]): Promise<void> => {
    for (const selector of selectors) {

};

const getTextContent = async (page: Page, selector: string): Promise<string> => {
    const text = await page.locator(selector).textContent();

```

## Частые ошибки и как их избежать

**Ошибка 1: Функция объявлена но не вызвана**

```typescript
// ОШИБКА: определили функцию, забыли вызвать
function runTests(): void {
    console.log("Tests running...");
}
// Запускаем файл — тишина.

// ПРАВИЛЬНО
function runTests(): void {
    console.log("Tests running...");
}
runTests(); // вызов обязателен
```

**Ошибка 2: Передали ссылку на функцию вместо вызова (или наоборот)**

```typescript
// Передали результат вместо функции
sum(10, 20, displayResult()); // ОШИБКА: displayResult() вернёт undefined, а не функцию!
sum(10, 20, displayResult);   // ПРАВИЛЬНО: передаём ссылку на функцию

// В Playwright — то же самое
test.beforeEach(login());     // ОШИБКА: login() вызывается сразу, возвращает Promise
test.beforeEach(login);       // Правильно только если login принимает правильные параметры
```

**Ошибка 3: Optional параметр перед обязательным**

```typescript
// ОШИБКА
function register(email?: string, password: string): void { }
// A required parameter cannot follow an optional parameter

// ПРАВИЛЬНО: optional — только в конце
function register(password: string, email?: string): void { }
```

**Ошибка 4: Забыл return в функции с типом возврата**

```typescript
// TypeScript покажет ошибку:
function add(a: number, b: number): number {
    a + b; // вычислили, но не вернули! Функция вернёт undefined
}
// Not all code paths return a value.

// ПРАВИЛЬНО:
function add(a: number, b: number): number {
    return a + b;
}
```

**Ошибка 5: Передаём функции больше аргументов чем параметров**

```typescript
function add(x: number, y: number): number {
    return x + y;
}

add(1, 2, 3); // ОШИБКА: Expected 2 arguments, but got 3.
              // Используй rest параметры если нужно передавать разное количество
```

**Ошибка 6: В overloading — реализация не совпадает с сигнатурами**

```typescript
function format(val: number): string;
function format(val: string): string;

// ОШИБКА: тип параметра реализации не включает все типы из сигнатур
function format(val: number): string {  // только number, а string забыли!
    return String(val);
}

// ПРАВИЛЬНО: реализация принимает union всех типов из сигнатур
function format(val: number | string): string {
    return String(val);
}
```

## Шпаргалка — всё в одном месте

```typescript
// ─────────────────────────────────────────────────────────
// NAMED FUNCTION
function имя(param: тип): возврат { return value; }

// ANONYMOUS FUNCTION
let имя = function(param: тип): возврат { return value; };

// ARROW FUNCTION (современный стандарт)
let имя = (param: тип): возврат => { return value; };
let имя = (param: тип): возврат => value; // implicit return

// ─────────────────────────────────────────────────────────
// ПАРАМЕТРЫ

// Обязательный
function f(name: string): void { }

// Optional — может быть не передан, будет undefined
function f(name: string, age?: number): void { }

// Default — значение по умолчанию если не передан
function f(name: string, role: string = "user"): void { }

// Rest — принимает любое количество аргументов
function f(...nums: number[]): void { }
function f(...items: (number | string)[]): void { } // с union

// Порядок обязателен: обычные, optional/default, rest
function f(a: string, b?: number, ...c: string[]): void { }

// ─────────────────────────────────────────────────────────
// CALLBACK

// Тип параметра-функции: (params) => returnType
function run(callback: () => void): void { callback(); }
function run(callback: (n: number) => string): void { callback(42); }

// ─────────────────────────────────────────────────────────
// OVERLOADING

// Шаг 1: сигнатуры (без тела)
function fn(a: number): string;
function fn(a: string): number;

// Шаг 2: одна реализация (union параметры и возврат)
function fn(a: number | string): string | number {
    if (typeof a === "number") return String(a);
    else return a.length;
}

// Шаг 3: вызов
fn(42);      // вызывает первую сигнатуру
fn("hello"); // вызывает вторую сигнатуру

// ─────────────────────────────────────────────────────────
// В PLAYWRIGHT — типичные паттерны

// Helper функция
const login = async (page: Page, email: string, pass: string): Promise<void> => {
    await page.fill('#email', email);
    await page.fill('#password', pass);
    await page.click('#submit');
};

// С default таймаутом
const waitFor = async (page: Page, sel: string, ms: number = 5000): Promise<void> => {
    await page.waitForSelector(sel, { timeout: ms });
};

// С rest — проверить несколько элементов
const allVisible = async (page: Page, ...sels: string[]): Promise<void> => {
    for (const s of sels) await expect(page.locator(s)).toBeVisible();
};
```
