## Почему это самая важная тема для Playwright

Открой любой Playwright тест — и увидишь `async` и `await` буквально везде:

```typescript
test('login test', async ({ page }) => {
    await page.goto('/login');
    await page.fill('#email', 'user@test.com');
    await page.fill('#password', '123456');
    await page.click('#submit');
    await expect(page).toHaveURL('/dashboard');
});
```

Каждая строка с `await` — это асинхронная операция. Браузер не мгновенно загружает страницы, не мгновенно находит элементы, не мгновенно кликает. Всё это занимает время. Playwright нужно **ждать** завершения каждого действия перед следующим.

Если убрать `await` — тесты будут падать в случайных местах, потому что следующее действие начнётся до того как предыдущее завершилось.

## Синхронный vs Асинхронный код

### Синхронный код — построчно, одно за другим

```typescript
console.log("1 — начали");
console.log("2 — середина");
console.log("3 — конец");

// Вывод ВСЕГДА:
// 1 — начали
// 2 — середина
// 3 — конец
```

Каждая строка ждёт завершения предыдущей. Простой, понятный порядок.

### Асинхронный код — некоторые операции занимают время

```typescript
console.log("1 — начали");

setTimeout(() => {
    console.log("2 — setTimeout завершился");
}, 1000);

console.log("3 — конец");

// Реальный вывод:
// 1 — начали
// 3 — конец
// 2 — setTimeout завершился  ← через секунду!
```

`setTimeout` не блокирует код. JS продолжает выполнение и возвращается к колбэку когда таймер сработал.

### Какие операции асинхронны в реальном мире

- Запросы к серверу (API calls, fetch)
- Работа с файлами (чтение, запись)
- Работа с базой данных
- Таймеры (`setTimeout`, `setInterval`)
- Действия в браузере через Playwright (клик, заполнение, навигация)

## Promise — что это такое

**Promise** (промис, обещание) — это объект, который представляет результат асинхронной операции, которая ещё не завершилась.

Аналогия: ты заказал пиццу. Тебе дали **чек** (promise). Пицца ещё не готова, но чек говорит: "Когда будет готово, ты получишь пиццу либо сообщение об ошибке".

### Как выглядит Promise в коде

```typescript
// Создание Promise (обычно делают библиотеки, не ты сам)
const myPromise = new Promise<string>((resolve, reject) => {
    // resolve — вызывается если всё OK
    // reject — вызывается если ошибка

    setTimeout(() => {
        const success = true;
        if (success) {
            resolve("Данные получены!");
        } else {
            reject(new Error("Что-то пошло не так"));
        }
    }, 1000);
});

// Использование Promise через .then() / .catch() — старый стиль
myPromise
    .then((result) => console.log(result))      // "Данные получены!"
    .catch((error) => console.log(error.message));
```

Синтаксис `.then()/.catch()` — старый способ работы с промисами. `async/await` — современный и более читаемый способ делать то же самое.

## Три состояния Promise

Каждый промис находится в одном из трёх состояний:

| Состояние | Описание | Что происходит дальше |
|-----------|----------|-----------------------|
| **pending** | Ожидает, операция ещё выполняется | Ждём результата |
| **fulfilled** | Успешно завершён | Вызывается `.then()` или продолжается после `await` |
| **rejected** | Завершён с ошибкой | Вызывается `.catch()` или блок `catch` в try/catch |

```typescript
// Пример с разными состояниями:
const p1 = Promise.resolve("Сразу fulfilled");  // fulfilled мгновенно
const p2 = Promise.reject(new Error("Ошибка"));  // rejected мгновенно
const p3 = new Promise(resolve => setTimeout(resolve, 5000)); // pending 5 секунд
```

## async — ключевое слово для функции

Ключевое слово `async` перед функцией делает две вещи:

1. Функция **автоматически возвращает Promise**
2. Внутри функции можно использовать `await`

```typescript
// Обычная функция — возвращает string
function greet(): string {
    return "Hello!";
}

// Async функция — возвращает Promise<string>
async function greetAsync(): Promise<string> {
    return "Hello!";  // TypeScript автоматически оборачивает в Promise
}

// Вызов:
const result = greetAsync();
console.log(result);  // Promise { 'Hello!' } — не строка!

// Правильный способ получить значение:
const text = await greetAsync();
console.log(text);  // "Hello!"
```

### async со всеми видами функций

```typescript
// Named async function
async function fetchData(): Promise<string> {
    return "data";
}

// Arrow async function (самый частый вид в Playwright)
const fetchUser = async (id: number): Promise<string> => {
    return `User ${id}`;
};

// Async метод в объекте
const api = {
    getData: async (): Promise<number[]> => {
        return [1, 2, 3];
    }
};

// Async метод в классе
class DataService {
    async load(): Promise<void> {
        // что-то делаем
    }
}
```

## await — ключевое слово для ожидания

`await` ставится перед вызовом async функции (или Promise) и **ждёт** пока промис завершится. Пока ждёт — остальной код в программе продолжает выполняться (JavaScript не блокируется).

**Важно:** `await` можно использовать **только внутри async функций**.

```typescript
// Функция имитирующая задержку (как будто запрос к серверу)
const delay = (ms: number): Promise<void> => {
    return new Promise(resolve => setTimeout(resolve, ms));
};

// Функция имитирующая загрузку данных
const loadUserName = async (id: number): Promise<string> => {
    await delay(500);  // ждём 500ms
    return `User_${id}`;
};

// Использование await:
async function main() {
    console.log("Начинаем загрузку...");
    const name = await loadUserName(42);  // ждём здесь
    console.log(`Получили: ${name}`);     // выполнится после загрузки
    console.log("Загрузка завершена");
}

main();

// Вывод:
// Начинаем загрузку...
// Получили: User_42
// Загрузка завершена
```

### Без await — ошибка которую сложно найти

```typescript
async function wrongExample() {
    console.log("Начали");
    const name = loadUserName(42);  // ЗАБЫЛИ await!
    console.log(`Получили: ${name}`);  // Promise { <pending> } — не строка!
}
```

## Обработка ошибок — try / catch / finally

Когда промис отклоняется (rejected), он выбрасывает исключение. Его нужно поймать через `try/catch`.

```typescript
const riskyOperation = async (): Promise<string> => {
    const success = Math.random() > 0.5;
    if (!success) throw new Error("Операция не удалась");
    return "Успех!";
};

async function handleErrors(): Promise<void> {
    try {
        const result = await riskyOperation();
        console.log("Результат:", result);
    } catch (error) {
        if (error instanceof Error) {
            console.log("Поймали ошибку:", error.message);
        }
    } finally {
        console.log("Это выполнится ВСЕГДА — и при успехе, и при ошибке");
    }
}
```

### finally — для очистки ресурсов

`finally` выполняется всегда, независимо от результата. Полезен для:
- Закрытия соединений
- Освобождения ресурсов
- Логирования завершения операции

```typescript
async function readFile(): Promise<void> {
    let fileHandle = null;
    try {
        fileHandle = await openFile("data.txt");
        const content = await fileHandle.read();
        console.log(content);
    } catch (error) {
        console.log("Ошибка чтения файла");
    } finally {
        if (fileHandle) await fileHandle.close();  // всегда закрываем файл
    }
}
```

## Promise.all — параллельное выполнение

По умолчанию `await` выполняет операции последовательно. Но иногда операции не зависят друг от друга — тогда их можно запустить параллельно.

```typescript
const delay = (ms: number): Promise<void> =>
    new Promise(resolve => setTimeout(resolve, ms));

// МЕДЛЕННО — последовательно (итого ~3 секунды)
async function sequential(): Promise<void> {
    await delay(1000);  // ждём 1 секунду
    await delay(1000);  // ещё 1 секунду
    await delay(1000);  // ещё 1 секунду
    console.log("Готово через 3 секунды");
}

// БЫСТРО — параллельно (итого ~1 секунда)
async function parallel(): Promise<void> {
    await Promise.all([
        delay(1000),
        delay(1000),
        delay(1000),
    ]);
    console.log("Готово через 1 секунду!");
}
```

### Promise.all с результатами

```typescript
const fetchUser = async (id: number): Promise<string> => {
    await delay(200);
    return `User_${id}`;
};

async function loadAllUsers(): Promise<void> {
    const [user1, user2, user3] = await Promise.all([
        fetchUser(1),
        fetchUser(2),
        fetchUser(3),
    ]);
    console.log(user1, user2, user3);
    // User_1 User_2 User_3
}
```

**Важно:** если хотя бы один промис в `Promise.all` выбрасывает ошибку — весь `Promise.all` завершается с ошибкой.

## Promise.allSettled — когда важен каждый результат

В отличие от `Promise.all`, `Promise.allSettled` ждёт завершения ВСЕХ промисов, даже если некоторые упали с ошибкой.

```typescript
const results = await Promise.allSettled([
    fetchUser(1),          // успех
    Promise.reject(new Error("Упал")), // ошибка
    fetchUser(3),          // успех
]);

results.forEach((result) => {
    if (result.status === "fulfilled") {
        console.log("Успех:", result.value);
    } else {
        console.log("Ошибка:", result.reason.message);
    }
});
// Успех: User_1
// Ошибка: Упал
// Успех: User_3
```

## Async/Await в Playwright — практический гайд

### Почему каждая строка в Playwright — async

Playwright управляет реальным браузером через сеть (даже локально). Каждое действие:
- отправляет команду браузеру
- ждёт её выполнения
- получает результат

Это принципиально асинхронно.

```typescript
import { test, expect } from '@playwright/test';

test('пример async/await в тесте', async ({ page }) => {
    // goto — ждём полной загрузки страницы
    await page.goto('https://example.com');

    // fill — ждём заполнения поля
    await page.fill('#email', 'user@test.com');

    // click — ждём клика и реакции страницы
    await page.click('#submit');

    // expect — ждём соответствия условию (с таймаутом)
    await expect(page.locator('h1')).toHaveText('Welcome!');
});
```

### Helper функции — всегда async

```typescript
// Хелпер — async функция которую вызываем с await
const login = async (page: Page, email: string, password: string): Promise<void> => {
    await page.goto('/login');
    await page.fill('#email', email);
    await page.fill('#password', password);
    await page.click('[type="submit"]');
    await page.waitForURL('/dashboard');
};

test('тест после логина', async ({ page }) => {
    await login(page, 'admin@test.com', 'secret123');
    // после await login — мы точно на /dashboard
    await expect(page.locator('h1')).toHaveText('Dashboard');
});
```

### Параллельное выполнение в тестах

```typescript
test('параллельные проверки', async ({ page }) => {
    await page.goto('/dashboard');

    // Проверяем несколько элементов параллельно
    await Promise.all([
        expect(page.locator('h1')).toBeVisible(),
        expect(page.locator('#user-menu')).toBeVisible(),
        expect(page.locator('.sidebar')).toBeVisible(),
    ]);
});
```

### try/catch в тестах — для информативных ошибок

```typescript
test('тест с обработкой ошибок', async ({ page }) => {
    try {
        await page.goto('/admin');
        await expect(page.locator('#admin-panel')).toBeVisible({ timeout: 5000 });
    } catch (error) {
        console.log('Admin panel not found, checking permissions...');
        throw error;  // пробрасываем ошибку дальше — тест упадёт с нужным сообщением
    }
});
```

## Частые ошибки и как их избежать

### Ошибка 1 — забыть await

```typescript
// НЕВЕРНО — не ждём
async function badTest(page: Page): Promise<void> {
    page.goto('/login');  // не ждём загрузки страницы
    page.fill('#email', 'user@test.com');  // элемент может ещё не существовать!
}

// ВЕРНО
async function goodTest(page: Page): Promise<void> {
    await page.goto('/login');
    await page.fill('#email', 'user@test.com');
}
```

### Ошибка 2 — использовать await вне async функции

```typescript
// НЕВЕРНО — SyntaxError
function notAsync(): void {
    const result = await fetchData();  // нельзя!
}

// ВЕРНО
async function yesAsync(): Promise<void> {
    const result = await fetchData();
}
```

### Ошибка 3 — последовательно то, что можно параллельно

```typescript
// МЕДЛЕННО
async function slowChecks(page: Page): Promise<void> {
    const title = await page.title();
    const url = page.url();
    const h1 = await page.locator('h1').textContent();
}

// БЫСТРЕЕ — для независимых операций
async function fastChecks(page: Page): Promise<void> {
    const [title, h1] = await Promise.all([
        page.title(),
        page.locator('h1').textContent(),
    ]);
}
```

### Ошибка 4 — не обрабатывать ошибки

```typescript
// ПЛОХО — приложение упадёт с непонятной ошибкой
async function noErrorHandling(): Promise<void> {
    const data = await riskyFetch();
    process(data);
}

// ХОРОШО — понятное сообщение об ошибке
async function withErrorHandling(): Promise<void> {
    try {
        const data = await riskyFetch();
        process(data);
    } catch (error) {
        console.error('Не удалось загрузить данные:', error);
        throw error;
    }
}
```

## Шпаргалка — всё в одном месте

```typescript
// 1. async функция — всегда возвращает Promise
async function example(): Promise<string> {
    return "result";
}

// 2. await — ждёт Promise, только внутри async
const value = await example();

// 3. try/catch — перехватывает ошибки в async
try {
    const data = await riskyOperation();
} catch (error) {
    console.log("Ошибка:", error);
} finally {
    console.log("Всегда выполнится");
}

// 4. Promise.all — параллельное выполнение, ждёт ВСЕ
const [a, b, c] = await Promise.all([op1(), op2(), op3()]);

// 5. Promise.allSettled — параллельно, не падает при ошибке
const results = await Promise.allSettled([op1(), op2(), op3()]);

// 6. Типы async функций
async function named(): Promise<void> {}
const arrow = async (): Promise<string> => "result";
const arrowVoid = async (): Promise<void> => {};

// 7. В Playwright — await перед КАЖДЫМ действием
await page.goto('/url');
await page.fill('#selector', 'value');
await page.click('#button');
await expect(locator).toBeVisible();
```
