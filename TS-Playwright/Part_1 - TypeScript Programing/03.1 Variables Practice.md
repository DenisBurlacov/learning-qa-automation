## 🔬 ЧАСТЬ 3: Scope (область видимости)

### Задание 3.1 — Function scope vs Block scope

**Цель:** Наглядно увидеть разницу между `var` (function scope) и `let`/`const` (block scope).

**Шаги:**

1. Создай функцию `testScope`
2. Внутри функции создай блок `if (true) { ... }`
3. Внутри блока `if` создай три переменные: `varVar`, `letVar`, `constVar`
4. Внутри блока `if` — выведи все три (должно работать)
5. **Вне блока** `if`, но внутри функции — попробуй вывести каждую
6. Запусти и запиши что работает, а что даёт ошибку

<details> <summary>💡 Подсказка</summary>

`var` будет доступна вне блока `if`. `let` и `const` дадут ошибку вне блока `if`.

</details> <details> <summary>✅ Проверь себя</summary>

```typescript
function testScope() {
    if (true) {
        var varVar = "я var";
        let letVar = "я let";
        const constVar = "я const";

        // Внутри блока — работает всё:
        console.log(varVar);   // ✅ "я var"
        console.log(letVar);   // ✅ "я let"
        console.log(constVar); // ✅ "я const"
    }

    // Вне блока if:
    console.log(varVar);   // ✅ "я var" — function scope, доступна везде в функции
    // console.log(letVar);   // ❌ ОШИБКА: Cannot find name 'letVar'
    // console.log(constVar); // ❌ ОШИБКА: Cannot find name 'constVar'
}

testScope();
```

</details>

### Задание 3.2 — Объясни поведение

**Цель:** Проверить понимание scope без запуска кода.

**Посмотри на код ниже и ответь: что выведет каждый `console.log`? Ошибка или значение?**

```typescript
function quiz() {
    var a = 1;
    let b = 2;

    if (true) {
        var a = 10;     // (A)
        let b = 20;     // (B)
        const c = 30;   // (C)

        console.log(a); // (1) — что выведет?
        console.log(b); // (2) — что выведет?
        console.log(c); // (3) — что выведет?
    }

    console.log(a); // (4) — что выведет?
    console.log(b); // (5) — что выведет?
    // console.log(c); // (6) — ошибка или нет?
}

quiz();
```

<details> <summary>💡 Подсказка</summary>

Обрати внимание: `var a` внутри if — это **та же самая переменная** что и `var a` снаружи (function scope). А `let b` внутри if — это **новая отдельная переменная** (block scope).

</details> <details> <summary>✅ Проверь себя</summary>

```
(1) → 10    — var a внутри блока, уже переписана через (A)
(2) → 20    — let b внутри блока, это новая переменная из (B)
(3) → 30    — const c внутри блока ✅
(4) → 10    — var a (function scope) была изменена через (A)!
(5) → 2     — let b снаружи — это ДРУГАЯ переменная, (B) её не трогала
(6) → ОШИБКА — const c вне своего блока недоступна
```

Вот почему `var` опасен: строка (A) незаметно изменила внешнюю переменную `a`!

</details>

**Блок A:**


**Блок B:**

```typescript
console.log(fruit);

```

**Блок C:**

```typescript
console.log(color);
const color = "Red";
console.log(color);
```

<details> <summary>✅ Проверь себя</summary>

**Блок A (var):**

```
undefined    ← hoisting: var существует, но значения ещё нет
Cat          ← после присвоения значения
```

**Блок B (let):**

```

← let в "temporal dead zone" до строки объявления

```

```
← const ведёт себя так же как let
```

**Вывод:** `var` скрывает ошибку, возвращая `undefined`. `let` и `const` дают явную ошибку — это безопаснее!
