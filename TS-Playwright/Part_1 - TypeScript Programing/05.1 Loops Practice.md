## Ответы на контрольные вопросы

**1. В чём разница между `while` и `do...while`?** `while` сначала проверяет условие, потом выполняет тело. Если условие сразу false — тело не выполнится ни разу. `do...while` сначала выполняет тело, потом проверяет условие. Тело выполнится минимум один раз в любом случае.

**2. Что произойдёт если в цикле `while` забыть увеличивать счётчик?** Бесконечный цикл. Условие никогда не станет false и цикл будет работать вечно. Остановить можно только через Ctrl+C в терминале.

**3. Почему `i === 3 && i === 5` — всегда false?** Потому что переменная `i` не может одновременно равняться 3 и 5. Нужно использовать `||` (или): `i === 3 || i === 5`.

**4. Переменная объявленная внутри `for` — видна ли она снаружи?** Нет. Она существует только внутри фигурных скобок цикла. После закрывающей скобки она недоступна. Если нужна снаружи — объяви до цикла.

**5. `break` во вложенных циклах — из скольких циклов он выходит?** Только из одного — из того, в котором написан. Чтобы выйти из обоих, нужен `break` в каждом цикле (обычно через вспомогательную переменную `found`).

**6. Чем отличается `i++` от `i += 2`?** `i++` это `i = i + 1` — увеличивает на 1. `i += 2` это `i = i + 2` — увеличивает на 2. Шаг можно сделать любым: `i += 5`, `i += 10` и т.д.

**7. Какой цикл выполнится ноль раз, если условие false с самого начала?** `while`. Цикл `do...while` в этом случае выполнится один раз.

**8. Что напечатает этот код?**

```typescript
let i: number = 0;
for (i = 1; i <= 5; i++) { /* ничего */ }
console.log(i);
```

Напечатает `6`. Цикл прокрутил `i` от 1 до 5, последний шаг `i++` сделал её 6, при 6 условие `6 <= 5` стало false и цикл остановился. Так как `i` объявлена снаружи — она доступна после цикла и равна 6.
