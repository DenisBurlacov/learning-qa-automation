## JavaScript vs TypeScript — в чём разница

### Динамическая типизация (JavaScript)

**Динамически типизированный язык** — переменная может хранить данные разных типов и менять тип в любой момент. JavaScript именно такой.

Представь переменную как **коробку без этикетки** — в неё можно положить что угодно, а потом заменить содержимое на что-то совсем другое:

```javascript
// JavaScript — динамическая типизация
let age = 30;
console.log(typeof age); // "number"

age = "тридцать лет";   // кладём строку — JS молчит!
console.log(typeof age); // "string"

age = true;              // теперь boolean — JS всё ещё молчит!
console.log(typeof age); // "boolean"
```

Никаких ошибок, никаких предупреждений. JavaScript просто меняет тип переменной — это и есть **динамическая типизация**.

---

### Статическая типизация (TypeScript)

**Статически типизированный язык** — тип переменной определяется **один раз** и больше не меняется. TypeScript именно такой.

Представь переменную как **коробку с этикеткой "только яблоки"** — положил яблоко и теперь банан туда уже не влезет, TypeScript не позволит:

```typescript
// TypeScript — статическая типизация
let age: number = 30;

age = "тридцать лет";  // ❌ Ошибка на этапе написания кода!
// Type 'string' is not assignable to type 'number'

age = true;            // ❌ Тоже ошибка!
// Type 'boolean' is not assignable to type 'number'

age = 31;              // ✅ OK — это всё ещё number
```

TypeScript кричит об ошибке **прямо в редакторе**, ещё до запуска программы. Это означает — ты находишь ошибки сразу, а не когда программа уже работает у пользователей.

---

### Важно: как запускать TypeScript файлы

Для запуска TypeScript используй **только `tsx`**, а не `node`:

```bash
# ✅ Правильно:
tsx day3/DataTypesDemo.ts

# ❌ Неправильно — node не понимает TypeScript-синтаксис:
node day3/DataTypesDemo.ts
```

Почему? Потому что `node` предназначен для JavaScript. Если в файле нет явных аннотаций типов — он может сработать случайно (код выглядит как JS). Но как только добавишь `let age: number = 30` — `node` упадёт с ошибкой, потому что это чистый TypeScript-синтаксис, который он не понимает.

## Type Safety — что это и зачем

**Type Safety (типобезопасность)** — это гарантия того, что переменная всегда хранит данные именно того типа, для которого создана. JavaScript не type-safe, TypeScript — type-safe.

### Почему JavaScript не type-safe — реальный пример

```javascript
// JavaScript — скрытая ошибка типов
let result = "5" + 3;
console.log(result); // "53" — строка! А не 8!
// JavaScript просто склеил "5" и 3 как строки
// Никакой ошибки, но результат НЕПРАВИЛЬНЫЙ
```

А теперь представь что эта строка — в расчёте зарплаты. Всё выглядит как работает, но считает неверно. Найти такой баг — nightmare.

### Как TypeScript защищает

```typescript
// TypeScript — с явными типами
let price: string = "5";
let discount: number = 3;

// TypeScript сразу подчёркивает проблему!
// Ты видишь ошибку ещё не запустив код

// Правильно — оба числа:
let correctPrice: number = 5;
let correctDiscount: number = 3;
let total: number = correctPrice + correctDiscount;
console.log(total); // 8 ✅
```

> 💡 Связь простая: динамическая типизация → не type-safe (JavaScript). Статическая типизация → type-safe (TypeScript). Это два аспекта одного и того же.

## Три важных термина

Эти три понятия — частые вопросы на собеседованиях.

### 1. Type / Data Type (тип данных)

Просто название конкретного типа: `number`, `string`, `boolean` и т.д. Тип говорит TypeScript — какие значения можно хранить в переменной.

```typescript
let age: number = 30;
//        ↑
//    это тип данных (data type)
```

---

### 2. Type Annotation (аннотация типа)

**Annotation** — это когда ты **явно** указываешь тип при создании переменной через двоеточие `:`.

```typescript
// Синтаксис аннотации:
// ключевое_слово  имя  :  тип      =  значение
    let          age  : number   =  30;
    let         name  : string   =  "Alice";
    let       active  : boolean  =  true;
//                  ↑
//      это аннотация — явное указание типа

// Ты сам говоришь TypeScript: "эта переменная — только number"
// и TypeScript будет строго это контролировать
```

---

### 3. Type Inference (вывод типа)

**Type Inference** — это когда ты **не пишешь** тип, и TypeScript **автоматически** определяет его по значению которое ты присвоил.

```typescript
// Без аннотации — TypeScript сам смотрит на значение и решает:
let age = 30;        // TypeScript: "вижу число → тип number"
let name = "Alice";  // TypeScript: "вижу строку → тип string"
let active = true;   // TypeScript: "вижу boolean → тип boolean"

// Наведи курсор в VS Code — увидишь подсказку:
// let age: number  ← TypeScript сам добавил тип!
```

Даже без аннотации TypeScript тебя защищает:

```typescript
let score = 100; // TypeScript вывел: number

score = "много"; // ❌ ОШИБКА!
// Type 'string' is not assignable to type 'number'
// Type Inference тоже даёт полную защиту!
```

---

### Annotation vs Inference — сравнение

| | Type Annotation | Type Inference |
|-|-----------------|----------------|
| Кто указывает тип | Ты — явно через `:` | TypeScript — автоматически |
| Синтаксис | `let x: number = 5` | `let x = 5` |
| Защита от ошибок | ✅ Есть | ✅ Тоже есть |
| Когда применять | Параметры функций, неочевидные случаи | Простые переменные с очевидным значением |

> 💡 Главное: annotation — явно, inference — автоматически. Результат защиты **одинаковый** в обоих случаях.

## Категории типов данных

Все типы в TypeScript делятся на две категории:

```
TypeScript Types
│
├── Primitive (примитивные) ← изучаем сегодня
│   │   Хранят ОДНО значение в переменной
│   ├── number      — числа (целые и дробные)
│   ├── string      — текст
│   ├── boolean     — true / false
│   ├── null        — намеренно пустое значение
│   ├── undefined   — значение не присвоено
│   ├── any         — любой тип (избегай!)
│   ├── union type  — несколько конкретных типов через |
│   └── void        — функция ничего не возвращает
│
└── Non-Primitive (непримитивные) ← следующие уроки
        Хранят МНОГО значений в одной переменной
    ├── Array       — список значений [1, 2, 3]
    ├── Tuple       — массив фиксированной структуры
    ├── Object      — объект с полями { name: "Alice" }
    ├── Function    — функция как тип
    ├── Class       — класс (ООП)
    └── Interface   — интерфейс (ООП)
```

**Главное отличие:**
- **Примитивные** — одно значение: `let age = 30`
- **Непримитивные** — много значений: `let ages = [25, 30, 35]`

## number — числа

### Что такое number?

В TypeScript **один числовой тип** — `number`. Он покрывает и целые числа, и дробные. Никаких `int`, `float`, `double` как в Java — только `number`.

```typescript
// Целые числа:
let age: number = 30;
let year: number = 2024;
let temperature: number = -15;
let zero: number = 0;

// Дробные числа — тот же тип number:
let price: number = 99.99;
let pi: number = 3.14159;
let weight: number = 73.5;

// Отрицательные:
let debt: number = -500;
let belowZero: number = -273.15;

// Большие числа — тоже number, никаких ограничений:
let population: number = 8100000000;
let bankBalance: number = 1234567.89;
```

### Математические операции

```typescript
let a: number = 10;
let b: number = 3;

console.log(a + b);   // 13    — сложение
console.log(a - b);   // 7     — вычитание
console.log(a * b);   // 30    — умножение
console.log(a / b);   // 3.333 — деление
console.log(a % b);   // 1     — остаток от деления (10 = 3×3 + 1)
console.log(a ** b);  // 1000  — возведение в степень (10³)

// Порядок операций — как в математике:
let result: number = (a + b) * 2;
console.log(result); // 26
```

### Как красиво выводить в консоль

```typescript
let age: number = 30;
let price: number = 99.99;
let bigNum: number = 1000000;

// Плохо — непонятно что за числа:
console.log(age, price, bigNum); // 30 99.99 1000000

// Хорошо — через запятую с текстовой подписью:
console.log("Возраст:", age);        // Возраст: 30
console.log("Цена:", price);         // Цена: 99.99
console.log("Большое число:", bigNum); // Большое число: 1000000

// Хорошо — через шаблонную строку:
console.log(`Возраст: ${age} лет, цена: ${price}€`);
// Возраст: 30 лет, цена: 99.99€
```

### Проверить тип через typeof

```typescript
let score: number = 95;
let pi: number = 3.14;

console.log(typeof score); // "number"
console.log(typeof pi);    // "number" — дробное тоже number!

// Полезно при отладке:
console.log("Тип переменной score:", typeof score); // Тип переменной score: number
```

### Практический пример

```typescript
// Расчёт скидки в магазине:
let originalPrice: number = 150;
let discountPercent: number = 20;
let discountAmount: number = (originalPrice * discountPercent) / 100;
let finalPrice: number = originalPrice - discountAmount;

console.log("Оригинальная цена:", originalPrice);  // 150
console.log("Скидка:", discountAmount);             // 30
console.log("Итоговая цена:", finalPrice);          // 120

// Или через шаблонные строки — красивее:
console.log(`Цена ${originalPrice}€ → скидка ${discountPercent}% → итого ${finalPrice}€`);
// Цена 150€ → скидка 20% → итого 120€
```

## string — строки

### Что такое string?

`string` — любой текст, набор символов. В TypeScript строку можно создать **тремя способами**, и все три дают тип `string`.

### Способ 1 и 2: двойные и одинарные кавычки

Эти два способа **полностью равнозначны** — используй тот, который удобнее. Они подходят для фиксированного текста без переменных внутри.

```typescript
// Двойные кавычки:
let firstName: string = "John";
let city: string = "Valencia";
let errorMsg: string = "File not found";

// Одинарные кавычки — то же самое:
let lastName: string = 'Kennedy';
let country: string = 'Spain';
let btnLabel: string = 'Submit';

// Оба варианта — тип string, разницы нет
console.log(firstName); // John
console.log(lastName);  // Kennedy
```

### Склеивание строк через `+` (конкатенация)

```typescript
let firstName: string = "John";
let lastName: string = "Kennedy";

// Склеиваем через +:
let fullName: string = firstName + " " + lastName;
console.log(fullName); // John Kennedy

// Можно склеивать сколько угодно:
let greeting: string = "Привет, " + firstName + " " + lastName + "!";
console.log(greeting); // Привет, John Kennedy!
```

> ⚠️ Когда переменных много — склеивание через `+` превращается в "лапшу" из плюсов. Именно для этого придумали способ 3.

### Способ 3: backtick — строки с переменными внутри

**Backtick** (обратная кавычка `` ` ``) — особый вид строки. Внутри через `${ }` можно вставить любую переменную или выражение. Это самый мощный и читаемый способ.

```typescript
let firstName: string = "John";
let lastName: string = "Kennedy";
let age: number = 35;
let city: string = "Valencia";

// Старый неудобный способ (через +):
let msg1 = "Привет, " + firstName + " " + lastName + "! Тебе " + age + " лет. Город: " + city;

// Новый удобный способ (backtick + ${}):
let msg2: string = `Привет, ${firstName} ${lastName}! Тебе ${age} лет. Город: ${city}`;

// Результат одинаковый, но второй в разы читабельнее:
console.log(msg2);
// Привет, John Kennedy! Тебе 35 лет. Город: Valencia
```

### Выражения внутри `${}`

В фигурных скобках можно не только подставлять переменные, но и писать **любой код**:

```typescript
let a: number = 10;
let b: number = 5;

// Математика прямо внутри строки:
console.log(`Сумма: ${a + b}`);           // Сумма: 15
console.log(`Разница: ${a - b}`);         // Разница: 5
console.log(`Произведение: ${a * b}`);    // Произведение: 50
console.log(`${a} > ${b}: ${a > b}`);    // 10 > 5: true

// Тернарный оператор внутри строки:
let score: number = 75;
console.log(`Статус: ${score >= 60 ? "Сдал" : "Не сдал"}`); // Статус: Сдал

// Вызов метода внутри строки:
let name: string = "alice";
console.log(`Имя: ${name.toUpperCase()}`); // Имя: ALICE
```

### Когда какой способ использовать

```typescript
// Фиксированный текст без переменных → кавычки:
let errorText: string = "Something went wrong";
let buttonText: string = 'Click me';

// Нужно вставить переменные → backtick:
let userName: string = "Boris";
let loginCount: number = 42;
let welcomeMsg: string = `Добро пожаловать, ${userName}! Вы вошли ${loginCount} раз.`;

// НЕ нужно использовать backtick без причины:
let bad: string = `Hello World`;  // работает, но backtick здесь лишний
let good: string = "Hello World"; // правильно
```

### Пример из автоматизации тестирования

```typescript
// В автоматизации часто нужны динамические URL и сообщения:
let baseUrl: string = "https://api.example.com";
let userId: number = 42;
let endpoint: string = "profile";

// Собираем URL из переменных:
let apiUrl: string = `${baseUrl}/users/${userId}/${endpoint}`;
console.log(apiUrl); // https://api.example.com/users/42/profile

// Сообщение в лог теста:
let browser: string = "Chrome";
let testName: string = "Login Test";
let version: number = 120;
let status: string = "PASSED";

let logEntry: string = `[${browser} v${version}] ${testName}: ${status}`;
console.log(logEntry); // [Chrome v120] Login Test: PASSED

// Сообщение об ошибке теста:
let element: string = "#submit-button";
let page: string = "Login Page";
let errMsg: string = `Element '${element}' not found on '${page}'`;
console.log(errMsg); // Element '#submit-button' not found on 'Login Page'
```

## boolean — булевы значения

### Что такое boolean?

`boolean` — тип с **только двумя возможными значениями**: `true` или `false`. Ничего другого в эту переменную не положить.

```typescript
let isStudent: boolean = true;
let hasJob: boolean = false;
let isLoggedIn: boolean = true;
let isAdmin: boolean = false;
let isPremiumUser: boolean = true;
```

### Откуда берётся boolean в реальном коде

На практике boolean значения чаще **получаются как результат сравнений**, а не задаются вручную:

```typescript
let age: number = 20;
let score: number = 85;
let balance: number = -50;

// Результаты сравнений — это boolean:
let isAdult: boolean = age >= 18;           // true  (20 >= 18)
let passed: boolean = score >= 60;          // true  (85 >= 60)
let isInDebt: boolean = balance < 0;        // true  (-50 < 0)
let isPerfect: boolean = score === 100;     // false (85 ≠ 100)

console.log("Совершеннолетний:", isAdult);  // true
console.log("Сдал:", passed);               // true
console.log("В долгу:", isInDebt);          // true
console.log("Идеальный результат:", isPerfect); // false
```

### Boolean в условиях (if/else)

Главное применение boolean — принятие решений:

```typescript
let isLoggedIn: boolean = true;
let isAdmin: boolean = false;
let hasPremium: boolean = true;

if (isLoggedIn) {
    console.log("Добро пожаловать!");
} else {
    console.log("Пожалуйста, войдите в систему");
}
// Добро пожаловать!

if (isAdmin) {
    console.log("Панель администратора");
} else {
    console.log("Обычная страница пользователя");
}
// Обычная страница пользователя

// Логические операторы:
let canAccessPremium: boolean = isLoggedIn && hasPremium; // true (И — оба true)
let needsLogin: boolean = !isLoggedIn;                    // false (НЕ — инверсия)
console.log(`Доступ к премиум: ${canAccessPremium}`);     // true
```

### Проверка типа и типичные ошибки

```typescript
let isActive: boolean = true;
console.log(typeof isActive); // "boolean"

// ❌ Нельзя положить строку в boolean — даже "true":
// let flag: boolean = "true";  // ОШИБКА! Строка не boolean

// ❌ Нельзя положить число:
// let flag: boolean = 1;  // ОШИБКА! В отличие от JS, в TS это не работает

// ✅ Только настоящие true/false:
let flag: boolean = true;
let done: boolean = false;
```

## null и undefined — особые типы

### Что такое undefined?

`undefined` появляется когда переменная объявлена, но **значение ей ещё не присвоено**. TypeScript возвращает его автоматически.

```typescript
let price: number; // объявили, но не присвоили

// TypeScript даст предупреждение:
// Variable 'price' is used before being assigned.

console.log(price); // undefined
// Переменная существует, но значения нет
```

### Что такое null?

`null` — это **намеренное, осознанное отсутствие значения**. Разница принципиальная:
- `undefined` — значение **ещё не присвоено** (случайно, временно нет)
- `null` — значение **намеренно установлено как "пусто"** (ты сам так решил)

```typescript
// Только null можно положить в null-переменную:
let emptyValue: null = null; // ✅

// Попытка положить что-то другое:
// let emptyValue: null = 10; // ❌ ОШИБКА

// Только undefined в undefined-переменную:
let notAssigned: undefined = undefined; // ✅
```

### Разница между null и undefined

```typescript
let a: undefined = undefined;
let b: null = null;

console.log(a); // undefined
console.log(b); // null

// typeof возвращает разные результаты:
console.log(typeof a); // "undefined"
console.log(typeof b); // "object" — историческая особенность JS, просто запомни

// Сравнение:
console.log(a == b);  // true  — нестрогое равенство считает их похожими
console.log(a === b); // false — строгое: они разные типы!
```

### На практике — почти всегда в union

Чистые `null` и `undefined` типы используются редко сами по себе. Чаще они встречаются в **union types** когда переменная может быть либо значением, либо пустой:

```typescript
// Пользователь либо залогинен, либо нет:
let currentUser: string | null = null;  // не залогинен
currentUser = "Alice";                   // залогинился

// Результат поиска — либо нашли, либо null:
let foundElement: string | null = null; // ещё не нашли
foundElement = "#submit-button";        // нашли!

// Функция возвращает результат или null:
function findUser(id: number): string | null {
    if (id === 1) return "Alice";
    if (id === 2) return "Bob";
    return null; // не найден
}

console.log(findUser(1));  // "Alice"
console.log(findUser(99)); // null
```

> 💡 **Вывод:** В реальном коде ты будешь встречать `null` и `undefined` не как самостоятельные типы, а как часть union: `string | null`, `number | undefined`. Просто помни разницу — `null` намеренное, `undefined` случайное.

## any — любой тип

### Что такое any?

`any` — это "выключатель TypeScript" для конкретной переменной. Когда переменная имеет тип `any` — в неё можно записать значение **любого типа** без ошибок. TypeScript перестаёт её проверять.

```typescript
let value: any = "строка";
console.log(typeof value); // "string"

value = 42;                // ✅ никаких ошибок
console.log(typeof value); // "number"

value = true;              // ✅ никаких ошибок
console.log(typeof value); // "boolean"

value = [1, 2, 3];        // ✅ никаких ошибок
console.log(typeof value); // "object"
```

### Почему any — опасен?

`any` отключает **обе** ключевые фишки TypeScript одновременно: статическую типизацию и type safety.

```typescript
// С any TypeScript не защищает тебя:
let data: any = "5";
let result = data * 2;      // TypeScript молчит!
console.log(result);         // NaN — потому что строка * 2 = NaN

// С правильными типами TypeScript остановит тебя сразу:
let safeData: string = "5";
// let safeResult = safeData * 2; // ❌ ОШИБКА — TypeScript нас защищает
```

Если кто-то на собеседовании спросит: "TypeScript — статически типизированный и type-safe, можно ли сломать эти правила?" — ответ: **да, через `any`**. Именно поэтому `any` не рекомендуется.

### Сравнение: что теряется при any

```typescript
// Без any — TypeScript полностью защищает:
let age: number = 30;
// age = "старый"; // ❌ ОШИБКА — защита работает!

// С any — TypeScript отключает защиту:
let anyAge: any = 30;
anyAge = "старый"; // ✅ TypeScript молчит
anyAge = true;     // ✅ TypeScript молчит
anyAge = [];       // ✅ TypeScript молчит
```

### Когда всё же используют any

```typescript
// 1. Данные из внешнего API неизвестной структуры:
let apiResponse: any;
// позже разберёшься что пришло и поставишь нужный тип

// 2. Временно при переходе с JavaScript на TypeScript:
let legacyData: any = oldJavaScriptFunction();

// 3. Когда тип реально неизвестен заранее
// — но в большинстве таких случаев лучше union type!
```

> ⚠️ **Правило:** Никогда не используй `any` если есть альтернатива. Хочешь несколько типов — используй **union type**. `any` — крайний случай.

## Union Type — объединение типов

### Что такое Union Type?

**Union Type** — способ сказать TypeScript: "эта переменная может хранить значения **нескольких конкретных типов**". Используется символ `|` (вертикальная черта — читается "или").

```typescript
// Синтаксис: тип1 | тип2 | тип3
let id: number | string;

id = 123;    // ✅ number — разрешено
id = "ABC";  // ✅ string — разрешено
id = true;   // ❌ ОШИБКА — boolean не в списке
id = null;   // ❌ ОШИБКА — null не в списке
```

### Чем Union лучше any — наглядно

```typescript
// С any — принимает ВСЁЁ:
let badId: any;
badId = 123;      // ok
badId = "ABC";    // ok
badId = true;     // ok — но мы не хотели boolean!
badId = {};       // ok — но мы не хотели объект!
// Никакой защиты

// С Union — принимает только ТО ЧТО МЫ РАЗРЕШИЛИ:
let goodId: number | string;
goodId = 123;     // ✅
goodId = "ABC";   // ✅
goodId = true;    // ❌ ОШИБКА — TypeScript нас защищает!
goodId = {};      // ❌ ОШИБКА — TypeScript нас защищает!
// Точный контроль — только number или string
```

### Практические примеры

```typescript
// ID пользователя — может быть числом или строкой:
let userId: number | string = 42;
userId = "user-abc-123"; // тоже ок

// Выбранный результат — может быть числом или "нет выбора":
let selectedScore: number | null = null; // ещё не выбрано
selectedScore = 95;                      // выбрали

// Результат операции — число или сообщение об ошибке:
let operationResult: number | string = 0;
operationResult = "Division by zero!"; // что-то пошло не так

// Статус — в разных форматах:
let responseStatus: number | string | boolean;
responseStatus = 200;       // HTTP-код числом
responseStatus = "success"; // или строкой
responseStatus = true;      // или булевым флагом
```

### Union с null — самый частый паттерн в реальном коде

```typescript
// Переменная либо содержит значение, либо null:
let currentUser: string | null = null;  // нет пользователя
currentUser = "Alice";                   // пользователь вошёл

let pageTitle: string | null = null;     // элемент не найден
pageTitle = "Главная страница";          // элемент найден

// Три типа в union:
let flexibleData: string | number | null = null;
flexibleData = "текст"; // ✅
flexibleData = 42;      // ✅
flexibleData = null;    // ✅
// flexibleData = true; // ❌ ОШИБКА — boolean не разрешён
```

## void — тип возврата функций

### Что такое void?

`void` используется как **тип возврата функции** когда функция **ничего не возвращает** — она выполняет действие (например, выводит в консоль), но не передаёт результат обратно.

> 💡 `void` почти никогда не используется для переменных — только для функций.

### Функция без возвращаемого значения — void

```typescript
// Просто выводит в консоль, ничего не возвращает:
function greet(): void {
    console.log("Hello, World!");
}

greet(); // Hello, World!

// Функция с параметрами, но без возврата:
function printUserInfo(name: string, age: number): void {
    console.log(`Имя: ${name}`);
    console.log(`Возраст: ${age}`);
}

printUserInfo("Alice", 30);
// Имя: Alice
// Возраст: 30
```

### Функция С возвращаемым значением — не void

```typescript
// Возвращает number:
function sum(x: number, y: number): number {
    return x + y;
}
let total: number = sum(10, 20);
console.log(`Сумма: ${total}`); // Сумма: 30

// Возвращает string:
function createGreeting(name: string): string {
    return `Hello, ${name}!`;
}
let msg: string = createGreeting("Boris");
console.log(msg); // Hello, Boris!

// Возвращает boolean:
function isAdult(age: number): boolean {
    return age >= 18;
}
let canVote: boolean = isAdult(20);
console.log(`Может голосовать: ${canVote}`); // true

// Возвращает number | null (union):
function findScore(studentId: number): number | null {
    if (studentId === 1) return 95;
    if (studentId === 2) return 78;
    return null; // не найден
}
let result1 = findScore(1);  // 95
let result2 = findScore(99); // null
```

### void в TypeScript — необязателен, но полезен

```typescript
// В TypeScript void необязателен (в отличие от Java, где он обязателен):

// Без void — работает:
function doSomething() {
    console.log("working");
}

// С void — явно видно что функция ничего не возвращает:
function doSomethingVoid(): void {
    console.log("working");
}

// Оба варианта правильны — второй более читаем
```

### Параметры функций — тип обязателен!

```typescript
// Параметры функции всегда требуют аннотацию типа:
function add(x: number, y: number): number {
    return x + y;
}

// Вызов с правильными типами:
console.log(add(10, 20)); // 30
console.log(add(5, 3));   // 8

// TypeScript защитит от неправильных аргументов:
// add("hello", 5); // ❌ ОШИБКА — строка не number
// add(10);         // ❌ ОШИБКА — нужны два аргумента
```

## Итоговая шпаргалка

### Все примитивные типы одним взглядом

```typescript
// number — целые и дробные числа:
let age: number = 30;
let price: number = 99.99;

// string — текст тремя способами:
let name1: string = "Alice";           // двойные кавычки
let name2: string = 'Bob';             // одинарные кавычки
let msg: string = `Hello, ${name1}!`;  // backtick + переменные

// boolean — только true или false:
let isActive: boolean = true;
let isDeleted: boolean = false;

// null — намеренно пустое значение:
let emptyField: null = null;

// undefined — значение не присвоено:
let notSet: undefined = undefined;

// any — любой тип (избегай!):
let anything: any = "строка";
anything = 42;    // TypeScript не возражает
anything = true;  // TypeScript не возражает

// union type — несколько конкретных типов:
let id: number | string = 101;
id = "ID-101";  // ✅

let nullable: string | null = null;
nullable = "теперь есть значение"; // ✅

// void — функция ничего не возвращает:
function logMsg(msg: string): void {
    console.log(msg);
}
```

### Вопросы с собеседований

**Q: Чем TypeScript отличается от JavaScript?**
A: TypeScript — статически типизированный (тип определяется один раз) и type-safe (переменная хранит только свой тип). JavaScript — динамически типизированный и не type-safe.

**Q: Что такое Type Annotation?**
A: Явное указание типа через двоеточие: `let age: number = 30`. Программист сам говорит TypeScript какой тип у переменной.

**Q: Что такое Type Inference?**
A: Автоматическое определение типа TypeScript'ом по значению: `let age = 30` — TypeScript сам понимает что это `number`. Защита работает так же как при annotation.

**Q: Чем Union Type лучше any?**
A: Union ограничивает набор допустимых типов и сохраняет Type Safety. `any` снимает все ограничения и отключает защиту TypeScript.

**Q: Можно ли нарушить type safety в TypeScript?**
A: Да — через тип `any`. Именно поэтому его не рекомендуют использовать.

**Q: В чём разница между null и undefined?**
A: `undefined` — значение ещё не присвоено (случайно нет). `null` — намеренное отсутствие значения (программист так решил).
