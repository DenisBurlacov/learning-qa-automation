## Комментарии

### Зачем нужны комментарии?

Комментарий — это текст в коде, который **не выполняется**. JavaScript и TypeScript его просто игнорируют. Комментарии нужны для двух вещей:

1. **Улучшить читаемость кода** — другой человек (или ты сам через месяц) прочитает комментарий и поймёт, зачем написан этот код
2. **Временно отключить код** — не удалять строку, а закомментировать, чтобы она не выполнялась

### Однострочный комментарий (`//`)

Два слеша — и всё что после них на этой строке игнорируется:

```typescript
// Это комментарий — TypeScript его не выполнит
console.log("Hello"); // Можно писать комментарий после кода

// console.log("Эта строка не выполнится");
console.log("А эта выполнится");
```

### Многострочный комментарий (`/* */`)

Когда нужно закомментировать сразу несколько строк:

```typescript
/*
  Всё что внутри этого блока
  не будет выполнено.
  Можно писать сколько угодно строк.
*/
console.log("Эта строка выполнится");
```

### Горячие клавиши в VS Code

|Действие|Windows|Mac|
|---|---|---|
|Однострочный комментарий|`Ctrl + /`|`Cmd + /`|
|Многострочный комментарий|`Shift + Alt + A`|`Shift + Option + A`|

> 💡 Один и тот же шоткат **добавляет** и **убирает** комментарий — не нужно запоминать два разных.

## Что такое переменная?

**Переменная** — это контейнер, который может хранить какое-то значение (данные).

Представь переменную как **именованную коробку**: у коробки есть имя, и внутри неё лежит какое-то значение. В любой момент можно заглянуть в коробку и посмотреть что там, или положить туда что-то новое.

```
┌─────────────┐     ┌───────────────┐
│  age = 30   │     │ name = "John" │
└─────────────┘     └───────────────┘
  переменная           переменная
  "age"                "name"
  значение: 30         значение: "John"
```

```typescript
let age = 30;           // переменная age хранит число 30
let studentName = "John"; // переменная studentName хранит строку "John"
```

### Как называть переменные?

В начале обучения можно называть `x`, `y`, `z` — это нормально для упражнений. Но в реальных проектах имя переменной должно **отражать её смысл** — что именно в ней хранится:

```typescript
// ❌ Плохо — непонятно что за x
let x = 25;

// ✅ Хорошо — сразу ясно что хранится
let userAge = 25;
let productPrice = 99.90;
let isLoggedIn = true;
```

> 💡 В TypeScript и JavaScript принято писать имена переменных в **camelCase** (первое слово строчными, каждое следующее с заглавной): `firstName`, `productPrice`, `isUserLoggedIn`.

## Синтаксис объявления переменной

В TypeScript и JavaScript переменные создаются тремя ключевыми словами: `var`, `let`, `const`.

### Полный синтаксис (с указанием типа данных):

```
ключевое_слово  имя_переменной : тип_данных = значение
```

```typescript
var   age    : number  = 30;
let   name   : string  = "John";
const PI     : number  = 3.14;
```

### Краткий синтаксис (тип данных необязателен):

Указывать тип данных в TypeScript **необязательно** — TypeScript сам определит тип по значению. Это называется **type inference** (вывод типа).

```typescript
// Краткая запись — TypeScript сам поймёт что age это number
var   age   = 30;
let   name  = "John";
const PI    = 3.14;
```

> ⚠️ Хотя тип данных необязателен, TypeScript всё равно **знает** какого типа переменная. Если написать `let age = 30`, а потом попробовать присвоить `age = "hello"` — TypeScript выдаст ошибку, потому что переменной-числу нельзя присвоить строку.

## var vs let vs const — 5 отличий

Это самая важная часть урока. Все три ключевых слова создают переменные, но ведут себя по-разному в пяти аспектах:

|Аспект|Что это значит|
|---|---|
|**Scope**|Где переменная доступна (видна)|
|**Declaration**|Можно ли объявить без значения|
|**Re-declaration**|Можно ли объявить ту же переменную ещё раз|
|**Re-initialization**|Можно ли изменить значение переменной|
|**Hoisting**|Что будет если использовать переменную до её объявления|

---

## Отличие 1: Scope (область видимости)

**Scope** — это "зона", в которой переменная существует и доступна. Вышел из зоны — переменная больше не видна.

В TypeScript/JavaScript есть два вида scope:

- **Function scope** (функциональная область) — переменная видна **во всей функции**, независимо от того, в каком блоке внутри функции она создана
- **Block scope** (блочная область) — переменная видна **только внутри блока** `{ }`, где она создана

> 💡 **Блок** — это всё что находится между фигурными скобками `{ }`. Например, тело `if`, тело `for`, тело `while` — всё это отдельные блоки.

```
var  → Function scope  (видна во всей функции)
let  → Block scope     (видна только в своём блоке)
const → Block scope    (видна только в своём блоке)
```

### Пример: var — функциональная область

```typescript
function varScopeDemo() {
    if (true) {
        var msg = "Hello World"; // создали ВНУТРИ блока if
    }
    // Мы ВЫШЛИ из блока if, но var всё ещё доступна!
    console.log(msg); // ✅ "Hello World" — работает, потому что var = function scope
}

varScopeDemo();
```

`var` создана внутри блока `if`, но доступна **везде внутри функции** — потому что её scope — вся функция.

### Пример: let/const — блочная область

```typescript
function blockScopeDemo() {
    if (true) {
        let msg = "Hello World";   // создали ВНУТРИ блока if
        const greet = "Hi";        // тоже ВНУТРИ блока if
        console.log(msg);   // ✅ работает — мы внутри блока
        console.log(greet); // ✅ работает — мы внутри блока
    }
    // Мы ВЫШЛИ из блока if
    console.log(msg);   // ❌ ОШИБКА — msg не существует здесь
    console.log(greet); // ❌ ОШИБКА — greet не существует здесь
}
```

`let` и `const` созданы внутри блока `if` — и **умирают**, как только блок заканчивается. Вне блока они недоступны.

### Пример: все три в одном месте

```typescript
function scopeDiff() {
    if (true) {
        var num1 = 10;   // function scope
        let num2 = 20;   // block scope
        const num3 = 30; // block scope
        
        // Внутри блока — доступны все три ✅
        console.log(num1); // 10
        console.log(num2); // 20
        console.log(num3); // 30
    }
    
    // Вне блока:
    console.log(num1); // ✅ 10  — var доступна (function scope)
    console.log(num2); // ❌ ОШИБКА — let недоступна (block scope)
    console.log(num3); // ❌ ОШИБКА — const недоступна (block scope)
}
```

> ⚠️ Именно поэтому `var` не рекомендуется — она "утекает" из блоков и может давать **неожиданное поведение** в условиях и циклах. Код делает не то, что ожидаешь.

## Отличие 2: Declaration (объявление)

**Declaration** — создание переменной. **Initialization** — присвоение ей значения.

```typescript
let x;      // Declaration (объявление) — переменная есть, но значения нет
x = 10;     // Initialization (инициализация) — присваиваем значение

let y = 20; // Declaration + Initialization одновременно
```

### var — можно объявить без значения

```typescript
var x;
console.log(x); // undefined (переменная есть, но значения нет)

x = 30;
console.log(x); // 30
```

`undefined` — это специальное значение TypeScript, которое означает "переменная объявлена, но значение ещё не присвоено". Не путай с ошибкой — это нормальное поведение.

### let — тоже можно объявить без значения

```typescript
let y;
console.log(y); // undefined

y = 50;
console.log(y); // 50
```

Поведение такое же как у `var` — объявить без значения можно.

### const — НЕЛЬЗЯ объявить без значения

```typescript
const z; // ❌ ОШИБКА! Compile time error!
// SyntaxError: Missing initializer in const declaration
```

Почему? Потому что `const` означает **константу** — значение которое **никогда не изменится**. Если мы её создаём — мы обязаны сразу сказать какое у неё значение. Константа без значения не имеет смысла.

```typescript
const z = 50; // ✅ Правильно — создаём с значением
console.log(z); // 50
```

## Отличие 3: Re-declaration (повторное объявление)

**Re-declaration** — это когда ты объявляешь переменную с тем же именем **второй раз** в том же коде.

### var — повторное объявление РАЗРЕШЕНО

```typescript
var city = "New York";
var city = "Los Angeles"; // ✅ Не ошибка — var позволяет это!

console.log(city); // "Los Angeles" — значение заменилось
```

Это **опасно**! Представь: ты написал 500 строк кода, объявил переменную `city = "New York"`. Через 200 строк случайно написал `var city = "Los Angeles"` — и старое значение молча перезаписалось. Это сложно найти при отладке.

### let — повторное объявление ЗАПРЕЩЕНО

```typescript
let country = "India";
let country = "USA"; // ❌ ОШИБКА!
// Error: Cannot redeclare block-scoped variable 'country'
```

TypeScript сразу скажет тебе: "Эта переменная уже существует!" — это **безопаснее**.

### const — повторное объявление ЗАПРЕЩЕНО

```typescript
const planet = "Earth";
const planet = "Mars"; // ❌ ОШИБКА!
```

То же что и `let` — нельзя объявить повторно.

> 💡 Именно из-за разрешённого re-declaration `var` нарушает **type safety** — главную идею TypeScript. TypeScript создан чтобы код был предсказуемым и безопасным, а `var` позволяет случайно перезаписать переменную без предупреждения.

## Отличие 4: Re-initialization (переприсвоение значения)

**Re-initialization** (или **re-assignment**) — это изменение значения уже существующей переменной. Обрати внимание: это НЕ то же самое что re-declaration! Мы не пишем `let` снова — мы просто присваиваем новое значение.

```typescript
// Re-declaration — пишем ключевое слово снова (ПЛОХО):
let age = 25;
let age = 30; // ❌ Ошибка

// Re-initialization — просто меняем значение (НОРМАЛЬНО):
let age = 25;
age = 30;     // ✅ Просто меняем значение, без ключевого слова
```

### var — изменение значения РАЗРЕШЕНО

```typescript
var age = 25;
console.log(age); // 25

age = 30; // ✅ Меняем значение
console.log(age); // 30
```

### let — изменение значения РАЗРЕШЕНО

```typescript
let score = 100;
console.log(score); // 100

score = 200; // ✅ Меняем значение
console.log(score); // 200
```

Именно поэтому `let` используется когда значение будет **меняться в процессе работы программы**: счётчик, текущий пользователь, статус и т.д.

### const — изменение значения ЗАПРЕЩЕНО

```typescript
const PI = 3.14159;
PI = 3; // ❌ ОШИБКА!
// Error: Cannot assign to 'PI' because it is a constant
```

`const` — это **неизменяемое значение**. Объявил один раз — навсегда. Используй для значений которые не должны меняться: математические константы, конфигурационные значения, URL-адреса и т.д.

```typescript
// Хорошие примеры для const:
const MAX_RETRIES = 3;
const API_URL = "https://api.example.com";
const TAX_RATE = 0.21;
```

## Отличие 5: Hoisting (поднятие)

**Hoisting** — это поведение переменной когда ты пытаешься её использовать **до того как объявил**.

В нормальном коде мы всегда сначала объявляем переменную, потом используем. Но что если сделать наоборот? Вот тут и видна разница между `var`, `let` и `const`.

> 💡 На практике **hoisting не нужен** — никогда не стоит использовать переменную до её объявления. Это плохой стиль кода. Но знать как это работает важно для понимания различий между var/let/const.

### var — hoisting возвращает `undefined`

```typescript
console.log(a); // undefined (не ошибка!)
var a = 10;
console.log(a); // 10
```

Как так? JavaScript "поднимает" (`hoists`) объявление `var a` наверх области видимости, но **без значения**. Поэтому первый `console.log` видит что переменная `a` существует, но значения у неё ещё нет → `undefined`.

Это **скрытая опасность** `var` — код не падает с ошибкой, а тихо возвращает `undefined`. Ты можешь долго не замечать проблему.

### let — hoisting возвращает ОШИБКУ

```typescript
console.log(b); // ❌ ОШИБКА!
// ReferenceError: Cannot access 'b' before initialization
let b = 20;
console.log(b); // 20
```

`let` тоже "поднимается", но попадает в так называемую **"temporal dead zone"** (временную мёртвую зону) — переменная существует, но недоступна до строки объявления. Попытка обратиться → **явная ошибка**. Это лучше, чем тихое `undefined`!

### const — то же что и let

```typescript
console.log(c); // ❌ ОШИБКА!
// ReferenceError: Cannot access 'c' before initialization
const c = 30;
```

Точно такое же поведение как у `let` — явная ошибка. Это правильно и безопасно.

### Итог по hoisting

```
var   → undefined (опасно — ошибка скрыта)
let   → ReferenceError (безопасно — ошибка видна сразу)
const → ReferenceError (безопасно — ошибка видна сразу)
```

|Аспект|`var`|`let`|`const`|
|---|---|---|---|
|**Scope**|Function scope (вся функция)|Block scope (только блок)|Block scope (только блок)|
|**Объявление без значения**|✅ Можно|✅ Можно|❌ Нельзя (нужно значение)|
|**Re-declaration**|✅ Разрешено|❌ Запрещено|❌ Запрещено|
|**Re-initialization**|✅ Разрешено|✅ Разрешено|❌ Запрещено|
|**Hoisting**|`undefined`|Ошибка (not initialized)|Ошибка (not initialized)|
|**Рекомендуется?**|❌ Избегать|✅ Да|✅ Да|

## Главное правило

Запомни три простых правила для реального кода:

**`var` — не используем.** Из-за function scope и разрешённого re-declaration поведение непредсказуемо. В современном TypeScript и JavaScript `var` устарел.

**`let` — используем когда значение будет меняться:**

```typescript
let counter = 0;
counter = counter + 1; // значение меняется — это нормально для let

let currentUser = "Alice";
currentUser = "Bob"; // переключили пользователя — тоже нормально
```

**`const` — используем когда значение НЕ должно меняться:**

```typescript
const PI = 3.14159;
const MAX_USERS = 100;
const BASE_URL = "https://api.example.com";
const TAX = 0.21;
```

> 💡 **Практический совет:** Начинай всегда с `const`. Если в процессе написания кода понял что значение нужно менять — переключись на `let`. Так ты автоматически используешь `const` там где возможно, что делает код безопаснее и понятнее.
