## Что такое строка

Строка (string) — это тип данных, который хранит текст. Любой набор символов в кавычках — это строка.

Главный признак строки — кавычки. Внутри может быть:

- обычный текст: `"hello"`
- цифры: `"123"` — но это строка, не число!
- специальные символы: `"passed!"`, `"https://api.example.com"`
- пустота: `""` — пустая строка, но всё равно строка

**Важно для тестирования:** статусы тестов — это строки: `"passed"`, `"failed"`, `"skipped"`. URL тоже строка. Имя браузера — строка.

```typescript
let status: string = "passed";
let url: string = "https://api.example.com/users";
let browser: string = "Chrome";
let env: string = "staging";
```

Строка в TypeScript имеет тип `string`. Благодаря выводу типов (type inference), тип можно не писать явно — TypeScript сам поймёт. Но явное указание типа считается хорошей практикой.

## Три способа объявить строку

В JavaScript и TypeScript можно создать строку тремя способами:

```typescript
let str1: string = 'одиночные кавычки';
let str2: string = "двойные кавычки";
let str3: string = `обратные кавычки (backtick)`;
```

Все три выведут одинаковый результат. Разница — в назначении:

- Одиночные и двойные кавычки — для обычных строк (90% случаев)
- Обратные кавычки (backtick) — для шаблонных строк с переменными

**Из Java/Python:** в Java только двойные кавычки. В Python одиночные и двойные. В JS/TS все три варианта.

## Экранирование символов

Иногда внутри строки нужно использовать символ, который сам по себе "работает" как часть синтаксиса. Например, кавычка внутри строки в кавычках.

Решение — поставить `\` (обратный слеш, backslash) перед таким символом. Это называется экранирование.

```typescript
// Кавычка внутри строки
let phrase: string = "Тест \"login\" прошёл";

// Одиночная кавычка
let note: string = 'Can\'t connect to server';

// Сам обратный слеш
let path: string = "C:\\tests\\results";
```

**Важно:** знак `\` при подсчёте длины строки не учитывается — он только "экранирует" следующий символ.

---

## Специальные символы в строках

Кроме экранирования символов, `\` используется для особых управляющих символов:

```typescript
// Перенос строки (новая строка)
let report: string = "Test: login\nStatus: passed\nTime: 320ms";

// Возврат каретки (устаревший, но встречается)
let old: string = "line1\rline2";

// Табуляция
let table: string = "Browser\tStatus\tTime";
```

**Разница `\n` и `\r`:**

- `\n` — переходит на следующую строку (используй это)
- `\r` — ставит курсор на начало той же строки (устаревшее поведение Windows)
- `\r\n` — комбинация для Windows-совместимости
- `\t` — отступ (tab)

**В Playwright:** `\n` полезен для логирования многострочных отчётов о тестах.

## Шаблонные строки (Template Literals)

Обратные кавычки `` ` `` открывают мощный способ работы со строками — шаблонные строки (template literals / template strings).

**Проблема с обычными кавычками:**

```typescript
let testName: string = "login test";
let status: string = "passed";

// Так не работает — переменная не подставляется, просто текст:
console.log("Test: testName is status");  // "Test: testName is status"
```

**Решение — обратные кавычки + `${}`:**

```typescript
let testName: string = "login test";
let status: string = "passed";

// Так работает:
console.log(`Test: ${testName} is ${status}`);
// Выведет: "Test: login test is passed"
```

**Внутри `${}` можно писать любой код:**

```typescript
let responseTime: number = 320;
let threshold: number = 500;

// Выражение прямо в строке:
console.log(`Response time: ${responseTime}ms (limit: ${threshold}ms)`);
console.log(`Within limit: ${responseTime < threshold}`);

// Математика:
console.log(`Tests: ${50 + 12} total`);

// Вызов метода:
console.log(`URL: ${url.toUpperCase()}`);
```

**Многострочные строки — только с backtick:**

```typescript
// С обычными кавычками — ошибка!
// let report = "line1
// line2";  // SyntaxError

// С backtick — работает:
let report: string = `Test Report
Browser: Chrome
Status: passed
Time: 320ms`;
```

**Когда использовать что:**

- Обычный текст без переменных — одиночные или двойные кавычки
- Нужно вставить переменную — backtick с `${}`
- Многострочный текст — backtick

## Копирование и конкатенация строк

**Копирование** строки — просто присвоение:

```typescript
let original: string = "staging";
let copy: string = original;  // копируем
console.log(copy);  // "staging"
```

**Конкатенация через `+`:**

```typescript
let baseUrl: string = "https://api.example.com";
let endpoint: string = "/users";
let fullUrl: string = baseUrl + endpoint;
// "https://api.example.com/users"
```

**Конкатенация через `.concat()`:**

```typescript
let str1: string = "Welcome";
let str2: string = " to TypeScript";
let str3: string = "!";

let result: string = str1.concat(str2).concat(str3);
// "Welcome to TypeScript!"
```

**Почему `+` не рекомендуют для строк:**

```typescript
// Легко запутаться — `+` для чисел это математика, для строк — склейка:
let a = 5;
let b = "10";
console.log(a + b);  // "510" — неожиданно!

// Лучше использовать шаблонные строки:
let env: string = "staging";
let baseUrl: string = `https://api.example.com/${env}/users`;
```

## Иммутабельность строк

**Иммутабельность (immutability)** — это когда значение нельзя изменить после создания.

Строки в JavaScript/TypeScript иммутабельны. Это значит: **ни один метод строки не меняет саму строку** — он всегда возвращает новую строку.

```typescript
let url: string = "https://API.EXAMPLE.COM/users";

// Метод возвращает новую строку:
let lower: string = url.toLowerCase();

console.log(url);    // "https://API.EXAMPLE.COM/users" — не изменилась!
console.log(lower);  // "https://api.example.com/users" — новая строка
```

**Аналогия:** Представь строку как надпись на камне. Ты можешь сфотографировать её и написать что-то новое на другом камне, но оригинал не меняется.

**Почему это важно в тестировании:** методы строк безопасны — они не ломают оригинальные данные. Можно передавать строки в разные функции без риска потерять исходное значение.

## Методы строк

### length — длина строки

`length` — это не метод, а **свойство** (атрибут). Скобки не нужны!

```typescript
let str: string = "Hello, TypeScript!";
console.log(str.length);  // 18

// Пример в тестировании:
let selector: string = "#login-button";
console.log(selector.length);  // 13

// Подсчёт учитывает ВСЕ символы: пробелы, запятые, точки
let status: string = "test passed";
console.log(status.length);  // 11 (включая пробел)
```

### toUpperCase() и toLowerCase() — изменение регистра

```typescript
let env: string = "Production";

console.log(env.toUpperCase());  // "PRODUCTION"
console.log(env.toLowerCase());  // "production"
console.log(env);  // "Production" — оригинал не изменился!
```

**Зачем:** сравнение без учёта регистра — распространённая задача в тестах:

```typescript
let actualStatus: string = "PASSED";
let expected: string = "passed";

// Небезопасное сравнение:
console.log(actualStatus === expected);  // false!

// Безопасное — приводим к одному регистру:
console.log(actualStatus.toLowerCase() === expected);  // true
```

### charAt() и indexOf() — символ по индексу и индекс по символу

**Строка — это как массив символов.** У каждого символа есть индекс, начиная с нуля:

```
H  e  l  l  o
0  1  2  3  4
```

**`charAt(index)`** — возвращает символ по индексу:

```typescript
let str: string = "Hello, TypeScript!";

console.log(str.charAt(0));   // "H"
console.log(str.charAt(4));   // "o"
console.log(str[0]);          // "H" — через квадратные скобки тоже работает
console.log(str[str.length - 1]);  // "!" — последний символ
```

**`indexOf(substring)`** — возвращает индекс первого вхождения подстроки:

```typescript
let str: string = "Hello, TypeScript!";

console.log(str.indexOf("TypeScript"));  // 7
console.log(str.indexOf("xyz"));         // -1 (не найдено)
console.log(str.indexOf("l"));           // 2 (первое вхождение)
```

**`lastIndexOf(substring)`** — возвращает индекс последнего вхождения:

```typescript
let str: string = "Hello World Hello";
console.log(str.lastIndexOf("Hello"));  // 12 (последнее вхождение)
```

**В тестировании:**

```typescript
let url: string = "https://api.example.com/users/123";
let paramStart: number = url.indexOf("/users");
console.log(paramStart);  // 22
```

### substring() — извлечение подстроки

`substring(start, end)` — возвращает часть строки от `start` до `end` (не включая `end`).

```
W  e  l  c  o  m  e
0  1  2  3  4  5  6

substring(2, 4) → "lc"
```

**Логика подсчёта:**

- Начальный индекс `start` — считаем с нуля
- Конечный индекс `end` — **не включается** (exclusive)
- Если `end` не указан — до конца строки

```typescript
let url: string = "https://api.example.com/users";

// Извлечь протокол:
console.log(url.substring(0, 5));    // "https"

// Извлечь путь:
console.log(url.substring(23));      // "/users" (до конца)

// Имя браузера из строки лога:
let logEntry: string = "[Chrome] Login test passed";
console.log(logEntry.substring(1, 7));  // "Chrome"
```

**`slice(start, end)`** — похож на `substring`, но принимает отрицательные числа:

```typescript
let str: string = "Hello, TypeScript!";

console.log(str.slice(1, 4));    // "ell"
console.log(str.slice(-1));      // "!" (последний символ)
console.log(str.slice(-10));     // "TypeScript!" (последние 10 символов)
```

**Разница substring vs slice:**

- `substring` — не принимает отрицательные числа
- `slice` — принимает отрицательные (отсчёт с конца)

### includes() — проверка наличия подстроки

Возвращает `true` / `false`. Чувствителен к регистру!

```typescript
let str: string = "Hello, TypeScript!";

console.log(str.includes("TypeScript"));  // true
console.log(str.includes("typescript"));  // false — регистр важен!
console.log(str.includes("!"));           // true
console.log(str.includes("Java"));        // false
```

**В тестировании — очень часто:**

```typescript
let pageTitle: string = "Welcome to Dashboard - Staging";

// Проверка что мы на нужной странице:
if (pageTitle.includes("Dashboard")) {
  console.log("Test passed: correct page");
}

// Проверка окружения:
let url: string = "https://staging.example.com/login";
console.log(url.includes("staging"));  // true
console.log(url.includes("production"));  // false
```

### startsWith() и endsWith() — начало и конец строки

Оба возвращают `true` / `false`. Оба чувствительны к регистру.

```typescript
let url: string = "https://api.example.com/users";

console.log(url.startsWith("https"));   // true
console.log(url.startsWith("http://"));  // false
console.log(url.endsWith("/users"));    // true
console.log(url.endsWith("products"));  // false
```

**В тестировании:**

```typescript
let selector: string = "#login-button";
let classSelector: string = ".modal-overlay";

console.log(selector.startsWith("#"));      // true — ID-селектор
console.log(classSelector.startsWith("."));  // true — class-селектор

let status: string = "Error: timeout exceeded";
console.log(status.startsWith("Error"));    // true — это ошибка
```

### replace() и replaceAll() — замена подстроки

`replace(what, withWhat)` — заменяет **первое** вхождение. `replaceAll(what, withWhat)` — заменяет **все** вхождения.

```typescript
let url: string = "https://staging.example.com/staging/users";

// Замена первого вхождения:
let newUrl1: string = url.replace("staging", "production");
console.log(newUrl1);  // "https://production.example.com/staging/users"

// Замена всех вхождений:
let newUrl2: string = url.replaceAll("staging", "production");
console.log(newUrl2);  // "https://production.example.com/production/users"

// Оригинал не изменился:
console.log(url);  // "https://staging.example.com/staging/users"
```

**В тестировании:**

```typescript
let testData: string = "User_email@test.com";
let formatted: string = testData.replace("_", " ");
// "User email@test.com"
```

### split() — разбиение строки на массив

`split(delimiter)` — разбивает строку по разделителю и возвращает **массив** строк.

```typescript
// Разбить по пробелу:
let str: string = "Hello, TypeScript!";
let words: string[] = str.split(" ");
console.log(words);  // ["Hello,", "TypeScript!"]

// Разбить по запятой:
let credentials: string = "abc@gmail.com,xyz123";
let parts: string[] = credentials.split(",");
console.log(parts[0]);  // "abc@gmail.com"
console.log(parts[1]);  // "xyz123"

// Разбить каждый символ отдельно (пустая строка):
let chars: string[] = "ABC".split("");
console.log(chars);  // ["A", "B", "C"]
```

**В тестировании — очень часто:**

```typescript
// Список браузеров из строки конфига:
let browsers: string = "Chrome,Firefox,Safari";
let browserList: string[] = browsers.split(",");

for (let browser of browserList) {
  console.log(`Running tests on: ${browser}`);
}
// "Running tests on: Chrome"
// "Running tests on: Firefox"
// "Running tests on: Safari"
```

### trim(), trimStart(), trimEnd() — удаление пробелов

```typescript
let messy: string = "   welcome to TypeScript   ";

console.log(messy.trim());       // "welcome to TypeScript" — оба конца
console.log(messy.trimStart());  // "welcome to TypeScript   " — только начало
console.log(messy.trimEnd());    // "   welcome to TypeScript" — только конец
```

**В тестировании:** пользователи часто случайно вводят пробелы — нужно их убирать перед сравнением:

```typescript
let inputFromUser: string = "  john.doe@email.com  ";
let cleanEmail: string = inputFromUser.trim();
console.log(cleanEmail === "john.doe@email.com");  // true
```

### concat() — объединение строк

```typescript
let base: string = "https://api.example.com";
let path: string = "/users";
let full: string = base.concat(path);
console.log(full);  // "https://api.example.com/users"

// Цепочка concat:
let withParams: string = base.concat(path).concat("?limit=10");
console.log(withParams);  // "https://api.example.com/users?limit=10"
```

## Общие методы (isNaN, parseInt, parseFloat)

Эти методы не принадлежат строкам или числам — они глобальные.

### isNaN() — проверка "не число ли это"

`isNaN` расшифровывается как "is Not a Number" — "является ли не-числом".

```typescript
let num: number = 5;
let str: string = "hello";

console.log(isNaN(num));    // false — это число
console.log(isNaN(str));    // true — это не число

// В тестировании:
let responseTime: string = "350";
console.log(isNaN(Number(responseTime)));  // false — можно преобразовать
```

### parseInt() и parseFloat() — строка в число

```typescript
let strNum: string = "42";
let parsed: number = parseInt(strNum);
console.log(parsed);  // 42 (число)

let strFloat: string = "3.14";
let parsedFloat: number = parseFloat(strFloat);
console.log(parsedFloat);  // 3.14

// Если строка не число:
console.log(parseInt("hello"));  // NaN (Not a Number)

// В тестировании — парсинг значений из UI:
let priceText: string = "299.99";
let price: number = parseFloat(priceText);
console.log(price > 100);  // true
```

## Математические методы (Math)

`Math` — встроенный объект с математическими методами. Пишется с большой буквы.

```typescript
// Абсолютное значение:
console.log(Math.abs(-5));   // 5
console.log(Math.abs(5));    // 5

// Округление вверх (потолок):
console.log(Math.ceil(2.1));  // 3

// Округление вниз (пол):
console.log(Math.floor(2.9)); // 2

// Обычное округление:
console.log(Math.round(2.4)); // 2
console.log(Math.round(2.5)); // 3

// Случайное число от 0 до 1:
console.log(Math.random());   // например 0.73421...

// Максимальное безопасное целое число:
console.log(Number.MAX_SAFE_INTEGER);  // 9007199254740991
console.log(Number.MIN_SAFE_INTEGER);  // -9007199254740991
```

**В тестировании:**

```typescript
// Генерация случайного числа в диапазоне (например, порт для сервера):
let randomPort: number = Math.floor(Math.random() * 1000) + 3000;
console.log(randomPort);  // 3000-3999

// Округление времени ответа:
let responseTime: number = 320.7;
console.log(Math.round(responseTime));  // 321
```

## Что такое массив

**Массив (array)** — это тип данных, который хранит коллекцию значений в одной переменной.

**Проблема без массивов:**

```typescript
// Неудобно — много переменных:
let browser1: string = "Chrome";
let browser2: string = "Firefox";
let browser3: string = "Safari";
// Работать с ними одновременно неудобно
```

**Решение — массив:**

```typescript
// Удобно — одна переменная:
let browsers: string[] = ["Chrome", "Firefox", "Safari"];
```

**Аналогия:** массив — это как полка в библиотеке. У каждой книги есть своё место (индекс), начиная с 0.

Главные свойства массива:

- Хранит несколько значений
- Каждое значение доступно по индексу (с нуля!)
- Может расти и уменьшаться
- Может содержать разные типы данных (хотя в TypeScript лучше хранить один тип)

---

## Создание массивов

**Способ 1 — литеральный (рекомендуемый):**

```typescript
// Пустой массив:
let emptyArr: string[] = [];

// Массив со значениями:
let statuses: string[] = ["passed", "failed", "skipped"];
let browsers: string[] = ["Chrome", "Firefox", "Safari"];
let responseTimes: number[] = [320, 150, 480, 95];
```

**Способ 2 — через `new Array()`:**

```typescript
let arr1 = new Array();  // пустой
let arr2 = new Array("Chrome", "Firefox", "Safari");
```

Большинство разработчиков используют литеральный синтаксис (квадратные скобки) — он проще.

**Массив может содержать разные типы:**

```typescript
// В JavaScript можно смешивать (в TypeScript лучше так не делать):
let mixed = ["Chrome", 320, true, undefined];
```

**Массив из переменных:**

```typescript
let env1: string = "staging";
let env2: string = "production";
let env3: string = "dev";

let environments: string[] = [env1, env2, env3];
```

## Методы массивов

### push() — добавить в конец

Добавляет элемент в конец массива. Возвращает **новую длину** массива. Изменяет оригинальный массив.

```typescript
let results: string[] = ["passed", "passed"];
let newLength: number = results.push("failed");

console.log(results);    // ["passed", "passed", "failed"]
console.log(newLength);  // 3
```

### unshift() — добавить в начало

Добавляет элемент в начало. Возвращает **новую длину**. Изменяет оригинальный массив.

```typescript
let queue: string[] = ["task2", "task3"];
queue.unshift("task1");
console.log(queue);  // ["task1", "task2", "task3"]
```

### pop() — удалить из конца

Удаляет последний элемент. Возвращает **удалённый элемент**. Изменяет оригинальный массив.

```typescript
let tests: string[] = ["login", "checkout", "profile"];
let removed: string = tests.pop();

console.log(removed);  // "profile"
console.log(tests);    // ["login", "checkout"]
```

### shift() — удалить из начала

Удаляет первый элемент. Возвращает **удалённый элемент**. Изменяет оригинальный массив.

```typescript
let queue: string[] = ["task1", "task2", "task3"];
let first: string = queue.shift();

console.log(first);  // "task1"
console.log(queue);  // ["task2", "task3"]
```

### Итог — 4 основных метода:

|Метод|Действие|Возвращает|Меняет массив|
|---|---|---|---|
|`push`|Добавить в конец|Новую длину|Да|
|`unshift`|Добавить в начало|Новую длину|Да|
|`pop`|Удалить из конца|Удалённый элемент|Да|
|`shift`|Удалить из начала|Удалённый элемент|Да|

### at() — получить элемент по индексу (с поддержкой отрицательных)

```typescript
let arr: string[] = ["a", "b", "c"];

console.log(arr.at(0));   // "a"
console.log(arr.at(-1));  // "c" — последний элемент
console.log(arr.at(-2));  // "b" — предпоследний
console.log(arr.at(10));  // undefined
```

**Удобнее для последнего элемента, чем `arr[arr.length - 1]`!**

### toString() — массив в строку

```typescript
let browsers: string[] = ["Chrome", "Firefox", "Safari"];
let str: string = browsers.toString();
console.log(str);  // "Chrome,Firefox,Safari"
```

## Многомерные массивы

Массив может содержать другие массивы. Это называется многомерный (вложенный) массив.

```typescript
// Двумерный массив:
let testResults = [
  ["login", "passed", "320ms"],
  ["checkout", "failed", "timeout"],
  ["profile", "passed", "150ms"]
];
```

**Обращение к элементам:** первые скобки — индекс внешнего массива, вторые — внутреннего.

```typescript
console.log(testResults[0]);       // ["login", "passed", "320ms"]
console.log(testResults[0][0]);    // "login"
console.log(testResults[1][1]);    // "failed"
console.log(testResults[2][2]);    // "150ms"
```

**Изменение элемента:**

```typescript
testResults[1][1] = "passed";  // исправили статус второго теста
```

Обычно используют двумерные массивы. Глубже — редко, и это усложняет код.

## Связь строк и массивов (split и join)

Строки и массивы тесно связаны. Их можно конвертировать друг в друга.

### split() — строка в массив

```typescript
let str: string = "Hello World";

// Без параметра — весь текст как один элемент:
console.log(str.split());      // ["Hello World"]

// По пробелу:
console.log(str.split(" "));   // ["Hello", "World"]

// По каждому символу:
console.log(str.split(""));    // ["H","e","l","l","o"," ","W","o","r","l","d"]

// По букве "o":
console.log(str.split("o"));   // ["Hell", " W", "rld"]
```

**В тестировании:**

```typescript
let envList: string = "staging,production,dev";
let envs: string[] = envList.split(",");

for (let env of envs) {
  console.log(`Testing on: ${env}`);
}
```

### join() — массив в строку

```typescript
let browsers: string[] = ["Chrome", "Firefox", "Safari"];

// Без параметра — с запятой:
console.log(browsers.join());    // "Chrome,Firefox,Safari"

// С пробелом:
console.log(browsers.join(" ")); // "Chrome Firefox Safari"

// С тире:
console.log(browsers.join("-")); // "Chrome-Firefox-Safari"

// С пустой строкой:
console.log(browsers.join(""));  // "ChromeFirefoxSafari"
```

**В тестировании:**

```typescript
let failedTests: string[] = ["login", "checkout", "search"];
let report: string = failedTests.join(", ");
console.log(`Failed tests: ${report}`);
// "Failed tests: login, checkout, search"
```
