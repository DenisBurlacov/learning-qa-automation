## Как работать с этим файлом

1. Читаешь условие задачи
2. Пишешь решение самостоятельно в VS Code
3. Запускаешь: `npx playwright test имя_файла.spec.ts --headed`
4. Если застрял — возвращаешься в NOTES, а не в РЕШЕНИЯ
5. Проверяешь решение только после своей попытки

## Блок 1 — has-text фильтрация

### Задача 1.1

**Название:** CSS has-text — найти ячейку по содержимому

**Условие:** Открой страницу `https://practice.expandtesting.com/dynamic-table`. В таблице есть несколько строк. Для первой строки таблицы найди ячейку (`td`), которая содержит символ `%`. Выведи её текст в консоль. Используй CSS-синтаксис `td:has-text("%")`.

**Ожидаемый вывод:**

```
CPU первой строки: X.X%
```

---

### Задача 1.2

**Название:** Playwright-синтаксис hasText

**Условие:** Сделай то же самое что в задаче 1.1, но используй Playwright-синтаксис `{ hasText: "%" }` вместо CSS `has-text`. Убедись что результат одинаковый.

**Ожидаемый вывод:**

```
CSS вариант: X.X%
Playwright вариант: X.X%
Результаты совпадают: true/false
```

---

### Задача 1.3

**Название:** has-text для разных единиц измерения

**Условие:** На той же странице для строки "Chrome" найди и выведи:

- ячейку с `%` (CPU)
- ячейку с `MB` (Memory)

Для каждой — используй отдельный `has-text` локатор. Не используй жёсткие индексы `nth()`.

**Ожидаемый вывод:**

```

Chrome Memory: XXMB
```

### Задача 6.3

**Название:** while vs for — когда что выбрать

**Условие:** Напиши два варианта кода (без браузера):

Вариант 1 — симуляция пагинации через `while`:

- `let page = 1`, `let hasMore = true`
- В цикле: вывести "Читаем страницу N", увеличить счётчик
- Выйти когда page достигнет 6

Вариант 2 — то же самое через `for`:

- `for (let page = 1; page <= 6; page++)`

Объясни: когда нужен `while` (не знаем заранее сколько итераций), а когда `for` (знаем заранее).

**Ожидаемый вывод:**

```

Читаем страницу 2
...
Читаем страницу 6
Завершили.
```

4. Почему переменную `cpuLoad` нужно объявлять ВНЕ цикла `for...of`?
    
5. Что такое scope (область видимости)? Чем отличается `const` внутри цикла от `let` снаружи?
    
6. Почему для обхода пагинации используется `while`, а не `for`?

## Ответы на контрольные вопросы

**1. Почему нельзя nth(2) для динамической таблицы** Потому что порядок колонок меняется при каждом обновлении страницы. CPU может быть в колонке 1, 2, 3 или 4. Жёсткий индекс будет правильным только в один из четырёх случаев.

**2. Что делает td:has-text("%")** Находит среди всех `td` в данном контексте ту ячейку, которая содержит символ `%`. Это фильтрация по содержимому, а не по позиции. В отличие от `nth(2)` работает независимо от порядка колонок.

**3. Два синтаксиса has-text** CSS-синтаксис: `td:has-text("%")` — это чистый CSS псевдокласс. Playwright-синтаксис: `locator("td", { hasText: "%" })` — это опция фильтрации в Playwright API. Оба дают одинаковый результат.

**4. Переменная cpuLoad вне цикла** Потому что переменная живёт только внутри блока `{}` где объявлена. Если объявить `let cpuLoad` внутри цикла — после выхода из цикла переменная перестаёт существовать, и использовать её для ассерции невозможно.

**5. Scope** Scope — это область где существует переменная. `const` внутри цикла создаётся заново на каждой итерации и уничтожается в её конце. `let` снаружи живёт пока не закончится функция или блок где объявлена.

**6. Почему while для пагинации** Потому что количество страниц неизвестно заранее — зависит от данных в таблице. `while` выполняется пока условие истинно, без фиксированного числа итераций. `for` используют когда диапазон итераций известен заранее.

**7. Как определить disabled кнопку** Смотрим значение атрибута `class`. Когда кнопка активна: `class="paginate_button next"`. Когда заблокирована: `class="paginate_button next disabled"`. Разработчики добавляют слово `disabled` в класс. Мы проверяем это через `getAttribute("class")?.includes("disabled")`.

**8. getAttribute возвращает** `string | null` — строку с значением атрибута или `null` если атрибут отсутствует. Поэтому TypeScript требует тип `string | null` для переменной.

**9. Optional chaining** `?.` перед методом говорит: если значение слева равно `null` или `undefined` — не вызывай метод, просто верни `undefined`. Без `?.` вызов метода на `null` даст `TypeError: Cannot read properties of null`.

**10. toContain vs includes** `.includes()` — метод строки JavaScript, возвращает `boolean`, используется в условной логике (`if`). `toContain` — ассерция Playwright/Jest, прерывает тест с ошибкой если условие не выполняется. Используй `includes` для ветвления кода, `toContain` для проверок в тестах.

**11. toBe(true) vs toBeTruthy** `toBe(true)` — строгое равенство: значение должно быть именно булевым `true`. `toBeTruthy()` — значение должно быть "правдивым": `true`, ненулевое число, непустая строка, массив и т.д. Для булевых переменных оба эквивалентны. `toBe(true)` точнее и рекомендуется.

**12. parseFloat с replace** `"$200.98"` — это строка с символом `$`. `parseFloat` не может парсить строки начинающиеся с символов. `.replace("$", "")` убирает `$` и даёт `"200.98"`. Затем `parseFloat("200.98")` даёт `200.98`. Последовательность: убрать лишнее — конвертировать в число.

**13. break vs hasMorePages** `break` немедленно выходит из цикла — полезно когда нашли что искали и дальше повторять бессмысленно (поиск Chrome). `hasMorePages = false` не прерывает текущую итерацию, но предотвращает следующую — цикл завершится естественно после текущего прохода (пагинация: мы на последней странице, уже прочитали её данные, теперь просто не входим снова).

**14. has-text находит несколько элементов** Если есть несколько `td` с `%` — локатор вернёт их все. Тогда `.innerText()` выбросит ошибку "strict mode: multiple elements found". Решение: использовать `.first()` или более специфичный фильтр.

**15. Зачем проверять rows.length** `for...of` по пустому массиву просто не выполняется — это не ошибка. Но без проверки мы не знаем почему `matchFound` остался `false`: записи нет или поиск вернул ноль строк. Проверка `rows.length` позволяет напечатать понятное сообщение и разграничить эти случаи.
